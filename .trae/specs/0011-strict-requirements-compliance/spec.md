# 0011 严格需求合规实现 - Product Requirement Document

## Overview
- **Summary**: 严格按照《新任务.md》的12条要求（编号1/3/4/5/7/8/9/10/11/12/13/14）全面修复和完善GerClaw系统，确保每个功能都真实可用而非摆设，彻底解决之前未完成的任务和擅自添加的功能。
- **Purpose**: 解决用户反馈的"任务未完成、擅自添加老年模式普通聊天自动播放"等问题，使系统完全符合需求文档要求，所有按钮功能真实可用，交互体验对齐豆包应用。
- **Target Users**: 老年患者、老年科医生，访客模式用户。

## Goals
- 所有消息操作按钮（复制/语音播放/分享/赞/踩/重新生成/转为文档编辑/删除）功能完整实现且真实可用
- 加载动画优化为豆包风格：文本加载用三点动画（速率适中），思考用转圈动画（速率适中）
- 联网搜索结果链接可真实跳转至原文网页
- 所有聊天回复默认不自动播放TTS（包括老年模式普通聊天），仅CGA老年综合评估内可自动播放
- CGA老年综合评估：使用预录音频（或TTS预生成）、支持abcd/数字/文本语音答题、识别后不自动跳题、播放状态记忆、完成后右侧栏不自动弹出、三按钮操作页、支持多选量表批量作答、进度保持和退出逻辑正确
- 五大处方生成：固定欢迎语（不调LLM）、文件上传+MinerU解析、信息缺失时对话卡片（每轮4题，最多3轮）、先生成健康画像再到处方、本地知识库优先检索、PII隐私脱敏、右侧栏Markdown编辑/复制/导出
- 文件上传支持最多8个文件，文件信息真实通过MinerU提取并传给模型
- 所有导出按钮支持pdf/docx/markdown三种格式，分享按钮额外支持png
- 任务完成后Agent必须自行使用浏览器逐条核查所有需求，不符合立即修改直到全部通过

## Non-Goals (Out of Scope)
- 不新增需求文档以外的功能
- 不重构已有且符合需求的架构
- 不修改设计要求文档中已确定的UI规范（除需求明确要求的）

## Background & Context
- 之前多轮迭代未完全满足需求，存在按钮摆设、自动播放TTS、CGA流程bug、五大处方流程不完整等问题
- 用户明确要求"不懂不理解的地方要沟通对齐"、"任务完成后自行浏览器核查"
- 技术栈：Next.js 16 + React 19 + TypeScript + Tailwind CSS + shadcn/ui + Zustand

## Functional Requirements

### FR-1: 消息操作按钮完整实现（对应需求1）
- **FR-1.1**: 复制按钮：点击将AI最终回复纯文本复制到剪贴板，显示成功提示
- **FR-1.2**: 语音播放按钮：点击播放AI回复TTS，再次点击停止；默认不自动播放（普通聊天和老年模式普通聊天均不自动播放）；播放中按钮显示停止状态
- **FR-1.3**: 分享按钮：点击弹出导出对话框，默认勾选「该AI回复+对应的用户提问」（一组问答），用户可勾选/取消其他消息；支持PNG/PDF/DOCX/Markdown四种格式导出
- **FR-1.4**: 点赞按钮：点击可点赞+可选填评价反馈，再次点击取消；赞/踩互斥
- **FR-1.5**: 点踩按钮：点击可点踩+可选填问题反馈，再次点击取消；赞/踩互斥
- **FR-1.6**: 重新生成按钮：仅在最后一条AI消息且已完成时显示，点击重新生成该回复
- **FR-1.7**: 转为文档编辑按钮：放在三点菜单中，点击将该AI回复渲染到右侧栏Markdown编辑器，可编辑、复制、导出
- **FR-1.8**: 删除按钮：放在三点菜单中，点击弹出删除确认对话框，默认勾选「该AI回复+对应的用户提问」，用户可勾选/取消其他消息；有取消按钮，确认后删除且不进入后续模型上下文

### FR-2: 加载动画优化（对应需求3）
- **FR-2.1**: 普通文本加载（模型输出文字前的占位）：三点跳动动画，样式对齐豆包，速率适中（1.2-1.4s周期）
- **FR-2.2**: 模型思考中：转圈spinner动画，速率适中（1.5s周期，不要太快）
- **FR-2.3**: 停止按钮：静态显示，无脉冲闪烁动画

### FR-3: 搜索结果链接跳转（对应需求4）
- **FR-3.1**: 联网搜索返回的每个结果标题链接，点击必须在新标签页打开对应网页
- **FR-3.2**: 链接使用`<a target="_blank" rel="noopener noreferrer">`，不阻止默认跳转行为

### FR-4: TTS自动播放严格控制（对应需求5）
- **FR-4.1**: 普通聊天（包括老年模式）：所有AI回复默认不自动播放TTS，只有用户点击播放按钮才播放
- **FR-4.2**: CGA老年综合评估内：可以自动播放题目音频，播放状态记忆用户选择
- **FR-4.3**: 彻底移除所有普通聊天场景的autoReadIfSeniorMode/老年模式自动TTS逻辑

### FR-5: CGA老年综合评估完善（对应需求7/8/9/10/11）
- **FR-5.1**: CGA题目和选项音频：优先使用预录音频文件，若无预录文件则用TTS预生成缓存；播放按钮状态正常（播放中显示停止按钮）
- **FR-5.2**: 语音答题识别：支持识别选项文本、选项数字（如选项1）、选项abcd（如选项a、a）
- **FR-5.3**: 选择确认：语音识别或手动选择选项后，不自动跳转到下一题，用户需自行点击"下一题"按钮确认
- **FR-5.4**: 播放状态记忆：用户选择关闭/打开播放后，后续题目维持该选择；切换题目时若用户未关闭则播放新题目音频，返回量表选择页停止播放
- **FR-5.5**: 量表选择界面：支持多选、全选按钮；已完成的量表显示"已作答"状态且不可勾选；所有量表完成后显示"所有量表已作答完毕"提示和"生成评估报告"按钮
- **FR-5.6**: 批量作答：用户选择多个量表后，按量表顺序一次性答完所有题目，中途无量表切换
- **FR-5.7**: 完成页修复：完成量表评估后右侧栏不自动弹出；中间栏显示"作答完毕"和已完成量表列表，三个按钮并列：「重新评估」、「继续作答其他量表」、「查看已作答量表评估报告」
- **FR-5.8**: 评估报告：点击查看报告后，所有已完成量表结果整合由模型生成报告，在右侧栏Markdown编辑器显示，可编辑、复制、导出（PDF/DOCX/Markdown）
- **FR-5.9**: 继续作答其他量表：返回量表选择页，已完成量表不可选，可选择未完成量表
- **FR-5.10**: 进度保持：只要用户不退出CGA，保持所有作答进度；生成报告时整合所有历史和当前作答结果
- **FR-5.11**: 退出逻辑：无作答/无报告时退出显示"退出后当前进度将不会保存，确认退出吗？"；已有报告时退出显示"您已完成量表评估，是否要退出？"，确认退出则保存最近一次报告
- **FR-5.12**: 历史对话恢复：从历史对话进入之前的CGA对话时，显示上一次的评估报告，可选择继续作答/重新作答/查阅报告
- **FR-5.13**: 量表题数正确：PHQ-9共9题，GAD-7共7题，PSQI共7题，Mini-Cog共5题（简化版），MMSE共30题

### FR-6: 五大处方生成完善（对应需求12）
- **FR-6.1**: 固定欢迎语：进入五大处方功能立即显示固定欢迎语（不调用LLM），说明功能和使用方法
- **FR-6.2**: 文件上传支持：支持图片/PDF/Markdown/DOCX格式，最多8个文件
- **FR-6.3**: MinerU API集成：上传文件后调用MinerU API解析提取信息，若API未配置则降级为提示用户或直接读取文本
- **FR-6.4**: 信息收集对话卡片：信息缺失时在聊天区显示对话卡片（参照Trae Work样式），每轮最多4个问题，支持文本/语音输入；最多3轮收集，超过3轮用已有信息生成
- **FR-6.5**: 诊疗顺序：先生成用户健康画像，再根据健康画像生成五大处方（运动/营养/心理/用药/戒烟限酒）
- **FR-6.6**: 用药审查：生成处方时检查药物相互作用，确保处方不冲突、临床可行、安全无害
- **FR-6.7**: 知识库优先：优先检索本地知识库（`/Users/qizs/conclusion/gerclaw/本地知识库/md`），找不到再联网搜索补充
- **FR-6.8**: 结构化输出：参照五大处方报告模板，生成JSON格式结构化输出，校验后转为Markdown
- **FR-6.9**: 右侧栏展示：生成的五大处方Markdown在右侧栏编辑器显示，可编辑、复制、导出（PDF/DOCX/Markdown）
- **FR-6.10**: PII隐私脱敏：用户输入信息（姓名/身份证/医保号/手机号/住院号）在传给LLM前正则脱敏
- **FR-6.11**: 输入模板：参照hzj_case.json作为输入模板收集信息

### FR-7: 文件上传修复（对应需求13）
- **FR-7.1**: 支持最多8个文件同时上传，修复只能加载5个的bug
- **FR-7.2**: 上传的文件必须通过MinerU API解析提取信息，并将提取的信息整合后作为上下文传给模型

### FR-8: 导出格式统一（对应需求14）
- **FR-8.1**: 所有导出按钮（右侧栏编辑器导出、评估报告导出、五大处方导出等）一律支持PDF、DOCX、Markdown三种格式
- **FR-8.2**: 分享（单条消息导出）额外支持PNG格式

## Non-Functional Requirements
- **NFR-1**: 所有功能真实可用，按钮点击有实际效果，不能是摆设
- **NFR-2**: 控制台无React错误、无hydration错误、无Audio playback error
- **NFR-3**: npm run lint 0错误0警告
- **NFR-4**: npm run build成功，TypeScript检查通过
- **NFR-5**: 适老化规范：老年模式按钮≥48px，正文≥18px（默认已18px，老年模式20px）
- **NFR-6**: 医疗安全：所有医疗输出带免责声明，高风险症状提示立即就医
- **NFR-7**: 配置不硬编码：API Key/URL通过环境变量

## Acceptance Criteria

### AC-1: 消息操作按钮全部可用
- **Given**: 用户进入聊天页面，有AI回复消息
- **When**: 鼠标悬停在AI消息上或在老年模式下

- **Verification**: `human-judgment` + `programmatic`（点击测试）
- **Notes**: 三点菜单展开后有"转为文档编辑"和"删除"选项

### AC-2: 复制功能正常
- **Given**: 有AI回复
