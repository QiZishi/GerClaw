## 1. 设计理念

GerClaw的设计理念是**专业可信 + 温暖亲和 + 适老优先**：

- **专业可信**：对齐Trae Work的AI IDE交互体验，三栏布局、智能体执行过程可视化、工具调用透明展示、循证引用可追溯
- **温暖亲和**：医疗健康产品不是冷冰冰的工具，语音合成用"冰糖"温柔女声，配色使用医疗蓝+健康绿，消息气泡圆角柔和
- **适老优先**：患者端默认老年模式，以"奶奶也能看懂、爷爷也能操作"为标准——大字体、大按钮、高对比度、语音一句话就能用
- **清晰不花哨**：医疗产品以信息清晰度为第一优先级，动画精简（仅加载/状态过渡），不做炫技效果，深色/浅色主题切换不影响可读性

## 3. 交互规范

### 3.1 反馈原则

- **即时反馈**：用户点击/说话后100ms内给出视觉反馈（按钮按下态、录音波形动画）
- **加载指示**：AI思考超过1s显示"思考中..."动画；流式输出首token等待时显示呼吸点动画
- **成功/失败提示**：操作成功用绿色toast（2s自动消失），错误用红色toast+错误描述+解决建议
- **危险确认**：删除对话、清除历史、导出重要报告前需二次确认对话框
- **语音反馈**：录音开始/结束有轻微提示音和震动（如设备支持），TTS播放有可视化进度条

### 3.2 错误处理

- 错误信息用老人能理解的自然语言："网络连接有点问题，请检查网络后重试"，不显示"TypeError: Failed to fetch"
- 每个错误都提供解决建议或重试按钮
- 表单/输入错误：inline显示在对应输入框下方，红色文字+图标
- 系统错误：顶部横幅提示+错误追踪ID（便于排查），不阻塞用户查看已有内容
- API降级提示：主模型切换到备用模型时，不打断对话，控制台记录即可；所有搜索不可用时在消息中注明

### 3.3 状态设计

每个界面/组件必须处理四种状态：

1. **主流程（Happy Path）**：正常数据展示，流式输出中
2. **加载中**：
   - 首屏：骨架屏（Skeleton）占位
   - AI回复：打字机光标闪烁
   - 语音识别：实时波形+转写预览
3. **为空**：
   - 新对话：欢迎页（功能快捷入口：开始对话、CGA评估、生成处方、语音咨询）
   - 会话列表空：提示"开始您的第一次健康咨询"
   - 搜索无结果：提示"未找到相关医学信息，建议换个关键词或咨询医生"
4. **错误**：友好错误提示+重试按钮+离线检测

### 3.4 消息交互

- 用户消息：右侧蓝色气泡，右对齐
- AI消息：左侧白色卡片（浅色模式）/深灰卡片（深色模式），左对齐，含头像
- 悬停消息显示操作按钮：复制、重新生成（仅AI）、语音朗读、导出（仅医疗报告）
- 代码块：语法高亮+复制按钮+语言标签
- Markdown表格：支持横向滚动，表头固定
- 引用角标：[1][2]蓝色上标，点击展开引用卡片或跳转到右侧面板

**智能体执行过程可视化（对齐 §4.2.3，共 7 项）**：
- 思维链：可折叠"思考过程"区块（默认折叠，浅灰背景，"思考中..."动画）
- 工具调用：独立卡片，工具图标+名称+状态徽章（旋转中→✓→✗），展开显示参数/结果/耗时
- 子智能体加载：折叠树形结构，展示主智能体→子智能体调用链
- 决策过程：垂直时间线，每步图标+名称+状态+耗时，当前步高亮动画
- 联网搜索结果：独立卡片，标题+来源 favicon+摘要+链接，AI正文 [1][2] 角标引用
- 文档解析状态：文件标签+工具卡片双重反馈（上传中→解析中→完成/失败）
- 流式文本输出：打字机效果，末尾光标闪烁，实时 Markdown 渲染

## 4. 响应式设计

- 桌面端（≥1280px）：完整三栏布局
- 平板（768-1279px）：默认收起右侧面板，左侧边栏可折叠，主聊天区优先
- 移动端（<768px）：单栏布局，侧边栏通过抽屉(Sheet)展开，右侧面板覆盖层展示
- 断点策略：使用Tailwind默认断点（sm:640px/md:768px/lg:1024px/xl:1280px/2xl:1536px）
- 老年模式在移动端额外优化：按钮更大(≥56px)，间距更宽松

## 5. 无障碍（A11y）

- 语义化HTML标签：header/main/nav/aside/section/article/button等语义标签优先使用
- 图片有alt文本，装饰性图片alt为空字符串
- 键盘可导航：Tab顺序合理，Enter激活按钮，Esc关闭弹窗/面板，空格播放/暂停TTS
- 颜色对比度≥4.5:1（WCAG AA），老年模式≥7:1（WCAG AAA）
- ARIA标签用于复杂组件（角色切换、折叠面板、工具调用卡片、录音按钮状态）
- 表单控件有关联label
- 焦点可见：focus-visible样式明显（蓝色外发光）
- 减少动画偏好：尊重`prefers-reduced-motion`系统设置，老年模式减少动画
- 屏幕阅读器：录音状态、AI回复进度、新消息到达有aria-live区域播报

## 6. 双主题规范

- 默认浅色主题
- 深色主题跟随系统或手动切换
- 两套主题颜色通过CSS变量/Tailwind dark:前缀定义
- 老年模式强制浅色高对比度（即使系统深色模式）
- 主题切换有0.2s过渡动画（尊重reduced-motion）
