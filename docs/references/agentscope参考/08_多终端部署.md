## 1. 模块映射总览

GerClaw 面向 **Web（Next.js）/ 移动端 App（React Native）/ 微信小程序 / 智能音箱** 四类终端提供统一医疗对话服务，核心部署诉求与 AgentScope Agent Service 的对应关系如下：

| GerClaw 需求 | AgentScope 对应能力 | 关键模块/API |
|---|---|---|
| 统一 HTTP API 接入层 | `create_app()` FastAPI 工厂 | `agentscope.app.create_app` |
| 多终端 SSE 流式响应 | `/sessions/{id}/stream` SSE 端点 + 心跳保活 | `_router/_session.py`、`_service/_chat.py` |
| 非流式对话接口（音箱/小程序低带宽场景） | `POST /chat` fire-and-forget + 消息查询 | `_router/_chat.py`、`/sessions/{id}/messages` |
| 批量问诊/批量审查 | 多 session 并发触发 + 后台任务 | `BackgroundTaskManager`、`asyncio.gather` |
| 多租户隔离（家庭/社区/医院） | `X-User-ID` 头 + `extra_agent_middlewares` 工厂 + Redis key 按 `user_id` 分区 | `deps.py`、`storage/_redis_storage.py` |
| 会话持久化与续接 | `SessionService` + Redis 持久化（滑动 TTL） | `_service/_session.py`、`RedisStorage` |
| 跨终端会话同步 | SSE 缓冲回放（多订阅者扇出 + 历史重放） | `SessionProjection`、message bus |
| 模型凭证管理 | Credential CRUD API | `_router/_credential.py` |
| CORS 跨域（Web/小程序） | `extra_middlewares` 注入 `CORSMiddleware` | FastAPI `CORSMiddleware` |
| 高并发分布式部署 | RedisStorage + RedisMessageBus + 独立 worker 进程 | `message_bus/_redis_message_bus.py`、`rag/index_worker` |
| 协议适配（AG-UI/A2A） | `ProtocolMiddlewareBase` 可插拔协议转换 | `middleware/_protocol/_base.py` |

**部署拓扑**（Agent Service 推荐模式 → GerClaw 映射）：

```
[Web/App/小程序/音箱]
        │  HTTP/SSE
        ▼
[API 网关: APISIX/Kong]  ← JWT鉴权、限流、CORS、灰度
        │
        ▼
[Agent Service (FastAPI, uvicorn)]
  ├─ storage: RedisStorage          ← 会话/凭证/Agent元数据
  ├─ message_bus: RedisMessageBus   ← 事件流/唤醒/收件箱
  ├─ workspace_manager: Local/DockerWorkspaceManager
  ├─ knowledge_base_manager: CollectionPerKbManager  ← RAG
  └─ extra_middlewares: [CORSMiddleware, AuthMiddleware]
        │
        ▼
[LiteLLM Proxy / 直接 LLM API]  ← 豆包/DeepSeek/通义/vLLM
```

## 2. 核心 API 参考

### 2.1 `create_app` — FastAPI 应用工厂

**入口**：`agentscope.app.create_app`
**源码**：`src/agentscope/app/_app.py`

```python
from agentscope.app import create_app

def create_app(
    storage: StorageBase,                          # 必填：持久化后端
    message_bus: MessageBus,                       # 必填：消息总线
    workspace_manager: WorkspaceManagerBase,       # 必填：工作区管理器
    knowledge_base_manager: KnowledgeBaseManagerBase | None = None,
    knowledge_parsers: list[ParserBase] | dict | None = None,
    knowledge_chunker: ChunkerBase | None = None,
    blob_store: BlobStoreBase | None = None,
    enable_index_worker: bool = True,              # True=嵌入式索引; False=分布式worker
    *,
    extra_credentials: list[Type[CredentialBase]] | None = None,
    extra_middlewares: list[Middleware] | None = None,     # CORS/鉴权中间件
    extra_agent_middlewares: AgentMiddlewareFactory | None = None,  # 多租户钩子
    extra_agent_tools: AgentToolFactory | None = None,
    custom_subagent_templates: list[SubAgentTemplate] | None = None,
    custom_agent_cls: Type[Agent] | None = None,
    title: str = "AgentScope",
    version: str = "2.0.0",
) -> FastAPI
```

**关键参数说明**：

| 参数 | 用途 | GerClaw 配置建议 |
|---|---|---|
| `storage` | Agent/Session/Credential/Schedule 持久化 | `RedisStorage(host=..., port=6379, key_ttl=86400*7)` |
| `message_bus` | 会话事件扇出、唤醒信号、收件箱队列 | 生产用 `RedisMessageBus`；原型可 `InMemoryMessageBus` |
| `workspace_manager` | 工作区生命周期 | 医疗场景优先 `LocalWorkspaceManager`（数据合规） |
| `extra_middlewares` | ASGI 中间件链 | 必加 `CORSMiddleware`；可加自定义鉴权/Tracing |
| `extra_agent_middlewares` | 每次组装 Agent 时的工厂函数 | 用于注入租户级 RAG/审计/限流中间件 |
| `enable_index_worker` | 是否在 API 进程内跑索引 worker | 单进程 `True`；分布式 `False` |

### 2.2 Agent 生命周期管理（AgentManager 内建于 ChatService）

Agent Service 不暴露独立 `AgentManager` 类，Agent 生命周期由 `ChatService`（`_service/_chat.py`）在每次 chat run 时动态组装：

1. **创建 Agent 模板**：`POST /agent` → 存储 `AgentRecord`（name、system_prompt、config）
2. **创建会话**：`POST /sessions` → 生成 `session_id`，绑定 `(user_id, agent_id, workspace_id)`
3. **触发对话**：`POST /chat` → `ChatService` 运行时：
   - 通过 `extra_agent_middlewares` 工厂注入租户级中间件
   - 通过 `workspace_manager.get_workspace()` 获取/创建工作区
   - 组装 `Toolkit`（基础工具 + MCP + skill + `extra_agent_tools`）
   - 创建 `Agent` 实例，调用 `reply_stream()` 产出事件流
4. **事件发布**：事件通过 message bus 扇出到所有 SSE 订阅者
5. **会话缓存**：`ChatRunRegistry` 保证单 session 同时只有一个 run（防并发，409 Conflict）

### 2.3 `/api/chat` 非流式接口（fire-and-forget）

**端点**：`POST /chat`
**源码**：`_router/_chat.py`、`_service/_chat.py`

请求体：
```json
{
  "agent_id": "string (必填)",
  "session_id": "string (必填)",
  "input": {
    "name": "user_name",
    "role": "user",
    "content": [{"type": "text", "text": "老人症状描述..."}]
  }
}
```

响应（立即返回）：
```json
{ "status": "started", "session_id": "session-xxx" }
```

**适用场景**：智能音箱语音输入、小程序后台批量问诊（不关心逐字输出，等待完整结果后通过 `/sessions/{id}/messages` 拉取）。

> 注意：Agent Service 原生端点为 `/chat`（无 `/api` 前缀）。挂载到 GerClaw 网关时可用 `root.mount("/api", agentscope_app)` 加前缀。

### 2.4 `/api/chat/stream` SSE 流式接口

**端点**：`GET /sessions/{session_id}/stream?agent_id=xxx`
**源码**：`_router/_session.py`

- Header：`X-User-ID: {user_id}`（必填）
- 返回：`text/event-stream`
- 心跳：每 30 秒发送 `:\n\n` 注释帧，防止中间层断开
- 回放：新订阅者先收到缓冲历史事件（多 tab/重连同步）
- 事件类型：`TextBlockDeltaEvent`（逐字输出）、`ToolCallStartEvent`、`ToolResultEndEvent`、`HintBlockEvent`、`ReplyEndEvent` 等

SSE 帧格式：
```
event: message
data: {"type":"text_delta","data":{"delta":"您","block_id":"..."}}

event: done
data: [DONE]
```

**Nginx/Ingress 配置要点**（GerClaw 必须）：
```
proxy_read_timeout 300s;
proxy_send_timeout 300s;
proxy_buffering off;
add_header X-Accel-Buffering no;
```

### 2.5 批量接口（Batch）

Agent Service 无独立 `/chat/batch` 端点，批量能力通过以下方式组合：

1. **多 session 并发**：客户端并发调用多个 `POST /chat`（不同 session_id），通过 `asyncio.Semaphore` 控制并发度
2. **后台任务**：`BackgroundTaskManager` 支持长耗时工具调用切后台
3. **Cron 调度**：`POST /schedule` 定时批量触发（如批量用药提醒、批量健康报告生成）

**客户端批量调用模式**：
```python
import asyncio
import httpx

async def batch_chat(queries: list[str], agent_id: str, user_id: str, concurrency: int = 5):
    sem = asyncio.Semaphore(concurrency)
    async def one(q: str):
        async with sem:
            session_id = await create_session(agent_id)
            async with httpx.AsyncClient() as client:
                await client.post(f"{BASE}/chat", json={
                    "agent_id": agent_id, "session_id": session_id,
                    "input": {"name": user_id, "role": "user", "content": [{"type":"text","text":q}]}
                }, headers={"X-User-ID": user_id})
            return session_id
    return await asyncio.gather(*[one(q) for q in queries])
```

### 2.6 多租户 `tenant_id` 隔离

Agent Service 的多租户隔离分三层：

| 层 | 机制 | 粒度 |
|---|---|---|
| **存储层** | Redis key 模板含 `{user_id}`，天然隔离 | 所有持久化数据（agent/session/credential/KB） |
| **请求层** | `X-User-ID` Header → `get_current_user_id` 依赖，所有路由校验 | API 访问隔离 |
| **运行时层** | `extra_agent_middlewares` 工厂 `(user_id, agent_id, session_id) -> list[MiddlewareBase]`，每次组装 Agent 时调用 | 可注入租户专属 RAG 知识库、审计日志、限流 |

**GerClaw 多租户映射**：将 `X-User-ID` 扩展为 `{tenant_type}:{tenant_id}:{user_id}` 格式（如 `family:f001:elder_zhang`、`hospital:h002:dr_wang`），在 `extra_agent_middlewares` 中解析后注入对应租户的知识库。

### 2.7 会话管理 `session_id`

- 创建：`POST /sessions` → 返回 `session_id`
- 约束：同一 `(user_id, agent_id, workspace_id)` 三元组最多一个活跃 session（重复调用返回已有）
- 续接：后续 `POST /chat` 携带相同 `session_id` 继续多轮
- 历史：`GET /sessions/{id}/messages?offset=0&limit=50` 分页拉取
- 删除：`DELETE /sessions/{id}` 级联清理消息/team 引用
- 消息持久化：通过 `RedisStorage.upsert_message` 追加到 Redis List，相同 id 去重

### 2.8 CORS 配置

通过 `extra_middlewares` 注入 FastAPI 官方 `CORSMiddleware`：

```python
from fastapi.middleware import Middleware
from fastapi.middleware.cors import CORSMiddleware

app = create_app(
    ...,
    extra_middlewares=[
        Middleware(
            CORSMiddleware,
            allow_origins=[
                "https://app.gerclaw.com",      # Web
                "https://app.gerclaw.cn",       # 国内
                "capacitor://localhost",         # App 内嵌 WebView
                "https://servicewechat.com",     # 微信小程序
            ],
            allow_credentials=True,
            allow_methods=["GET", "POST", "PATCH", "DELETE"],
            allow_headers=["*"],
        ),
    ],
)
```

> 小程序端需在后台配置 request 合法域名；音箱端通常走后端中转不需要 CORS。

### 2.9 部署模式

| 模式 | 组件拓扑 | 适用场景 |
|---|---|---|
| **单进程** | API + IndexWorker 同进程；`InMemoryMessageBus` 或 Redis；`enable_index_worker=True` | 本地开发、原型、轻流量（<50 并发） |
| **单节点多 worker** | 多个 uvicorn worker + RedisMessageBus + RedisStorage | 单机生产（200-500 QPS） |
| **分布式（推荐生产）** | API 副本（`enable_index_worker=False`）+ 独立 RAG worker 进程 + Redis 集群 + K8s HPA | GerClaw 万级 DAU 生产环境 |

**分布式 worker 启动**：
```bash
AGENTSCOPE_WORKER_BOOTSTRAP=gerclaw.deploy.worker_bootstrap:bootstrap \
  python -m agentscope.app.rag.index_worker
```

## 6. GerClaw 适配要点

### 6.1 多终端统一接入层：HTTP/WebSocket/SSE

GerClaw 四类终端接入建议：

| 终端 | 协议 | 端点模式 | 特殊处理 |
|---|---|---|---|
| **Web（Next.js）** | SSE over HTTP/2 | `GET /sessions/{id}/stream` | CORS 配置；使用 `@microsoft/fetch-event-source` 库；支持断点重连+Last-Event-ID |
| **App（React Native）** | SSE | 同上 | 使用 `react-native-sse` 或 `EventSource` polyfill；后台保活需配合原生通知 |
| **微信小程序** | HTTPS 轮询或 SSE（基础库 2.22+） | 小程序基础库<2.22 用 `POST /chat` + `GET /sessions/{id}/messages` 轮询；新版本可用 SSE | 域名必须 ICP 备案；request 合法域名配置；不支持自定义 Header（通过 query 传 token） |
| **智能音箱** | HTTPS 非流式 | `POST /chat` + 轮询结果，或使用 Webhook 回调 | 响应时间敏感（<3s 首字）；建议走小模型快速路径；音频 TTS 在网关层转换 |

**统一接入网关职责**（APISIX/Kong）：
1. JWT 验签 → 转换为 `X-User-ID` 头
2. 终端类型识别（User-Agent / 自定义头）→ 路由到不同 Agent 模板
3. 限流：IP 层 60 次/分、用户层 30 次/小时
4. 协议转换：音箱 Webhook ↔ SSE；小程序轮询 ↔ SSE
5. 灰度：按 `tenant_id` 或 `user_id` 比例分流到不同版本

### 6.2 推荐配置：AgentService + 多租户

GerClaw 生产环境推荐最小配置：

```python
# gerclaw/deploy/app.py
from fastapi.middleware import Middleware
from fastapi.middleware.cors import CORSMiddleware
from agentscope.app import create_app
from agentscope.app.message_bus import RedisMessageBus
from agentscope.app.storage import RedisStorage
from agentscope.app.workspace_manager import LocalWorkspaceManager
from agentscope.app.rag.knowledge_base_manager import CollectionPerKbManager
from agentscope.rag import QdrantStore

storage = RedisStorage(host="redis.gerclaw.internal", port=6379, key_ttl=604800)
message_bus = RedisMessageBus(host="redis.gerclaw.internal", port=6379)
workspace_manager = LocalWorkspaceManager(basedir="/data/workspaces", ttl=3600)
kb_manager = CollectionPerKbManager(storage=storage, vector_store=QdrantStore(url="http://qdrant:6333"))

app = create_app(
    storage=storage,
    message_bus=message_bus,
    workspace_manager=workspace_manager,
    knowledge_base_manager=kb_manager,
    enable_index_worker=False,  # 分布式部署，独立 worker 进程
    extra_middlewares=[Middleware(CORSMiddleware, allow_origins=CORS_ORIGINS, allow_credentials=True, allow_methods=["*"], allow_headers=["*"])],
    extra_agent_middlewares=gerclaw_tenant_middleware_factory,  # 租户中间件工厂
    extra_agent_tools=gerclaw_medical_tools_factory,            # 医疗工具注入
)
```

**多租户 Agent 中间件工厂示例**：
```python
async def gerclaw_tenant_middleware_factory(user_id: str, agent_id: str, session_id: str):
    """解析 user_id 中的租户信息，注入对应 RAG 知识库"""
    parts = user_id.split(":")
    tenant_type, tenant_id = parts[0], parts[1]
    kb = await get_tenant_knowledge_base(tenant_type, tenant_id)
    return [RAGMiddleware(knowledge_bases=[kb], parameters=RAGMiddleware.Parameters(mode="static", top_k=3))]
```

### 6.3 风险：并发会话限制 / 流式响应中断处理

#### 6.3.1 并发会话限制

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 单 session 双重提交 | `ChatRunRegistry` 返回 409，但客户端可能重试导致消息重复 | 客户端做幂等（`Idempotency-Key` 头）；服务端 409 时不要自动重试 |
| 单实例 SSE 连接数过多 | uvicorn 异步连接数受文件描述符限制 | K8s HPA 按 `sse_active_connections` 指标扩容；单实例软限 200 SSE 连接 |
| LLM 并发打满 | 请求堆积，TTFT 飙升 | `asyncio.Semaphore(50)` 控制单实例 LLM 并发；LiteLLM 层 RPM/TPM 限流；队列溢出降级提示 |
| Redis 连接池耗尽 | 服务雪崩 | `max_connections=200`；PgBouncer 模式管理连接；监控连接数>80% 告警 |
| 多终端同时发消息 | 同一 session 并发触发 run | 服务端 409 保护；客户端串行化发送队列；设计上建议"一个终端一个 session" |

#### 6.3.2 流式响应中断处理

| 场景 | 现象 | 处理策略 |
|---|---|---|
| 客户端主动断开 | SSE 连接关闭，后台 run 继续执行浪费 Token | `asyncio.CancelledError` 传播 → 取消 LLM 请求；`task.cancel()` 终止 run |
| 网络瞬断 | SSE 中断，前端重连 | 前端使用 Last-Event-ID 续传；服务端缓冲 100 条事件，重连时自动回放 |
| LLM API 超时/错误 | 流中断，无 done 帧 | 发送 `event: error` 帧 + 自动重试（最多 1 次 Fallback 到备用模型） |
| Nginx/Ingress 超时 | 60s 无数据断开 | 每 30s 心跳注释帧 `:\n\n`；Ingress `proxy_read_timeout 300s` |
| Pod 滚动重启 | 连接断开 | K8s `preStop` hook 等待 SSE 排空（graceful shutdown 30s）；客户端自动重连到新 Pod |
| 浏览器后台 Tab 限流 | EventSource 被挂起 | 前端检测 `visibilitychange`，回前台时拉取 `/sessions/{id}/messages` 补全丢失消息 |

**服务端优雅关闭要点**：
```python
@app.on_event("shutdown")
async def shutdown():
    await message_bus.close()    # 停止接收新消息
    await storage.close()        # 等待 Redis 操作完成
    # 给 SSE 连接发送 shutdown 事件，让客户端重连
```

pip install agentscope fastapi uvicorn httpx dashscope
