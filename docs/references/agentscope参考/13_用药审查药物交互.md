## 1. 模块映射总览

GerClaw 用药审查模块需要在 Agent 运行时暴露一组「药物安全工具」供药师/医生 Agent 在处方审查场景下调用。核心映射关系如下：

| GerClaw 业务能力 | AgentScope 对应抽象 | 实现方式 |
|------------------|---------------------|----------|
| DDI 药物-药物交互检查 | 自定义 `ToolBase` 子类 `DDITool` | 继承 `ToolBase`，实现 `check_permissions` + `call`，内置规则表 |
| Beers Criteria 老年 PIM 筛查 | 自定义 `ToolBase` 子类 `BeersTool` | 同 DDITool，规则表替换为 Beers 2023 PIM 列表 |
| 药物信息查询（通用名/分类/剂量） | `FunctionTool` 装饰器包装普通函数 | 用 `FunctionTool(query_drug_info)` 快速包装无状态查询函数 |
| 审查工具集注册到药师 Agent | `Toolkit` 容器 | `Toolkit(tools=[DDITool(), BeersTool(), FunctionTool(...)])` |
| 高风险处方强制拦截/药师确认 | `check_permissions` + `PermissionContext` | 禁忌级交互返回 `PermissionBehavior.ASK` 或自定义拦截规则 |
| 调用外部药物数据库 CLI（如 RxNav/RxNorm 本地命令行） | 内置 `Bash` 工具 | `Bash().call(command="rxnav-cli findInteraction --drugs ...")` |
| 审查操作审计日志 | `ToolMiddlewareBase` 洋葱中间件 | 在 `on_tool_call` 前后记录调用参数、结果、时间戳到审计日志 |
| 审查结果写入患者档案 | `Toolkit` + Agent `reply_stream` 事件 | 工具返回结构化 `ToolChunk`，Agent 组装最终审查报告 |
| 医生确认/HITL 流程 | `is_external_tool=True` 外部工具 | 定义 `DoctorConfirmTool`，运行时 yield `RequireExternalExecutionEvent` |

**整体调用链**：

```
医生处方/患者用药清单
        │
        ▼
┌──────────────────────┐
│  PharmacistAgent     │  system_prompt 中声明审查职责
│  (Agent + Toolkit)   │
└──────────┬───────────┘
           │  ReAct 循环自主选择工具
           ▼
  ┌────────┴────────┬──────────────┬───────────────┐
  ▼                 ▼              ▼               ▼
DDITool         BeersTool    query_drug_info   Bash(外部DB)
(ToolBase)      (ToolBase)   (FunctionTool)    (内置Bash)
  │                 │              │               │
  └────────┬────────┴──────┬───────┘               │
           ▼               ▼                        │
    ToolMiddleware(AuditLog)                        │
           │                                       │
           ▼                                       │
    结构化审查结果 (ToolChunk) ◄────────────────────┘
           │
           ▼
    Agent 生成 Markdown 报告 → 🔴/🟡/🟢 分级 → 医生确认/药师审核
```

## 2. 核心 API 参考

### 2.1 自定义 Tool 继承 ToolBase + ParamsBase 实现 DDI 检查

**`ToolBase`** (`src/agentscope/tool/_base.py`) 是所有工具的抽象基类，必须定义 5 个类属性 + 2 个方法：

```python
from agentscope.tool import ToolBase, ToolChunk, ToolMiddlewareBase
from agentscope.message import TextBlock
from agentscope.permission import PermissionContext, PermissionDecision, PermissionBehavior

class DDITool(ToolBase):
    name: str = "check_drug_interactions"
    description: str = "检查药物-药物交互作用(DDI)，返回交互严重级别和临床建议"
    input_schema: dict = {
        "type": "object",
        "properties": {
            "drug_list": {
                "type": "array",
                "items": {"type": "string"},
                "description": "待审查的药品通用名列表，如 ['华法林', '阿司匹林']",
            },
        },
        "required": ["drug_list"],
    }
    is_concurrency_safe: bool = True
    is_read_only: bool = True  # DDI 检查是只读查询

    async def check_permissions(self, tool_input, context):
        # DDI 检查是只读医疗查询，默认放行
        return PermissionDecision(
            behavior=PermissionBehavior.ALLOW,
            message="DDI 药物交互检查为只读查询，允许执行",
        )

    async def call(self, drug_list: list[str]) -> ToolChunk:
        # 执行两两配对检查，返回结构化结果
        interactions = self._check_all_pairs(drug_list)
        report = self._format_report(interactions)
        return ToolChunk(content=[TextBlock(text=report)])
```

**`ParamsBase`** (`src/agentscope/tool/_base.py`) 继承自 `pydantic.BaseModel`，用于声明工具参数并自动生成 JSON Schema（自动去掉 title 字段，更适合给 LLM 看）：

```python
from agentscope.tool import ParamsBase
from typing import Literal

class DDICheckParams(ParamsBase):
    drug_list: list[str]
    patient_age: int | None = None
    severity_filter: Literal["contraindicated", "major", "moderate", "minor"] = "minor"

# input_schema 可直接从 Params 类生成
input_schema = DDICheckParams.model_json_schema()
```

**`ToolChunk`** (`src/agentscope/tool/_response.py`) 是工具单次返回的内容块，包含 `content: list[TextBlock | DataBlock]` 和可选的 `metadata`。

### 2.2 FunctionTool 装饰器包装药物查询函数

**`FunctionTool`** (`src/agentscope/tool/_adapters.py`) 是适配器，把普通 Python 函数自动包装成 `ToolBase`：从函数名取 `name`，从 docstring 取 `description`，从类型注解推导 `input_schema`。

```python
from agentscope.tool import FunctionTool, Toolkit

def query_drug_info(drug_name: str, field: str = "all") -> str:
    """查询药物基本信息（通用名/分类/常规剂量/禁忌症）。

    Args:
        drug_name: 药品通用名，如 '华法林'、'氨氯地平'
        field: 查询字段，可选 'classification'/'dose'/'contraindication'/'all'
    """
    info = DRUG_DB.get(drug_name, {})
    if field == "all":
        return str(info)
    return str(info.get(field, "暂无数据"))

# 包装为 Tool 并注册
toolkit = Toolkit(tools=[FunctionTool(query_drug_info, is_read_only=True)])
```

`FunctionTool` 默认 `check_permissions` 返回 `ASK`（需要用户确认）。医疗只读查询场景建议在子类或自定义 Tool 中覆写为 `ALLOW`。

### 2.3 Toolkit 注册审查工具

**`Toolkit`** (`src/agentscope/tool/_toolkit.py`) 是工具注册中心，支持 tools / mcps / skills / tool_groups 四类资源：

```python
from agentscope.tool import Toolkit, ToolGroup

# 基础审查工具集（始终激活的 "basic" 组）
toolkit = Toolkit(
    tools=[
        DDITool(),
        BeersTool(),
        FunctionTool(query_drug_info, is_read_only=True),
    ],
)

# 进阶工具组（Agent 可通过 reset_tools 元工具按需激活）
advanced_group = ToolGroup(
    name="advanced_review",
    description="高级审查：肝肾剂量调整、药物-基因交互",
    tools=[DoseAdjustTool(), PGxTool()],
)
toolkit = Toolkit(
    tools=[DDITool(), BeersTool()],
    tool_groups=[advanced_group],
)
```

关键方法：
- `await toolkit.get_tool_schemas()` → 返回当前激活工具的 JSON Schema 列表，传给 LLM
- `await toolkit.call_tool(tool_call_block, state)` → 执行工具调用，yield `ToolChunk` 流 + 最终 `ToolResponse`
- `await toolkit.get_tool(name)` → 按名称获取工具实例

### 2.4 check_permissions 用药审查特殊权限

用药审查场景的权限设计遵循三级风险分级（对应需求文档 5.1 节）：

| 场景 | PermissionBehavior | 说明 |
|------|--------------------|------|
| 🔴 禁忌级交互（X级/严重） | `ASK` + `bypass_immune=True` | 必须医生确认，allow 规则无法静默放行 |
| 🟡 中度交互警告 | `ASK` | 弹出对话框，医生确认即可 |
| 🟢 轻度/信息提示 | `ALLOW` | 直接返回结果 |
| 写入患者档案/提交处方 | `PASSTHROUGH` | 交给全局权限引擎按规则匹配 |
| 调用 Bash 查询外部 DB | `PASSTHROUGH`（只读命令 Bash 自动识别为 ALLOW） | `ls/cat/grep/git status` 等自动放行 |

```python
async def check_permissions(self, tool_input, context):
    severity = self._assess_severity(tool_input.get("drug_list", []))
    if severity == "contraindicated":
        return PermissionDecision(
            behavior=PermissionBehavior.ASK,
            message="检测到禁忌级药物组合，必须医生确认后才能继续",
            bypass_immune=True,  # allow 规则无法绕过
        )
    elif severity == "major":
        return PermissionDecision(
            behavior=PermissionBehavior.ASK,
            message="检测到严重药物交互，建议医生审核",
        )
    return PermissionDecision(behavior=PermissionBehavior.ALLOW)
```

`PermissionContext` 支持 5 种模式：
- `DEFAULT` — 默认模式，按规则匹配
- `EXPLORE` — 只读探索，只读工具自动放行
- `ACCEPT_EDITS` — 接受编辑，工作目录内文件操作自动放行
- `BYPASS` — 绕过所有检查（除 `bypass_immune=True` 的 ASK）
- `DONT_ASK` — 禁止任何 ASK，转为 DENY

### 2.5 Bash 工具调用药物数据库命令行工具

**`Bash`** (`src/agentscope/tool/_builtin/_bash.py`) 可调用外部命令行工具（如本地部署的 RxNav-in-a-Box、DrugBank CLI、药智数据命令行查询工具）：

```python
from agentscope.tool import Bash

bash = Bash(cwd="/opt/drugdb")  # 指定工作目录

# 调用 RxNorm 命令行查询药物 RxCUI
result = await bash.call(command='rxnorm-cli getNameInfo "warfarin"')

# 调用本地 DDI 查询脚本
result = await bash.call(command='python ddi_query.py --drugs "warfarin,aspirin" --format json')
```

Bash 的安全机制：
- 自动识别只读命令（`ls/cat/grep/git status/...`）→ ALLOW
- 检测命令注入风险（`$(...)`、反引号）→ ASK
- 危险路径保护（`~/.ssh/`、`.env`、`.bashrc`）→ ASK
- 危险删除（`rm -rf /`、`rm ~`）→ ASK

**GerClaw 建议**：将外部药物数据库封装为独立 Python 脚本（输入 JSON、输出 JSON），通过 Bash 调用，参数使用 `--drugs "药1,药2"` 方式避免 shell 注入。

### 2.6 ToolMiddlewareBase 实现审查日志

**`ToolMiddlewareBase`** (`src/agentscope/tool/_base.py`) 采用洋葱模型，可以在工具调用前后插入横切逻辑。用药审查场景用于审计日志：

```python
from agentscope.tool import ToolMiddlewareBase, ToolBase, ToolChunk
from typing import AsyncGenerator, Any, Callable
from datetime import datetime
import json

class AuditLogMiddleware(ToolMiddlewareBase):
    """用药审查审计日志中间件 — 记录每次审查的药物清单、结果、时间戳"""

    async def on_tool_call(
        self,
        tool: ToolBase,
        input_kwargs: dict[str, Any],
        next_handler: Callable[..., AsyncGenerator[ToolChunk, None]],
    ) -> AsyncGenerator[ToolChunk, None]:
        start = datetime.now()
        print(f"[AUDIT] {start.isoformat()} 调用工具 {tool.name}, 参数={json.dumps(input_kwargs, ensure_ascii=False)}")
        try:
            async for chunk in next_handler(**input_kwargs):
                yield chunk
            elapsed = (datetime.now() - start).total_seconds()
            print(f"[AUDIT] {tool.name} 执行完成, 耗时{elapsed:.2f}s")
        except Exception as e:
            print(f"[AUDIT] {tool.name} 执行失败: {e}")
            raise

# 使用方式
ddi_tool = DDITool(middlewares=[AuditLogMiddleware()])
```

中间件链按注册顺序执行：`[AuditLog, RetryMiddleware, ...]` → 先注册的是最外层。

## 6. GerClaw 适配要点

### 6.1 DDI 规则库设计：严重级别分级 / 老年特殊考虑

**严重级别分级**（与 DrugBank/Micromedex 对齐）：

| 级别 | 风险评分 | 临床行为 | AgentScope 权限映射 |
|------|---------|----------|---------------------|
| Contraindicated (X级/禁忌) | ≥90 | 🔴 强制拦截，禁止联用 | `PermissionBehavior.ASK` + `bypass_immune=True` |
| Major/Severe (严重) | 70-89 | 🔴 避免联用/严密监测/调量 | `PermissionBehavior.ASK` |
| Moderate (中度) | 40-69 | 🟡 谨慎联用/加强监测 | `PermissionBehavior.ASK`（可配 allow 规则放行） |
| Minor/Mild (轻度) | 1-39 | 🟢 可联用/告知观察 | `PermissionBehavior.ALLOW` |
| Unknown | 0 | 🟢 密切观察 | `PermissionBehavior.ALLOW` |

**老年人特殊考虑**（规则库需额外标注）：

1. **年龄相关药效/药代改变**：老年人肝肾功能下降（eGFR 每年降 ~1mL/min），DDI 风险在老年人中被放大；规则库应包含 `age >= 65` 时的风险升级逻辑（如原本 Moderate 的交互在 ≥75 岁 + eGFR<60 时升级为 Major）。
2. **Beers PIM 交叉引用**：DDI 规则命中后应同时检查是否触发 Beers Criteria（如苯二氮卓类 + 阿片类 → 呼吸抑制 + Beers 双重 PIM）。
3. **抗胆碱能负荷叠加**：多种抗胆碱能药物（抗抑郁+抗组胺+抗帕金森）叠加使用需计算 ARS/DES 评分，≥3 分提示高风险。
4. **跌倒风险叠加**：降压药+利尿剂+镇静催眠药+阿片类 → 体位性低血压+跌倒风险升级。
5. **QT 间期延长叠加**：多种 QT 延长药物联用 → 尖端扭转型室速风险，老年人尤甚。
6. **中成药/保健品交互**：国内老年患者常自行服用中成药（如银杏、丹参、人参），需纳入 DDI 规则（如银杏+华法林→出血风险）。

### 6.2 推荐配置：DDITool + BeersTool + PermissionEngine

GerClaw 用药审查 Agent 的推荐工具配置：

```python
from agentscope.tool import Toolkit, FunctionTool, ToolGroup

# === 核心审查工具（basic 组，始终激活）===
review_toolkit = Toolkit(
    tools=[
        DDITool(middlewares=[AuditLogMiddleware()]),       # DDI 交互检查
        BeersTool(middlewares=[AuditLogMiddleware()]),     # Beers 2023 PIM 筛查
        FunctionTool(query_drug_info, is_read_only=True),  # 药物信息查询
        FunctionTool(check_dose, is_read_only=True),       # 剂量审查
    ],
    tool_groups=[
        ToolGroup(
            name="external_db",
            description="外部药物数据库命令行查询（RxNorm/DrugBank本地）",
            tools=[Bash(cwd="/opt/drugdb", middlewares=[AuditLogMiddleware()])],
        ),
        ToolGroup(
            name="advanced",
            description="高级审查：肝肾剂量调整、药物-基因交互、STOPP/START",
            tools=[
                DoseAdjustTool(),
                PGxTool(),
                STOPPSTARTTool(),
            ],
        ),
    ],
)
```

**权限配置建议**：
- 药师 Agent 运行在 `PermissionMode.EXPLORE` 模式（只读工具自动放行），但禁忌级交互通过 `bypass_immune=True` 强制拦截。
- 医生 Agent 运行在 `PermissionMode.ACCEPT_EDITS` 模式（可提交处方修改）。
- 患者/家属 Agent 运行在 `PermissionMode.EXPLORE` 模式（只能查询，不能开方）。

### 6.3 风险：规则库过期 / 假阳性需医生确认

1. **规则库过期风险**：
   - 药物交互知识持续更新（FDA 警告、新文献、新上市药物），DDI 规则库必须版本化管理，建议按季度更新。
   - 方案：规则数据存储在外部 JSON/SQLite 文件（而非硬编码在 Python 代码中），`DDITool` 在 `__init__` 时加载最新版本，版本号写入 `ToolChunk.metadata` 便于审计。
   - 结合 RAG 检索最新 FDA 警告/文献补充规则库不足。

2. **假阳性/假阴性风险**：
   - 规则引擎不可避免存在假阳性（理论交互但临床实际安全）和假阴性（未收录的新交互）。
   - **所有 AI 审查结果必须经医生/药师确认**，不能替代专业判断。系统定位是「辅助决策」而非「自动处方」。
   - 🔴 禁忌级 → 强制拦截 + 医生双签确认（输入理由密码）
   - 🟡 警告级 → 弹窗提示，医生确认后继续
   - 🟢 提示级 → 信息栏显示，不阻断

3. **中文药名映射风险**：
   - 通用名/商品名/别名/俗名众多（如"倍他乐克"=美托洛尔、"拜唐苹"=阿卡波糖），需维护中文别名映射表。
   - 建议：在 DDITool 前增加 `DrugNormalizer` 预处理步骤，将所有名称归一化为通用名后再做交互检查。

4. **数据合规与隐私**：
   - 患者处方数据属于敏感个人健康信息，工具调用日志需脱敏（不记录患者姓名，只记录患者 ID）。
   - AuditLogMiddleware 中注意日志脱敏。

5. **模型幻觉风险**：
   - LLM 可能生成不存在的药物交互或错误建议。关键交互信息必须由规则引擎（DDITool/BeersTool）给出，LLM 只负责自然语言解释和报告组装，不做事实判断。

### 7.1 ddi_custom_tool.py — 自定义 DDITool（纯工具演示，不需要 LLM）
