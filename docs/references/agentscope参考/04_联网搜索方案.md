## 1. 模块映射总览

GerClaw 的「联网搜索」能力在 AgentScope 中通过 **自定义 Tool（工具）** 机制实现。Agent 将搜索能力封装为 ToolBase 子类或 FunctionTool 包装的 Python 函数，注册到 Toolkit 后由 ReAct 循环在推理过程中自动调用。

### 1.1 核心概念关系

```
GerClaw 医生Agent (Agent)
  ├── model: DashScopeChatModel           # 通义千问等大模型
  ├── system_prompt: 医疗助手系统提示       # 包含搜索规则、免责声明
  ├── toolkit: Toolkit                    # 工具注册中心
  │   ├── [basic] 工具组
  │   │   ├── MedicalLiteratureSearchTool # 继承ToolBase — 医学文献检索(PubMed/指南)
  │   │   ├── FunctionTool(web_search)    # 装饰器/适配器 — 通用网络搜索(Tavily/Serper)
  │   │   ├── FunctionTool(drug_lookup)   # 装饰器/适配器 — 药品信息查询(OpenFDA/NMPA)
  │   │   └── FunctionTool(guideline_search) # 装饰器/适配器 — 临床指南检索
  │   └── 可选: ToolGroup("deep_research")  # 深度研究工具组（按需激活）
  ├── state: AgentState                   # 会话状态、权限上下文
  └── react_config: ReActConfig(max_iters=10)  # 允许多轮搜索-推理循环
```

### 1.2 两种工具创建方式对比

| 维度 | 继承 ToolBase | 使用 FunctionTool 包装函数 |
|------|--------------|--------------------------|
| 适用场景 | 复杂工具，需要精细权限控制、流式输出、状态注入 | 快速封装简单函数，原型开发 |
| 参数定义 | 内部定义 `Params(ParamsBase)` 类，手动设置 `input_schema` | 自动从函数签名+Pydantic类型注解提取 schema |
| 权限控制 | 必须实现 `check_permissions()`，可返回 ALLOW/ASK/DENY | 默认返回 ASK（需用户确认），只读可设 `is_read_only=True` |
| 流式输出 | `call()` 可 yield 多个 ToolChunk | 支持 async generator 自动转换 |
| 医疗场景推荐 | **推荐用于核心搜索工具**（需权威源分级、结果标注） | 适合快速原型、简单查询工具 |

### 1.3 搜索工具调用流程

```
用户: "老年高血压最新指南是什么？"
  ↓
Agent.reply() 启动 ReAct 循环
  ↓ Round 1: 模型推理 → 决定调用 medical_literature_search
  ↓
Toolkit.call_tool() → 权限检查(check_permissions) → 执行call()
  ↓
搜索工具内部: 构造查询 → (mock/真实API) → 权威分级 → 返回ToolChunk
  ↓
Agent接收ToolResult → Round 2: 基于搜索结果整合回答
  ↓
输出带来源标注的回答 + 免责声明
```

## 2. 核心API参考

### 2.1 ParamsBase — 工具参数基类

所有自定义 Tool 的参数类应继承 `ParamsBase`（而非直接继承 `pydantic.BaseModel`），它会自动移除 JSON Schema 中的 `title` 字段，使 schema 更适合 LLM 理解。

```python
from agentscope.tool import ParamsBase

class MedicalSearchParams(ParamsBase):
    """医学文献搜索参数"""
    query: str = Field(..., description="搜索关键词，如'老年高血压指南'")
    source: str = Field("all", description="数据源: pubmed/guideline/drug/all")
    max_results: int = Field(5, ge=1, le=20, description="返回结果数量，默认5")
    year_from: int | None = Field(None, description="起始年份，如2021")
```

**关键要点**：
- 使用 `pydantic.Field` 添加 `description`，这是 LLM 理解参数含义的唯一途径
- 参数类不需要手动实例化，AgentScope 会根据 LLM 返回的 JSON 自动构造
- `model_json_schema()` 已被重写，输出干净的 schema

### 2.2 ToolBase 继承方式

```python
from agentscope.tool import ToolBase, ToolChunk
from agentscope.message import TextBlock
from agentscope.permission import PermissionDecision, PermissionBehavior
from typing import Any

class MedicalLiteratureSearchTool(ToolBase):
    """医学文献检索工具 — 继承ToolBase的完整示例"""

    # ---- 必须定义的类属性 ----
    name: str = "medical_literature_search"
    description: str = (
        "搜索权威医学文献和临床指南数据库，包括PubMed、NICE、AHA、NCCN、"
        "中华医学会等来源。用于获取最新诊疗指南、循证医学证据、药品说明书。"
        "结果按来源权威性(S/A/B/C级)排序标注。"
    )

    class Params(ParamsBase):
        query: str = Field(..., description="搜索关键词")
        max_results: int = Field(5, ge=1, le=20, description="返回结果数")
        year_from: int | None = Field(None, description="起始年份")

    input_schema: dict = Params.model_json_schema()
    is_concurrency_safe: bool = True
    is_read_only: bool = True  # 搜索是只读操作

    # ---- 必须实现的方法 ----
    async def check_permissions(
        self, tool_input: dict[str, Any], context: PermissionContext
    ) -> PermissionDecision:
        """权限检查：搜索工具为只读，默认允许"""
        return PermissionDecision(
            behavior=PermissionBehavior.ALLOW,
            message="医学文献搜索为只读操作，允许执行。",
        )

    async def call(self, **kwargs: Any) -> ToolChunk:
        """执行搜索逻辑"""
        params = self.Params(**kwargs)
        # ... 执行搜索，返回结果
        results = await self._do_search(params)
        return ToolChunk(
            content=[TextBlock(text=results)],
        )

    async def _do_search(self, params: Params) -> str:
        """内部搜索实现（mock或真实API调用）"""
        # 实现细节...
        return "搜索结果..."
```

**必须实现的5个类属性**：
| 属性 | 类型 | 说明 |
|------|------|------|
| `name` | `str` | 工具名称，LLM通过此名称调用，建议使用snake_case |
| `description` | `str` | 工具描述，直接影响LLM是否选择调用此工具，务必清晰说明使用场景 |
| `input_schema` | `dict` | JSON Schema，通常通过 `Params.model_json_schema()` 生成 |
| `is_concurrency_safe` | `bool` | 是否并发安全，搜索工具一般为True |
| `is_read_only` | `bool` | 是否只读，搜索工具设为True可自动通过权限检查 |

**必须实现的2个方法**：
| 方法 | 说明 |
|------|------|
| `check_permissions(tool_input, context)` | 返回 `PermissionDecision`，控制ALLOW/ASK/DENY |
| `call(**kwargs)` | 工具执行逻辑，返回 `ToolChunk` 或 `AsyncGenerator[ToolChunk]` |

### 2.3 FunctionTool 装饰器/适配器用法

`FunctionTool` 是一个适配器类（非真正的装饰器，但可当作装饰器工厂使用），将普通 Python 函数包装为 ToolBase 实例。

```python
from agentscope.tool import FunctionTool

# 方式1：直接包装函数
async def web_search(query: str, search_depth: str = "basic") -> str:
    """搜索互联网获取医疗健康信息。

    Args:
        query: 搜索关键词
        search_depth: 搜索深度，basic快速，advanced全面
    """
    # 函数docstring的第一行作为工具描述
    # Args部分的描述作为参数描述
    return f"搜索结果: {query}"

search_tool = FunctionTool(
    func=web_search,
    name="web_search",  # 可选，默认用函数名
    description=None,   # 可选，默认从docstring提取
    is_concurrency_safe=True,
    is_read_only=True,  # 搜索是只读
)

# 方式2：作为装饰器使用（Python函数装饰器模式）
def medical_search_tool(func):
    """自定义装饰器，将函数包装为FunctionTool"""
    return FunctionTool(func, is_read_only=True, is_concurrency_safe=True)

@medical_search_tool  # 注意：这样返回的是FunctionTool实例，不是原始函数
async def guideline_search(condition: str, society: str = "general") -> str:
    """检索临床诊疗指南，支持NICE/WHO/AHA/NCCN/中华医学会等来源。

    Args:
        condition: 疾病名称
        society: 专科学会: general/cardiology/oncology/diabetes
    """
    return f"指南搜索结果: {condition}"
```

**FunctionTool自动提取规则**：
- `name`：未指定时使用函数的 `__name__`
- `description`：未指定时从docstring第一行提取（`_extract_func_description`）
- `input_schema`：从函数签名的类型注解自动生成（`_extract_input_schema`），支持Pydantic类型
- 支持同步函数和async函数
- 返回值自动转换：str→TextBlock，dict/list→JSON→TextBlock，ToolChunk直接返回，async generator流式返回

**注意事项**：
- FunctionTool的 `check_permissions()` 默认返回 `ASK`（需要用户确认）。对于只读搜索工具，创建时设置 `is_read_only=True` 可让权限引擎在EXPLORE模式下自动允许。
- 如果需要更精细的权限控制（如限制搜索域名），请继承ToolBase实现。

### 2.5 自定义Tool的ParamsBase定义规范

```python
from agentscope.tool import ParamsBase
from pydantic import Field
from typing import Literal

class GuidelineSearchParams(ParamsBase):
    """临床指南检索参数 — 完整示例"""
    condition: str = Field(
        ...,
        description="疾病名称，如'2型糖尿病'、'高血压'、'心力衰竭'",
    )
    society: Literal["general", "cardiology", "oncology", "diabetes", "all"] = Field(
        "general",
        description="专科学会过滤: general=通用, cardiology=心血管, oncology=肿瘤, diabetes=糖尿病, all=全部",
    )
    year_from: int | None = Field(
        None,
        ge=2015,
        description="指南发布起始年份，建议不早于2015年",
    )
    region: Literal["cn", "us", "uk", "who", "all"] = Field(
        "all",
        description="地区偏好: cn=中国指南, us=美国, uk=英国/NICE, who=WHO, all=全部",
    )
```

**参数设计最佳实践**：
- 每个Field必须有清晰的 `description`，这是LLM正确填写参数的关键
- 使用 `Literal` 类型限制枚举值，减少LLM幻觉
- 使用 `ge`/`le` 约束数值范围
- 搜索类参数提供合理默认值
- 医疗搜索参数应包含 `source`/`region` 等字段支持权威源筛选

### 2.6 check_permissions 实现

```python
from agentscope.permission import PermissionDecision, PermissionBehavior
from agentscope.permission import PermissionContext

async def check_permissions(
    self,
    tool_input: dict[str, Any],
    context: PermissionContext,
) -> PermissionDecision:
    """搜索工具权限检查实现。

    医疗搜索场景的权限策略：
    1. 纯搜索（只读）→ ALLOW
    2. 搜索+自动写入文件 → ASK
    3. 搜索含敏感关键词（如管制药品）→ ASK 并附加警告
    """
    query = tool_input.get("query", "")

    # 只读搜索 → 直接允许
    if self.is_read_only:
        return PermissionDecision(
            behavior=PermissionBehavior.ALLOW,
            message="医学信息检索为只读操作，已允许。",
        )

    # 检测敏感查询
    sensitive_keywords = ["麻醉药品", "精神药品", "毒药"]
    if any(kw in query for kw in sensitive_keywords):
        return PermissionDecision(
            behavior=PermissionBehavior.ASK,
            message=f"搜索关键词'{query}'涉及特殊管理药品，是否继续？",
            bypass_immune=True,  # 此类ASK不能被allow规则覆盖
        )

    # 默认：需要用户确认
    return PermissionDecision(
        behavior=PermissionBehavior.ASK,
        message=f"即将执行医学搜索: {query}，是否允许？",
    )
```

**PermissionBehavior 枚举值**：
| 值 | 含义 |
|----|------|
| `ALLOW` | 允许执行，不提示用户 |
| `ASK` | 需要用户确认后执行 |
| `DENY` | 拒绝执行 |

**医疗场景建议**：
- 纯搜索/查询操作设为 `is_read_only=True` + ALLOW，减少用户打扰
- 涉及用户数据修改、处方推荐等操作设为 ASK
- 高风险操作（如管制药品查询）设为 `bypass_immune=True` 的ASK

## 6. GerClaw适配要点

### 6.1 医学搜索工具特殊考虑

#### 6.1.1 权威来源优先

搜索工具返回结果时，必须按照GerClaw权威分级体系标注来源等级：

- **S级（权重1.0）**：NMPA、卫健委、WHO、FDA、NIH、PubMed等政府/国际组织
- **A级（权重0.9）**：NICE、AHA、NCCN、中华医学会、Lancet/NEJM/JAMA等顶级期刊
- **B级（权重0.6）**：丁香园、医脉通、药智网等专业平台
- **C级（权重0.3）**：百度健康、知乎等通用健康资讯
- **D级（权重0.0）**：商业推广、自媒体、论坛 → **直接过滤**

**实现建议**：在ToolBase的`call()`方法内部，搜索完成后执行权威分级逻辑，将等级信息包含在返回的文本中（如`[S级-国家卫健委]`前缀），便于LLM在整合答案时识别来源可靠性。

#### 6.1.2 结果必须标注来源

所有搜索结果必须包含：
1. **来源机构名称**（如"中华医学会心血管病学分会"）
2. **来源URL**（如可获取）
3. **发布日期/年份**（医疗指南时效性至关重要）
4. **来源等级标注**（S/A/B/C级）

在Tool返回的ToolChunk文本中，使用结构化格式如：
```
[S级-国家卫健委, 2024] 老年高血压诊疗规范要点...
URL: https://www.nhc.gov.cn/...
```

### 6.2 推荐配置

```python

from agentscope.agent import Agent, ReActConfig

doctor_agent = Agent(
    name="GerClaw医生助手",
    system_prompt=SYSTEM_PROMPT_MEDICAL,  # 包含搜索规则、免责声明
    model=model,
    toolkit=Toolkit(tools=[
        MedicalLiteratureSearchTool(),           # 核心：继承ToolBase，含权威分级
        FunctionTool(web_search, is_read_only=True),     # 通用搜索
        FunctionTool(drug_info_lookup, is_read_only=True), # 药品查询
        FunctionTool(guideline_search, is_read_only=True), # 指南检索
    ]),
    react_config=ReActConfig(
        max_iters=10,        # 允许多轮搜索-验证循环（医疗场景需交叉验证）
        stop_on_reject=True, # 用户拒绝工具调用时停止
    ),
)
```

**系统提示词必须包含**：
1. 涉及最新药品信息/临床指南/新闻时，必须调用搜索工具
2. S/A级来源优先引用，B/C级来源需提示"信息仅供参考"
3. 不同来源有矛盾时需指出差异
4. 回答末尾必须附免责声明："以上信息仅供参考，具体诊疗请遵医嘱"
5. 识别紧急症状（胸痛/呼吸困难/卒中）→ 直接建议拨打120
6. 不宣称"治愈"老年慢性病

### 6.3 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| **网络不可用** | 搜索工具调用失败，Agent无法获取最新信息 | 1. Tool内捕获网络异常，返回友好错误提示而非抛异常；2. 提示"网络连接异常，无法获取最新医学信息，建议咨询医生"；3. 可实现本地缓存降级（常见疾病知识库） |
| **搜索结果不可靠** | 返回非权威/错误/过时医疗信息 | 1. 权威分级过滤D级来源；2. S/A级来源交叉验证（至少2个一致）；3. 时效性检查，指南优先近5年；4. 医疗安全审核层检测危险建议 |
| **API配额耗尽** | Tavily/Serper等付费API额度用完 | 1. 多API降级策略（Tavily→Serper→免费Jina Reader）；2. 预算中间件监控调用次数；3. 缓存常见查询结果 |
| **LLM幻觉引用** | 模型编造不存在的文献/指南 | 1. 强制工具调用获取真实结果，禁止模型凭记忆回答时效性问题；2. 所有引用必须来自tool result；3. 结构化返回格式，包含可验证的URL/PMID |
| **隐私泄露** | 搜索关键词包含患者隐私 | 1. 搜索前脱敏处理（去除姓名/身份证号等PHI）；2. 系统提示词明确禁止发送可识别个人信息 |
| **延迟过高** | 多源并行搜索导致响应慢 | 1. 使用asyncio.gather并行调用多个API；2. 设置超时（如10秒）；3. 先返回快速结果，深度结果异步补充 |
