## 1. 模块映射总览

"问卷量表对话化"是 GerClaw CGA（老年综合评估）的核心交互形态，需要将 PHQ-9/GDS 抑郁、ADL/IADL 日常能力、MMSE 认知、MNA 营养、Morse 跌倒等 20+ 医疗量表转化为多轮自然对话。AgentScope 提供的以下能力可直接支撑该模块：

| GerClaw 需求 | AgentScope 对应能力 | 关键类/接口 |
|---|---|---|
| 分维度、分步骤提问（单轮单信息） | `system_prompt` 引导 + ReAct 循环 + `ReActConfig.max_iters` 控制 | `Agent`, `ReActConfig` |
| 问卷进度/已答条目持久化 | `AgentState.context` 消息列表 + `AgentState.middle_context` 字典 + `summary` 压缩 | `AgentState` |
| 下一题/回退/跳转/查看进度 | 自定义 `ToolBase`（或 `FunctionTool` 适配器）注入状态 | `ToolBase`, `FunctionTool`, `Toolkit` |
| 在对话中注入进度提示、检查点信息 | `HintBlock` 注入系统侧提示（不占 user 角色） | `HintBlock`, `HintBlockEvent` |
| 多步骤评估计划（ADL→IADL→MMSE→GDS→MNA） | 内置 Plan 工具（TaskCreate/TaskList/TaskUpdate） | `TaskCreate`, `TaskList`, `TaskUpdate`, `TaskGet` |
| 断点续答/中断恢复 | `AgentState` 可序列化 + 预置 Task + `observe()` 恢复上下文 | `Agent.state_dict`, `Agent.observe` |
| 老年友好：慢语速、大字体、允许重复 | TTS 参数控制 + 前端适老化 UI（非 AgentScope 范畴，见 6.1） | `DashScopeTTSModel` |
| 答非所问/模糊回答澄清 | LLM 自由对话 + system_prompt 约束澄清策略 | `Agent.reply` |
| 高危条目拦截（自杀意念等） | Tool 权限引擎 `PermissionBehavior.DENY/ASK` + 规则校验中间件 | `PermissionDecision`, `MiddlewareBase` |

**推荐架构**：混合式方案（状态机骨架 + LLM 弹性层），对应到 AgentScope 即「自定义 Tool 维护问卷状态机（骨架层）+ Agent 自然语言交互（弹性层）+ Plan 工具管理评估步骤（编排层）」。

## 2. 核心 API 参考

### 2.1 Agent ReAct 循环控制

Agent 内部自动执行 Reasoning-Acting 循环：模型推理 → 判断是否调工具 → 执行工具 → 把结果喂回模型 → 直到模型输出纯文本或达到 `max_iters`。

```python
from agentscope.agent import Agent, ReActConfig, ModelConfig

react_config = ReActConfig(
    max_iters=30,          # 问卷场景建议放大（每题可能触发1-2次工具调用）
    stop_on_reject=False,  # 问卷工具无需用户拒绝确认，设为 False
)

agent = Agent(
    name="cga_survey_agent",
    system_prompt=SURVEY_SYSTEM_PROMPT,
    model=model,
    toolkit=toolkit,
    react_config=react_config,
)
```

**关键行为**：
- `agent.reply(UserMsg(name="user", content="..."))` 为一次用户输入的完整 ReAct 闭环。
- `agent.reply_stream()` 返回事件流，可监听 `ToolCallStartEvent/EndEvent` 做前端进度条。
- 循环终止条件：模型不再输出 tool_call、达到 `max_iters`（触发 `ExceedMaxItersEvent`）、`stop_on_reject=True` 时被用户拒绝。

### 2.2 state.context 管理问卷状态

`AgentState` 是 Agent 的持久化状态容器，核心字段：

| 字段 | 用途（问卷场景） |
|---|---|
| `context: list[Msg]` | 完整对话历史，自动维护 user/assistant 消息（含 tool_call/tool_result） |
| `summary: str` | 上下文压缩后的摘要，自动注入 system message 之后，适合存"已完成维度摘要" |
| `tasks_context: TaskContext` | Plan 工具的任务清单（评估步骤列表） |
| `tool_context: ToolContext` | 工具上下文（文件缓存等，问卷一般不用） |
| `middle_context: dict` | **推荐用于存放问卷业务状态**：当前量表、当前题号、已答答案 dict、checkpoint 等 |

```python
# 在自定义 Tool 中（is_state_injected=True）通过 _agent_state 参数读写
async def __call__(self, _agent_state=None, **kwargs):
    survey = _agent_state.middle_context.setdefault("survey", {})
    survey["current_section"] = "ADL"
    survey["answers"]["ADL_Q1"] = kwargs.get("answer")
```

问卷状态建议结构：

```python
survey_state = {
    "user_name": "李奶奶",
    "current_scale": "CGA",
    "current_section": "ADL",       # ADL/IADL/MMSE/GDS/MNA
    "current_question_index": 3,
    "answers": {                    # 已答题目
        "ADL_Q1": {"score": 10, "raw": "自己能吃，不用喂"},
        "ADL_Q2": {"score": 5,  "raw": "洗脸需要帮忙递毛巾"},
    },
    "section_scores": {"ADL": None, "IADL": None, ...},
    "checkpoint": {"section": "ADL", "q_index": 3, "progress": 0.15},
    "fatigue_score": 0.2,           # 对话疲劳检测
    "interrupted": False,
}
```

### 2.3 自定义 Tool 实现问卷流转

两种方式定义工具：

**方式一：继承 `ToolBase`（适合有状态注入的工具）**

```python
from agentscope.tool import ToolBase, ToolChunk
from agentscope.message import TextBlock
from agentscope.permission import PermissionDecision, PermissionBehavior, PermissionContext
from typing import Any

class NextQuestionTool(ToolBase):
    name: str = "next_question"
    description: str = "记录当前题答案并前进到下一题。每题回答后必须调用。"
    input_schema: dict = {
        "type": "object",
        "properties": {
            "answer": {"type": "string", "description": "老人的回答原文"},
            "score": {"type": "integer", "description": "对应量表评分"},
        },
        "required": ["answer", "score"],
    }
    is_concurrency_safe: bool = False   # 顺序敏感
    is_read_only: bool = False
    is_state_injected: bool = True      # 注入 _agent_state

    async def check_permissions(self, tool_input, context):
        return PermissionDecision(behavior=PermissionBehavior.ALLOW,
                                  decision_reason="问卷工具默认放行",
                                  message="")

    async def __call__(self, answer: str, score: int, _agent_state=None, **kw):
        survey = _agent_state.middle_context["survey"]
        # 记录答案 → 推进指针 → 返回下一题
        ...
        return ToolChunk(content=[TextBlock(text=next_question_text)])
```

**方式二：`FunctionTool` 适配器（适合简单无状态工具）**

```python
from agentscope.tool import FunctionTool

def get_progress() -> str:
    """获取当前问卷完成进度，返回百分比和已完成维度。"""
    ...
    return "已完成 ADL(10/10)、IADL(5/8)，总体进度 45%"

tool = FunctionTool(get_progress, is_read_only=True)
```

权限注意：`FunctionTool.check_permissions()` 默认返回 `ASK`（需用户确认），问卷场景应子类化或改用继承 `ToolBase` 的方式返回 `ALLOW`，避免老人被权限弹窗打断。

### 2.4 system_prompt 引导分步骤提问

system_prompt 是控制 LLM 对话策略的核心手段，问卷场景应在 prompt 中明确：

1. **角色设定**：老年科护士/健康助手"小葛"
2. **提问规则**：一次只问一个问题；用简单口语；允许老人答非所问时温和澄清
3. **评估流程**：按 ADL → IADL → MMSE → GDS → MNA 顺序
4. **工具使用规则**：每题回答后必须调用 `record_answer`/`next_question`；想回退调用 `previous_question`；查看进度调用 `get_progress`
5. **敏感问题处理**：自杀意念等条目需加铺垫语
6. **疲劳检测**：老人说"累了"时调用 `save_checkpoint` 并礼貌道别

### 2.5 HintBlock 用于注入问卷进度提示

`HintBlock` 是 AgentScope 提供的一种特殊 ContentBlock，在 ReAct 循环中作为"系统侧提示"注入给 LLM，但对用户不可见（传给 API 时转为 user message）。适合在每轮模型调用前注入动态的问卷进度、当前题目上下文，避免这些信息占用 system_prompt 或被当作用户消息。

```python
from agentscope.message import HintBlock, TextBlock

# 在中间件或应用层，每次 reply 前注入进度 hint
hint = HintBlock(hint=[
    TextBlock(text=(
        f"[问卷进度] 当前维度：{section}，第 {idx+1}/{total} 题。"
        f"已答 {len(answers)} 题，总体进度 {pct}%。"
        f"请只问当前题目，不要提前透露后面的题。"
    ))
])
# 通过 observe() 或中间件注入
await agent.observe(Msg(name="system", role="user", content=[hint]))
```

HintBlock 事件 `HintBlockEvent` 在 reply_stream 中一次性发出（非流式），前端可监听但不展示给用户。

### 2.6 Plan 模式使用

对于 CGA 这种明确的多步骤流程，使用 AgentScope 内置 Plan 工具管理评估维度非常合适：

```python
from agentscope.tool import Toolkit, TaskCreate, TaskList, TaskUpdate, TaskGet

toolkit = Toolkit(tools=[
    TaskCreate(), TaskList(), TaskUpdate(), TaskGet(),
    NextQuestionTool(), PreviousQuestionTool(),
    JumpToSectionTool(), GetProgressTool(), SaveCheckpointTool(),
])
```

**预置评估计划**（在 agent 首次 reply 前 seed 任务）：

```python
from agentscope.state import Task

agent.state.tasks_context.tasks.extend([
    Task(id="1", subject="ADL 日常活动能力评估",
         description="10个条目，评估吃饭、穿衣、洗漱、如厕、转移、行走、上下楼梯、二便控制",
         metadata={"scale": "ADL", "total_questions": 10}),
    Task(id="2", subject="IADL 工具性日常活动评估",
         description="8个条目，评估打电话、购物、做饭、家务、洗衣、服药、理财、交通出行",
         metadata={"scale": "IADL", "total_questions": 8}, blocked_by=["1"]),
    Task(id="3", subject="MMSE 认知功能筛查",
         description="30分，含时间/地点定向、记忆、计算、语言、视空间",
         metadata={"scale": "MMSE", "total_questions": 30}, blocked_by=["2"]),
    Task(id="4", subject="GDS 老年抑郁筛查",
         description="15个条目 yes/no 抑郁筛查",
         metadata={"scale": "GDS", "total_questions": 15}, blocked_by=["3"]),
    Task(id="5", subject="MNA 营养评估",
         description="18个条目营养筛查",
         metadata={"scale": "MNA", "total_questions": 18}, blocked_by=["4"]),
])
```

**Plan 工具使用流程**：
1. 开始一个维度前：`TaskUpdate(task_id="1", status="in_progress")`
2. 维度内题目通过自定义问卷工具推进（Plan 不管理题目级别）
3. 维度完成后：`TaskUpdate(task_id="1", status="completed")`，LLM 自动通过 TaskList 看到下一个可执行维度
4. 老人想跳过时：`TaskUpdate(task_id="3", status="deleted")` 并自动维护依赖边

## 6. GerClaw 适配要点

### 6.1 老年友好提问方式

在 system_prompt 中必须明确以下对话准则（LLM 遵循度约 80-90%，关键节点仍需 Tool 硬约束）：

- **一次只问一个问题**：禁止"您吃饭穿衣洗漱都能自己做吗？"这种复合问。每轮最多一个信息点。
- **简单语言**：避免医学术语，"日常生活能力"说成"平时自己能做哪些事"；选项用"一点都没有/有几天/经常/几乎每天"而非"0/1/2/3分"。
- **允许重复/回退**：老人说"没听清"必须重复问题（可通过 Tool 返回相同题目）；说"刚才说错了"调用 `previous_question` 工具回退。
- **场景化提问**：不要问"平地行走能否独立45米"，而是问"能自己走到小区门口吗？"
- **敏感问题后置+铺垫**：二便失禁、自杀意念等问题放在后半段，加"接下来这个问题有点直接，但对您健康很重要"铺垫。
- **情绪识别与共情**：老人抱怨、倾诉时先共情（"您确实不容易"），再温和引导回主线，禁止机械跳转。
- **进度透明**：每完成一个维度主动告知进度（"吃饭穿衣这些问完了，大概还剩一半"）。
- **疲劳检测**：识别"累了""不想说了""还有多少题"等信号，主动提议休息并保存 checkpoint。
- **避免恐吓词汇**：MMSE 反馈禁止"痴呆""智力"等词，用"记性可能需要多练练"。

### 6.2 推荐配置

```python
from agentscope.agent import Agent, ReActConfig, ContextConfig
from agentscope.model import DashScopeChatModel
from agentscope.credential import DashScopeCredential
from agentscope.tool import Toolkit
import os

# 模型配置：使用 qwen-plus 或 doubao-pro 级别模型，温度略低保证稳定性
model = DashScopeChatModel(
    credential=DashScopeCredential(api_key=os.environ["DASHSCOPE_API_KEY"]),
    model="qwen-plus",
    parameters=DashScopeChatModel.Parameters(temperature=0.3, max_tokens=1024),
    stream=True,
)

# ReAct 配置：问卷工具调用频繁，max_iters 设大
react_config = ReActConfig(
    max_iters=50,           # 每题约1-3次工具调用，50足够支撑完整CGA
    stop_on_reject=False,   # 问卷工具默认ALLOW，不触发确认
)

# 上下文配置：问卷对话较长但单轮信息密度低，压缩阈值可适当提高
context_config = ContextConfig(
    trigger_ratio=0.7,      # 70%时触发压缩
    reserve_ratio=0.15,     # 保留最近15%消息
)

toolkit = Toolkit(tools=[
    # === 问卷流转工具（核心）===
    RecordAnswerTool(),     # 记录答案
    NextQuestionTool(),     # 下一题
    PreviousQuestionTool(), # 回退
    JumpToSectionTool(),    # 跳转维度
    GetProgressTool(),      # 查看进度
    SaveCheckpointTool(),   # 保存断点
    # === Plan 工具 ===
    TaskCreate(), TaskList(), TaskUpdate(), TaskGet(),
])

agent = Agent(
    name="小葛",
    system_prompt=CGA_SURVEY_PROMPT,
    model=model,
    toolkit=toolkit,
    react_config=react_config,
    context_config=context_config,
)
```

### 6.3 风险与应对

| 风险 | 场景描述 | AgentScope 应对方案 |
|---|---|---|
| **答非所问** | 老人扯家常、抱怨、讲故事，不回答当前问题 | system_prompt 引导"先共情再温和拉回"；Tool 中检测答案无效时返回澄清提示；连续 3 次无效则切换为按钮选择模式（前端降级） |
| **中断恢复** | 老人累了/来电话/网络断，需从断点继续 | 每题答完自动写入 `middle_context["survey"]["checkpoint"]`；`AgentState` 可序列化保存到 Redis；下次启动时重建 Agent 并通过 `observe()` 注入恢复消息 + 预置 tasks_context |
| **LLM 跳题/漏题** | 模型可能跳过某些条目或一次问多个问题 | 自定义 Tool 内部做校验：题号必须连续，未答题目不能前进；在 `NextQuestionTool` 返回值中明确告知"当前第N题，共M题"，用 HintBlock 每轮注入 |
| **幻觉评分** | LLM 对老人模糊回答给错分数 | 评分逻辑放在 Tool 内部（基于规则/评分锚点），不让 LLM 直接打分；Tool 接受 score 参数时做范围校验 |
| **高危条目遗漏** | PHQ-9 第9题自杀意念必须问 | Plan 模式中标记为不可删除任务；或在 Tool 中做硬约束：总分达到阈值时强制插入该题 |
| **重复提问** | 老人记不清之前的回答，Agent 重复问已答题 | `middle_context` 已记录答案，Tool 检测到已答题时返回"您之前说的是..."避免重复 |
| **Token 超限** | CGA 完整评估可能 100+ 轮对话 | 利用 AgentScope 自动上下文压缩（`ContextConfig`）；每个维度完成后主动调用 `agent.compress_context()` 生成摘要；答案结构化存在 `middle_context` 不占对话 token |
| **权限弹窗打断** | FunctionTool 默认 ASK 权限，会弹出确认让老人困惑 | 所有自定义问卷 Tool 继承 ToolBase，`check_permissions` 硬编码返回 ALLOW |
| **疲劳导致数据质量下降** | 老人后期回答敷衍（"差不多""随便"） | 检测模糊回答频率，疲劳分数写入 `middle_context`；超过阈值主动提议休息，保存 checkpoint |

## 7. 可运行示例指引

本模块配套两个可运行示例，位于 `agentscope-examples/03_cga_survey/` 目录：

### 示例1：`cga_planning_agent.py`

**演示内容**：使用 Agent 的 `system_prompt` 引导 CGA 评估按 ADL→IADL→MMSE→GDS→MNA 顺序分步骤提问。每次只问一个问题，答案记录到 `state.context`（通过对话历史隐式存储）。使用 Plan 工具管理维度级进度，使用 HintBlock 在每轮注入进度提示。

**运行方式**：

```bash
cd agentscope-examples/03_cga_survey
export DASHSCOPE_API_KEY="your-key-here"
python cga_planning_agent.py
```

**核心模式**：
- 预置 5 个 Task 对应 5 个评估维度
- system_prompt 严格约束"一次只问一题"
- 使用 `FunctionTool`（is_read_only 版）或自定义 ToolBase 记录答案
- 演示前 3 轮对话（ADL 前 3 题），输出 agent 回复和状态

### 示例2：`survey_state_machine.py`

**演示内容**：基于 Agent 状态 + 自定义 Tool 的完整问卷状态机。实现 4 个核心工具：
- `next_question(answer, score)`：记录答案并前进
- `previous_question()`：回退上一题
- `jump_to_section(section_name)`：跳转到指定维度
- `get_progress()`：获取当前进度

模拟李奶奶 CGA 评估中"ADL 做到第 3 题时家里来客人中断，2 小时后回来继续"的断点恢复场景。展示：
1. 正常答题流程（ADL 前 3 题）
2. 中断 → 序列化保存 state
3. 恢复 → 重建 Agent + 反序列化 state + observe 恢复消息
4. 回退修改上一题答案
5. 跳转维度到 GDS

**运行方式**：

```bash
cd agentscope-examples/03_cga_survey
export DASHSCOPE_API_KEY="your-key-here"
python survey_state_machine.py
```

**前置依赖**：

```bash
pip install agentscope>=2.0.3
# 如使用 DashScope：
pip install openai
```

**注意事项**：
- 两个示例均使用 DashScopeChatModel（通义千问），需设置 `DASHSCOPE_API_KEY` 环境变量。
- 示例使用模拟老人回答（无需真实语音交互），方便开发调试。
- 生产环境需接入 ASR/TTS、适老化 UI、Redis 状态持久化、医疗规则引擎校验等模块，参考需求文档 §5 系统架构。
- 示例中的题目文本为简化版，实际使用需替换为经医学审核的完整量表条目。
