## 2. 核心API参考

### 2.1 Agent构造与核心方法

```python
from agentscope.agent import Agent, ContextConfig, ModelConfig, ReActConfig
```

**构造函数签名**：

```python
Agent(
    name: str,                              # Agent名称（如"老年问诊助手"）
    system_prompt: str,                     # 系统提示词
    model: ChatModelBase,                   # LLM模型实例（必填）
    toolkit: Toolkit | None = None,         # 工具注册中心
    middlewares: list[MiddlewareBase] | None = None,  # 中间件列表
    state: AgentState | None = None,        # 状态对象（None则自动创建）
    model_config: ModelConfig | None = None,      # 模型重试/fallback配置
    context_config: ContextConfig | None = None,  # 上下文压缩配置
    react_config: ReActConfig | None = None,      # ReAct循环配置
)
```

**核心方法**：

| 方法 | 签名 | 说明 |
|---|---|---|
| `reply()` | `async def reply(inputs: Msg \| list[Msg] \| ...) -> Msg` | 非流式回复，消费所有事件后返回最终Msg |
| `reply_stream()` | `async def reply_stream(inputs: ...) -> AsyncGenerator[AgentEvent, None]` | **流式回复**，yield AgentEvent事件流（核心方法） |
| `observe()` | `async def observe(msgs: Msg \| list[Msg] \| None) -> None` | 接收外部观察消息存入上下文（不触发推理） |
| `compress_context()` | `async def compress_context(...) -> None` | 手动触发上下文压缩 |

**inputs多态类型**：
- `Msg` / `list[Msg]`：传入用户消息开始新回复
- `UserConfirmResultEvent`：用户确认后恢复被中断的回复
- `ExternalExecutionResultEvent`：外部执行结果回传后恢复
- `None`：无新输入，从当前状态继续

### 2.2 Msg工厂函数

```python
from agentscope.message import Msg, UserMsg, AssistantMsg, SystemMsg, Usage
```

```python
# 用户消息（content可以是str或list[TextBlock | DataBlock]）
UserMsg(name: str, content: str | list[TextBlock | DataBlock],
        metadata: dict | None = None) -> Msg  # role="user"

# 助手消息（content可以包含所有Block类型）
AssistantMsg(name: str, content: str | list[ContentBlock],
             usage: Usage | None = None) -> Msg  # role="assistant"

# 系统消息（content只能是str或list[TextBlock]）
SystemMsg(name: str, content: str | list[TextBlock]) -> Msg  # role="system"
```

> **注意**：传入`str`类型content会自动包装为`TextBlock`。三个工厂函数返回的都是`Msg`实例。

**Msg类字段**：

| 字段 | 类型 | 说明 |
|---|---|---|
| `name` | `str` | 发送者名称 |
| `role` | `"user"` / `"assistant"` / `"system"` | 发送者角色 |
| `content` | `list[ContentBlock]` | 消息内容块列表 |
| `id` | `str` | 自动生成的唯一ID |
| `metadata` | `dict` | 附加元数据 |
| `created_at` | `str` | ISO格式创建时间 |
| `finished_at` | `str \| None` | 完成时间 |
| `usage` | `Usage \| None` | token使用统计 |

**Msg关键方法**：

| 方法 | 说明 |
|---|---|
| `get_text_content(separator="\n") -> str \| None` | 获取所有TextBlock拼接文本 |
| `get_content_blocks(block_type) -> Sequence[ContentBlock]` | 按类型过滤获取content blocks |
| `has_content_blocks(block_type) -> bool` | 检查是否包含指定类型block |
| `append_event(event: AgentEvent) -> Self` | 从流式事件增量构建消息 |

### 2.3 ContentBlock体系

```python
from agentscope.message import (
    TextBlock, ThinkingBlock, HintBlock,
    ToolCallBlock, ToolResultBlock, DataBlock,
    ToolCallState, ToolResultState,
    Base64Source, URLSource,
)
```

| Block类型 | type字段 | 关键字段 | 用途 |
|---|---|---|---|
| `TextBlock` | `"text"` | `text: str` | 普通文本内容（用户/助手/系统均可） |
| `ThinkingBlock` | `"thinking"` | `thinking: str` | 模型思考/推理过程（仅assistant） |
| `HintBlock` | `"hint"` | `hint: str \| list[TextBlock \| DataBlock]`, `source: str \| None` | ReAct循环中的指令提示（传给LLM时转为user message） |
| `ToolCallBlock` | `"tool_call"` | `id: str`, `name: str`, `input: str`(JSON), `state: ToolCallState` | 工具调用请求 |
| `ToolResultBlock` | `"tool_result"` | `id: str`, `name: str`, `output: str \| list`, `state: ToolResultState` | 工具执行结果 |
| `DataBlock` | `"data"` | `source: Base64Source \| URLSource`, `name: str \| None` | 多模态数据（图片/音频/文件） |

**角色约束**：
- `user`角色：content只能包含`TextBlock`和`DataBlock`
- `system`角色：content只能包含`TextBlock`
- `assistant`角色：允许所有Block类型

**ToolCallState状态机**：
```
pending → ASK → asking → 用户批准 → allowed → submitted → finished
pending → ALLOW → allowed → (本地执行) → finished
pending → DENY → finished
```

**ToolResultState枚举**：`SUCCESS` / `ERROR` / `INTERRUPTED` / `DENIED` / `RUNNING`

### 2.4 AgentEvent类型体系

```python
from agentscope.event import (
    EventType,
    ReplyStartEvent, ReplyEndEvent, ExceedMaxItersEvent,
    ModelCallStartEvent, ModelCallEndEvent,
    TextBlockStartEvent, TextBlockDeltaEvent, TextBlockEndEvent,
    ThinkingBlockStartEvent, ThinkingBlockDeltaEvent, ThinkingBlockEndEvent,
    DataBlockStartEvent, DataBlockDeltaEvent, DataBlockEndEvent,
    HintBlockEvent,
    ToolCallStartEvent, ToolCallDeltaEvent, ToolCallEndEvent,
    ToolResultStartEvent, ToolResultTextDeltaEvent, ToolResultEndEvent,
    RequireUserConfirmEvent, RequireExternalExecutionEvent,
    UserConfirmResultEvent, ExternalExecutionResultEvent,
    CustomEvent,
)
```

**事件生命周期顺序**（单次reply的典型序列）：

```
ReplyStartEvent
  └─ ModelCallStartEvent (model_name)
       ├─ ThinkingBlockStartEvent → ThinkingBlockDeltaEvent* → ThinkingBlockEndEvent  [可选，思考过程]
       ├─ HintBlockEvent                                                    [可选，提示指令]
       ├─ TextBlockStartEvent → TextBlockDeltaEvent* → TextBlockEndEvent   [文本输出]
       ├─ ToolCallStartEvent → ToolCallDeltaEvent* → ToolCallEndEvent      [工具调用请求]
       │    └─ ToolResultStartEvent → ToolResultTextDeltaEvent* → ToolResultEndEvent [工具结果]
       │         └─ [可能进入下一轮ModelCallStartEvent，ReAct循环]
       └─ ModelCallEndEvent (input_tokens, output_tokens)
  └─ ModelCallEndEvent
ReplyEndEvent
```

**事件分类**：

| 分类 | 事件 | 关键字段 | GerClaw用途 |
|---|---|---|---|
| 生命周期 | `ReplyStartEvent` | `session_id`, `reply_id`, `name` | 问诊会话开始 |
| 生命周期 | `ReplyEndEvent` | `session_id`, `reply_id` | 问诊回复完成 |
| 生命周期 | `ExceedMaxItersEvent` | `reply_id`, `name` | 推理轮次超限（转人工信号） |
| 模型调用 | `ModelCallStartEvent` | `model_name` | 展示"AI正在分析..." |
| 模型调用 | `ModelCallEndEvent` | `input_tokens`, `output_tokens` | token用量统计/审计 |
| 思考过程 | `ThinkingBlock*Event` | `block_id`, `delta` | **老年友好：展示推理过程** |
| 文本输出 | `TextBlock*Event` | `block_id`, `delta` | **逐字流式输出回复** |
| 工具调用 | `ToolCall*Event` | `tool_call_id`, `tool_call_name`, `delta` | 展示"正在查询药品库..." |
| 工具结果 | `ToolResult*Event` | `tool_call_id`, `delta`, `state` | 工具返回结果 |
| HITL | `RequireUserConfirmEvent` | `tool_calls: list[ToolCallBlock]` | **医生确认节点** |
| HITL | `UserConfirmResultEvent` | `confirm_results: list[ConfirmResult]` | 医生审批结果回传 |
| 自定义 | `CustomEvent` | `name: str`, `value: dict` | 前端状态通知 |

**事件分发模式（match语句）**：

```python
async for event in agent.reply_stream(user_msg):
    match event.type:
        case EventType.THINKING_BLOCK_DELTA:
            print(f"[思考中] {event.delta}", end="", flush=True)
        case EventType.TEXT_BLOCK_DELTA:
            print(f"{event.delta}", end="", flush=True)
        case EventType.TOOL_CALL_START:
            print(f"\n[调用工具] {event.tool_call_name}")
        case EventType.REQUIRE_USER_CONFIRM:
            # 触发医生确认UI
            ...
        case EventType.REPLY_END:
            print("\n[回复完成]")
```

### 2.5 配置类

```python
from agentscope.agent import ContextConfig, ReActConfig, ModelConfig
```

**ContextConfig（上下文压缩配置）**：

| 字段 | 类型 | 默认值 | GerClaw建议 |
|---|---|---|---|
| `trigger_ratio` | `float` | `0.8` | 建议`0.85`（医疗对话需更多上下文） |
| `reserve_ratio` | `float` | `0.1` | 建议`0.2`（保留更多医疗历史） |
| `compression_prompt` | `str` | 内置 | **必须自定义**：要求保留症状/诊断/用药锚点 |
| `tool_result_limit` | `int` | `50000` | 药品库查询结果需适当调大 |

**ReActConfig（ReAct循环配置）**：

| 字段 | 类型 | 默认值 | GerClaw建议 |
|---|---|---|---|
| `max_iters` | `int` | `20` | 建议`10`（医疗场景避免无限推理循环） |
| `stop_on_reject` | `bool` | `False` | 建议`True`（医生拒绝时停止等待） |

**ModelConfig（模型重试配置）**：

| 字段 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `max_retries` | `int` | `0` | 建议`3`（API超时重试） |
| `fallback_model` | `ChatModelBase \| None` | `None` | 降级模型（如qwen-plus降级为qwen-turbo不可用于诊断） |

### 2.6 DashScopeChatModel构造

```python
from agentscope.credential import DashScopeCredential
from agentscope.model import DashScopeChatModel
```

**标准构造方式**：

```python
model = DashScopeChatModel(
    credential=DashScopeCredential(
        api_key=os.environ["DASHSCOPE_API_KEY"],  # 从环境变量读取，不硬编码
    ),
    model="qwen-max",                              # 模型名称
    stream=True,                                   # 启用流式输出（老年友好必需）
    max_retries=3,                                 # 网络重试次数
    retry_delay=1.0,                               # 重试间隔（秒）
    context_size=131072,                           # 上下文窗口大小
    parameters=DashScopeChatModel.Parameters(
        thinking_enable=True,                      # 启用思考模式（诊断推理透明化）
        temperature=0.3,                           # 医疗场景建议低温度（0.1-0.4）
        max_tokens=4096,                           # 最大输出token
    ),
)
```

**DashScopeChatModel.Parameters可配参数**：

| 参数 | 类型 | 说明 |
|---|---|---|
| `max_tokens` | `int \| None` | 最大输出token数 |
| `thinking_enable` | `bool` | 是否启用思考模式 |
| `thinking_budget` | `int \| None` | 思考token预算 |
| `temperature` | `float \| None` | 温度参数（0-2） |
| `top_p` | `float \| None` | Top-P采样（0-1） |
| `top_k` | `int \| None` | Top-K采样（1-100） |
| `parallel_tool_calls` | `bool` | 是否允许并行工具调用 |

## 6. GerClaw适配要点

### 6.1 老年对话体验：流式输出/思考过程可见

老年用户对等待的容忍度较低，且需要理解AI"为什么这么判断"以建立信任。AgentScope的流式事件架构天然支持这一需求：

1. **逐字流式输出**：通过`reply_stream()`监听`TextBlockDeltaEvent`，将`event.delta`实时渲染到前端，模拟"逐字打字"效果，避免老年用户长时间面对空白屏幕。

2. **思考过程透明化**：启用`thinking_enable=True`后监听`ThinkingBlockDeltaEvent`，以折叠面板/灰色文字/标注"AI思考中"的方式展示推理链（症状分析→鉴别诊断→排除依据），让医生和患者家属都能看到诊断推理过程。

3. **工具调用状态提示**：监听`ToolCallStartEvent`展示"正在查询药品相互作用数据库..."、`ToolResultEndEvent`展示查询完成，消除等待焦虑。

4. **大字版+分段落**：在TextBlock输出后处理中，自动将长文本按句号分段，关键信息（用药剂量/就诊科室）加粗高亮。

### 6.2 推荐配置Python代码块

```python
import os
from agentscope.agent import Agent, ContextConfig, ReActConfig, ModelConfig
from agentscope.credential import DashScopeCredential
from agentscope.model import DashScopeChatModel
from agentscope.tool import Toolkit

# ===== GerClaw老年问诊Agent推荐配置 =====

model = DashScopeChatModel(
    credential=DashScopeCredential(
        api_key=os.environ["DASHSCOPE_API_KEY"],
    ),
    model="qwen-max",
    stream=True,                                    # 必须启用流式
    max_retries=3,                                  # API重试3次
    retry_delay=1.0,
    context_size=131072,
    parameters=DashScopeChatModel.Parameters(
        thinking_enable=True,                       # 启用思考过程（可解释性）
        temperature=0.3,                            # 医疗场景低温度
        max_tokens=4096,
        top_p=0.8,
    ),
)

agent = Agent(
    name="GerClaw老年问诊助手",
    system_prompt=(
        "你是GerClaw老年医疗AI平台的问诊助手。"
        "你的服务对象是75岁左右的老年患者及其家属。"
        "请用通俗易懂的语言回答，避免专业术语，必要时解释医学名词。"
        "对于诊断结论必须给出依据和置信度，不得做确定性断言。"
        "处方类建议必须标注'请咨询医生确认'。"
        "如发现红旗症状（胸痛/呼吸困难/意识障碍等），立即建议拨打120。"
    ),
    model=model,
    toolkit=Toolkit(),  # 后续注册药品查询/EMR读取等工具
    model_config=ModelConfig(
        max_retries=3,                               # 模型级别重试
    ),
    context_config=ContextConfig(
        trigger_ratio=0.85,                          # 85%时触发压缩（医疗需更多上下文）
        reserve_ratio=0.2,                           # 压缩后保留20%
        tool_result_limit=80000,                     # 药品库结果可能较长
        compression_prompt=(                         # 自定义压缩prompt：保留医疗锚点
            "请压缩以下问诊对话历史，生成结构化摘要。"
            "必须保留：1)患者基本信息(姓名/年龄/性别) 2)主诉症状 3)既往病史/用药清单 "
            "4)已排除的鉴别诊断 5)当前诊断假设和置信度 6)待确认的关键问题。"
            "可压缩：寒暄语、重复描述、思考过程细节。"
        ),
    ),
    react_config=ReActConfig(
        max_iters=10,                                # 最多10轮ReAct循环（防止无限推理）
        stop_on_reject=True,                         # 医生拒绝时停止等待
    ),
)
```

### 6.3 风险：长上下文压缩可能丢失医疗细节

**风险描述**：AgentScope的自动上下文压缩使用LLM生成摘要（SummarySchema），在多轮问诊对话中可能丢失以下关键医疗信息：

1. **症状时序细节**：如"胸痛3天，昨天加重"被压缩为"胸痛"，丢失时间演变信息
2. **药物剂量和频次**：如"氨氯地平5mg每日一次"被压缩为"服用降压药"，丢失精确用药信息
3. **鉴别诊断排除依据**：被排除的诊断及其排除理由可能被摘要省略
4. **患者口语化关键描述**：老年患者的非标准表述（如"心里像揣了个兔子"=心悸）可能在压缩中丢失

**缓解措施**：

1. **自定义compression_prompt**（见6.2代码）：明确要求保留症状时序、用药剂量、鉴别诊断锚点
2. **医疗锚点结构化存储**：通过`agent.observe()`注入结构化的`DataBlock`（JSON格式的患者摘要），而非完全依赖自然语言压缩
3. **提高reserve_ratio**：从默认0.1提高到0.2，保留更多原始对话
4. **关键信息不压缩**：将用药清单、过敏史、红旗症状标记为不可压缩的`SystemMsg`或`HintBlock`，始终保留在context中
5. **压缩后审计**：监听压缩事件（可通过中间件实现），在审计日志中记录压缩前后的token数和摘要内容
6. **转人工保护**：当`ExceedMaxItersEvent`触发或压缩次数超过阈值时，自动转人工医生
