## 一、项目长期目标

1. **MVP上线验证**：纯前端Next.js 15静态导出版本部署到IGA Pages，核心功能闭环（对话/语音/CGA/五大处方/用药审查/适老化），可演示可体验，收集真实用户反馈验证产品价值
2. **二阶段全栈架构**：平滑迁移到FastAPI + AgentScope多智能体 + PostgreSQL/Redis/Qdrant全栈架构，Docker部署ModelScope，实现账号系统/RAG知识库/权限引擎/HITL人工审批/双智能体复核，支撑企业级生产使用
3. **临床试点验证**：与医疗机构合作开展临床试点，在真实老年科诊疗场景中验证系统有效性和安全性，获取临床反馈持续迭代
4. **商业化落地**：基于试点验证结果，探索SaaS订阅、医疗机构授权等商业化路径，成为老年科AI辅助诊疗的标准工具

## 二、产品原则

（从PRD铁律和core-beliefs提取）

1. **医疗安全是不可逾越的底线**：宁可功能少做或提醒"不确定请就医"，也绝不能输出可能误导老年用户的错误医疗建议；禁止给出确定性诊断结论
2. **适老化不是放大版普通UI**：老年人需要的是大字体（≥18px）、大按钮（≥48px）、语音优先、步骤提示、确认对话框，不是把年轻人的UI放大字号
3. **简单优于复杂**：能用一个大麦克风按钮解决的，不要放三个小按钮；能用自然语言对话完成的，不要做表单
4. **专业可信胜过花哨**：医疗产品的可信度来自循证引用、结构化输出、免责声明，不是酷炫动画
5. **语音是患者端的第一入口**：老年人打字困难，语音识别+语音合成必须作为一等公民支持
6. **流式输出是刚需不是加分项**：老人等不了30秒看一大段文字，逐字流式输出让他们边看边读
7. **配置可变硬编码可耻**：API Key、模型名、URL、音色一切可配置，代码里不出现任何特定服务的硬编码
8. **模块化先行快速验证**：MVP纯前端快速验证核心价值，二阶段平滑迁移到全栈架构，不做过度设计
9. **循证可溯源**：所有医学建议必须标注来源依据（联网搜索结果/指南引用），禁止编造医学知识或引用不存在的文献
10. **工具调用透明可见**：AI搜索了什么、用了什么工具、思考过程是什么，用户都能看到，不做黑盒
11. **真实执行优先**：禁止空谈"应该可以"，必须实际运行npm run build/npm run dev并附完整输出
12. **项目视角优先**：禁止破坏现有逻辑，改完必须跑全量构建和关键路径测试
13. **禁止虚伪成功**：禁止模糊表述"已完成"，失败必须如实报告错误信息和复现步骤
14. **不吞错误**：API失败、解析失败、存储失败都要如实记录和提示用户，不静默失败
15. **失败快优于静默失败**：错误尽早抛出并展示给用户，不吞异常不假装成功
