# 长期规划

> 项目长期记忆。完成度以生产证据为准；MVP 页面存在不等于生产后端能力完成。

## 一、项目长期目标

1. 交付患者端与医生端均可真实使用的 FastAPI + AgentScope + PostgreSQL/Redis/Qdrant 全栈。
2. 让 CGA、五大处方、用药审查、健康画像、语音、文档和 Skill 进入同一 Runtime Agent Harness。
3. 建立医疗安全、循证引用、权限/HITL、Trace、反馈/Bad Case/Eval 和数据加密闭环。
4. 以可重复 Development Harness、≤10 并发证据和 Docker 空卷运行完成生产验收，再进入临床试点。

## 二、产品原则

1. 医疗安全不可让步：禁止确定性诊断；红旗症状立即就医；所有医疗输出带免责声明。
2. 无证据不输出建议：本地 RAG 优先，必要时联网，引用必须可追踪，失败必须 fail closed。
3. 适老化不是简单放大：患者端正文≥18px、控件≥48px、高对比、语音优先、步骤清晰。
4. 访客可用但不越权：访客无需登录体验核心功能；账号、角色、患者授权和资源归属全链路隔离。
5. 模型和工具不可信：输入、输出、上传、搜索、MinerU、缓存和向量 payload 都要 schema 与策略校验。
6. 真实执行优先：mock、静态示例、历史截图或“应该可用”都不是完成证据。
7. 模块化但不做 Demo：每个模块必须有 Protocol、生产实现、错误语义、测试、README 和真实集成证据；docs gate 会拒绝缺少 AGENTS.md/README.md 的含实现源码核心模块。
8. 只记录必要审计信息：不存 Chain-of-Thought，不在日志/Trace/向量库泄露 PHI 或密钥。
