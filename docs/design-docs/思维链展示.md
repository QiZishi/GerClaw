# 思维链展示 — 设计文档

> 模块：思维链展示 | 关联产品规格：[product-specs/思维链展示.md](../product-specs/思维链展示.md)

---

## 1. 设计目标

基于React组件实现模型思维链（Chain-of-Thought）的优雅折叠展示，满足以下设计目标：

1. **默认低干扰**：思考过程默认折叠为低对比度区块，不抢占用户注意力，正文是视觉焦点
2. **流式感知反馈**：AI思考时显示"思考中..."动画，让用户知道系统正在工作，减少等待焦虑
3. **可解释性按需查看**：用户可主动点击展开查看完整推理过程，满足医生用户验证AI逻辑的需求，老年用户可忽略
4. **多协议兼容解析**：支持三种thinking内容来源（OpenAI reasoning_content字段、Anthropic thinking block、`<think>`标签兜底），兼容不同模型
5. **安全内容过滤**：思维链内容在复制、导出操作中自动排除，不出现在最终输出文档中
6. **体验对齐Trae Work**：折叠/展开交互、视觉样式参考Trae Work的思考过程展示，保持交互一致性
7. **适老化无障碍**：组件支持键盘导航、ARIA属性，老年模式下字号放大、点击区域足够大

约束：
- 作为通用对话模块的子组件，与ContentBlockRenderer体系集成
- Vercel AI SDK流式delta解析逻辑在StreamingChatClient中处理，本组件只负责渲染
- 不依赖后端服务，纯前端展示组件

## 2. 架构设计

### 2.1 模块位置

在整体架构中的位置：**前端UI组件层**，位于`components/chat/blocks/thinking-block.tsx`，是通用对话模块中ContentBlockRenderer的子组件，专门负责thinking类型内容块的渲染。

- **上层依赖（使用者）**：
  - ContentBlockRenderer：消息内容块分发器，遇到`type: 'thinking'`的block时渲染本组件
  - Message组件：消息容器，提供消息上下文（是否流式、是否最后一条AI消息）
- **本模块内部**：ThinkingBlock组件自身（折叠/展开状态、样式、动画）
- **下层依赖**：
  - Vercel AI SDK流解析：在StreamingChatClient中解析thinking delta并构建ThinkingBlock数据
  - 结果导出模块：sanitizeExportContent过滤掉thinking-block元素
  - ShadCN/UI基础组件：Button、Collapsible等
  - Tailwind CSS样式

### 2.2 组件划分

| 组件 | 职责 | 文件位置（预期） |
|------|------|----------------|
| **ThinkingBlock** | 主组件，接收ThinkingBlock数据，管理展开/折叠状态，渲染折叠/思考中/展开三种UI状态 | `components/chat/blocks/thinking-block.tsx` |
| **ThinkingSpinner** | "思考中..."状态的旋转spinner+文字展示组件 | `components/chat/blocks/thinking-block/thinking-spinner.tsx` |
| **ThinkingCollapsed** | 折叠状态视图：截断2行文本、"展开"按钮、箭头图标 | `components/chat/blocks/thinking-block/thinking-collapsed.tsx` |
| **ThinkingExpanded** | 展开状态视图：完整文本、"收起"按钮、缩进边框样式 | `components/chat/blocks/thinking-block/thinking-expanded.tsx` |
| **useThinkingBlock** | 本组件的自定义Hook：管理折叠状态、动画控制、键盘交互 | `components/chat/blocks/thinking-block/use-thinking-block.ts` |
| **thinkingParser** | 工具函数：从SSE delta中解析thinking内容，支持三种协议/格式 | `lib/ai/thinking-parser.ts` |

> 注：MVP阶段为简化起见，子组件可合并到单文件中，不必过度拆分。

```

│                    StreamingChatClient (流式客户端)                      │
│                                                                         │
│  SSE chunk接收：                                                        │
│  ├─ OpenAI协议：delta.reasoning_content 存在？                           │
│  │   └─ 是 → 追加到 currentThinking 字符串                             │

│  │   └─ 是 → 追加 thinking.thinking 字符串                             │

│                                                                         │
│  构建 ThinkingBlock 对象：                                              │
│  { id, type: 'thinking', content, isCollapsed: true, isStreaming }      │

│                        ContentBlockRenderer                             │
│  遍历消息blocks数组，根据block.type分发渲染                             │
│  block.type === 'thinking' → 渲染 <ThinkingBlock block={block} />       │
└───────────────────────────────┬─────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                          ThinkingBlock 组件                             │
│                                                                         │
│  内部状态：                                                             │
│  - isCollapsed（默认true）                                              │
│  - isAnimating（动画进行中，防快速点击）                                │
│                                                                         │
│  渲染逻辑：                                                             │
│  ├─ block.isStreaming === true → <ThinkingSpinner />                   │
│  │   （同时后台追加内容，折叠视图显示前2行半透明预览）                   │
│  ├─ block.isStreaming === false && isCollapsed → <ThinkingCollapsed /> │
│  └─ !isCollapsed → <ThinkingExpanded />                                │
│                                                                         │
│  交互：                                                                 │
│  - 点击区块 → toggleCollapse() → 触发height动画                        │
│  - 键盘Enter/Space → 同上                                               │
│  - 自动：isStreaming从true变false时，确保isCollapsed=true（默认收起）   │
└───────────────────────────────┬─────────────────────────────────────────┘
                                │
        ┌───────────────────────┼───────────────────────┐
        ▼                       ▼                       ▼
┌───────────────┐     ┌─────────────────────┐   ┌─────────────────┐
│ 复制操作      │     │ 导出操作            │   │ 正常展示        │
│ copyMessage() │     │ sanitizeExport()    │   │ 用户可见        │
│ 过滤thinking  │     │ 移除.thinking-block │   │ 可展开/收起     │
│ 只复制正文    │     │ 不出现在导出文件    │   │ 查看推理过程    │
└───────────────┘     └─────────────────────┘   └─────────────────┘
```

## 3. 接口设计

### 3.1 对外接口

| 接口 | 类型 | 参数 | 返回值 | 说明 |
|------|------|------|--------|------|
| `<ThinkingBlock />` | React组件 | `{ block: ThinkingBlock, defaultCollapsed?: boolean }` | JSX.Element | 思维链折叠块主组件 |
| `parseThinkingDelta(chunk: any, protocol: string): string \| null` | 工具函数 | SSE chunk对象、协议类型 | 新的thinking文本增量或null | 从SSE chunk中提取thinking增量内容 |
| `stripThinkingFromContent(text: string): string` | 工具函数 | 含think标签的完整文本 | 过滤后的正文 | 兜底方案：从正文中移除`<think>...</think>`标签内容 |
| `sanitizeThinkingForCopy(blocks: ContentBlock[]): string` | 工具函数 | 消息content blocks数组 | 纯文本（不含thinking） | 复制消息时过滤思维链内容，只返回正文文本 |

**ThinkingBlock Props**：
```typescript
interface ThinkingBlockProps {
  block: ThinkingBlock;        // 思维链数据
  defaultCollapsed?: boolean;  // 默认折叠状态，默认true
  onToggle?: (isCollapsed: boolean) => void;  // 展开/折叠回调（可选）
}
```

### 3.2 依赖接口

| 依赖 | 来源 | 用途 | 失败处理 |
|------|------|------|---------|
| Vercel AI SDK useChat() onToolCall/onFinish回调 | npm包 `ai` | 流式delta接收，思考结束状态判断 | 如SDK未暴露thinking字段（旧版本），使用<think>标签兜底解析 |
| ShadCN/UI Collapsible组件 | `components/ui/collapsible` | 展开/折叠动画基础组件 | 如Collapsible有问题，使用纯CSS max-height transition实现 |
| Tailwind CSS utility classes | 全局样式 | 样式（背景、字号、颜色、间距、过渡动画） | 无降级，Tailwind是项目基础依赖 |
| Lucide React图标 | npm包 `lucide-react` | 展开/收起箭头、spinner图标（ChevronDown/ChevronUp/Loader2） | 图标库全局可用，无降级问题 |
| 结果导出模块sanitize函数 | `lib/export/sanitize.ts` | 导出时移除thinking-block元素 | 在sanitize中添加`.thinking-block`选择器，找不到则静默跳过 |

## 4. 数据设计

### 4.1 数据库表/集合（如适用）

本模块为纯UI展示组件，ThinkingBlock数据作为Message的一部分随消息一起存储到localStorage中，无独立存储结构。

**ThinkingBlock在LocalStorage中的序列化**：
- 完整的`content`文本字符串随消息保存
- `isCollapsed`状态不持久化（每次加载默认折叠，因为用户不会期望刷新后所有思考块都是展开的）
- `isStreaming`序列化时始终为false（刷新页面后不存在流式状态）

### 4.2 状态机

**ThinkingBlock自身状态机**：

```
idle（无内容，不渲染）→ 收到第一个thinking delta → streaming（思考中）
  │                            │
  │                            ├─ 流式追加content
  │                            └─ 收到text delta/流结束 → collapsed（折叠，默认状态）
  │                                                         │
  │                                                         ├─ 用户点击 → expanding（展开动画中）
  │                                                         │    └─ 动画结束 → expanded（展开状态）
  │                                                         │                    │
  │                                                         │                    └─ 用户点击 → collapsing（折叠动画中）
  │                                                         │                         └─ 动画结束 → collapsed
  │                                                         │
  │                                                         └─ 新消息重置 → idle
  └─ 无thinking内容 → 保持idle，不渲染
```

**展开/折叠动画状态**：
```
collapsed → 用户点击 → beforeExpand → expanding → expanded
                                                          ↓
collapsed ← afterCollapse ← collapsing ← 用户点击
```
使用CSS transition驱动max-height变化，无需复杂JS动画库。

## 5. 错误处理

| 错误类型 | 处理方式 | 用户反馈 | 日志级别 |
|---------|---------|---------|---------|
| thinking delta解析失败（格式异常、字段不存在） | 返回null，不追加内容，继续解析下一个chunk | 无感知（没有思维链内容时不显示区块），正文正常显示 | warn（记录无法识别的chunk片段，debug用） |
| `<think>`标签不闭合（流式过程中标签打开但未关闭） | 流式过程中保持收集，流结束时如仍未闭合则丢弃标签内内容（视为格式错误）或自动闭合标签 | 无感知 | warn（记录流结束时标签未闭合） |
| 思考过程超长导致渲染卡顿（>5万字） | 正常渲染，但展开后用overflow-auto滚动 | 无提示；实际模型输出不会达到这个长度 | info（记录超长thinking内容长度） |
| 展开/折叠动画被快速连续点击打断 | 动画状态锁（isAnimating），在动画期间忽略重复点击 | 无感知，动画平滑完成后可再次点击 | - |
| thinking内容包含特殊字符/Markdown/代码 | 作为纯文本渲染（不解析Markdown），避免格式混乱；HTML转义防止XSS | 按纯文本显示，即使包含**或#也原样显示 | -（设计如此） |
| 浏览器不支持CSS transition（极旧浏览器） | 降级为无动画直接切换显示/隐藏 | 功能可用，只是没有平滑动画 | warn（记录userAgent） |

## 6. 测试策略

- **单元测试覆盖**：
  - `lib/ai/thinking-parser.ts`：三种协议的delta解析、<think>标签提取、标签不闭合处理、空输入处理
  - stripThinkingFromContent：正确移除<think>标签、保留其他内容、无标签时原样返回
  - sanitizeThinkingForCopy：过滤掉thinking block，正确拼接其他文本块
  - ThinkingBlock组件：默认折叠渲染、点击展开/收起切换、isStreaming状态显示spinner

- **集成测试覆盖**：
  - 流式回复中包含thinking内容：验证"思考中..."显示→思考结束折叠→点击展开看到完整内容
  - 无thinking内容的模型回复：验证不渲染ThinkingBlock
  - 复制AI消息：验证剪贴板内容不含思考过程文本
  - 导出流程：验证sanitize正确移除thinking-block DOM元素

- **测试场景**：
  - **主流程**：发送消息→看到"思考中..."→思考结束显示折叠块→点击展开→看到完整思考内容→点击收起→复制消息→粘贴验证无思考内容
  - **失败路径1**：模型不返回reasoning字段，验证直接显示正文，无空区块
  - **失败路径2**：thinking内容中包含<think>标签嵌套或不完整，验证容错处理
  - **边界情况**：
    - thinking内容为空字符串（不渲染区块）
    - thinking内容只有1行（折叠时完整显示，无截断）
    - 流式过程中用户点击展开（验证内容实时追加）
    - 老年模式下字体/按钮尺寸符合适老化要求
    - 键盘Tab聚焦到区块→Enter展开→Enter收起
  - **跨协议测试**：分别用OpenAI（reasoning_content）、Anthropic（thinking block）、标签兜底三种格式验证解析正确性

## 7. 安全考虑

- **XSS防护**：
  - thinking内容作为纯文本渲染，使用React默认的文本转义（不使用dangerouslySetInnerHTML）
  - 不解析Markdown，避免thinking中的恶意链接/脚本被渲染
  - `<think>`标签解析只提取纯文本内容，标签本身在正文中被移除

- **医疗安全**：
  - 思维链明确标注为"思考过程"，视觉样式弱化，避免用户将推理过程误认为正式医疗建议
  - 最终导出/复制的内容不包含思维链，确保分享给医生的正式文档只有经过整理的最终回复
  - 免责声明随最终回复显示，不与思考块混淆

- **内容过滤合规**：
  - 思维链作为模型内部推理，可能包含自我修正、不确定的表述，这些内容不展示给最终用户（折叠状态），避免误导
  - 导出和复制时严格排除思维链，符合"AI输出需经整理"的医疗安全原则

- **隐私保护**：
  - 思维链内容仅作为消息的一部分存在于localStorage中（同消息正文），不单独上传
  - 用户删除会话时思维链随消息一起删除

## 8. 可观测性

- **日志（console）**：
  - `debug`：thinking chunk解析成功（增量长度、总长度）
  - `warn`：thinking解析失败（无法识别的chunk格式、标签不闭合）
  - 日志前缀：`[GerClaw][Thinking]`

- **指标（可选，前端内存统计）**：
  - 思维链平均长度（tokens/字符数）
  - 思考平均耗时（从第一个thinking delta到第一个text delta的时间）
  - 用户展开思维链的比例（多少用户会点击展开查看）

- **追踪**：
  - 每条AI消息的ThinkingBlock关联messageId，错误日志携带messageId便于排查

- **告警**：
  - 无服务端告警；解析错误仅记录console日志，不影响用户核心体验

## 9. 技术选型说明

| 技术/库 | 用途 | 选型理由 | 替代方案 |
|---------|------|---------|---------|
| ShadCN/UI Collapsible | 展开折叠容器 | 项目基础UI库已包含，基于Radix UI无障碍支持好（ARIA、键盘导航），内置动画支持 | 自实现div + CSS max-height transition（代码量增加但可更精细控制） |
| CSS max-height + transition | 展开/折叠动画 | 纯CSS动画性能好（GPU加速），无需JS动画库，实现简单 | react-spring/Framer Motion（动画更流畅但引入额外依赖，折叠动画不需要这么复杂） |
| 纯文本渲染（非Markdown） | thinking内容格式化 | 思维链是模型内部草稿，可能包含不完整的Markdown、修改痕迹，纯文本展示最稳定；避免错误格式影响视觉 | react-markdown渲染（可能因为标签不闭合导致渲染异常，且草稿不需要格式） |
| 多策略解析（字段+block+标签） | thinking内容提取 | 不同模型厂商返回思维链的方式不同：通义千问用reasoning_content、Claude用thinking block、本地模型可能需要prompt输出<think>标签，三种方式兼容覆盖面广 | 只支持一种协议（用户切换模型时思维链功能失效） |
| Lucide React Loader2图标 | 思考中spinner | 项目统一图标库，Loader2自带spin动画类（animate-spin），无需手写CSS动画 | 手写CSS spinner（增加CSS代码，图标库已有直接用） |

## 10. 开放问题

- **思考耗时显示**：是否在折叠块标题显示思考耗时（如"思考过程（2.3秒）"）？当前PRD未要求，可作为体验优化项后续添加，数据结构已预留startedAt/endedAt字段
- **多段思考支持**：某些模型可能输出多段thinking（思考→工具调用→再思考→回答），当前数据结构只支持单块，是否需要支持多个ThinkingBlock顺序展示？MVP先支持单块，遇到多段时合并为一个content
- **老年模式下是否默认展开**：老年用户视力/操作能力下降，折叠块可能被忽略，是否需要在老年模式下默认展开？当前决策：统一默认折叠，因为展开后信息密度大可能干扰老年用户；后续根据用户测试再调整
- **自动展开条件**：是否在思考过程中自动展开（显示实时思考流），还是始终等思考完再折叠显示？当前方案：思考中显示"思考中..."spinner，不实时滚动展开思考内容，避免正文位置频繁跳动影响体验
- **思维链与工具调用顺序**：当思维链和工具调用同时存在时（思考→调用工具→再思考→回答），ThinkingBlock和ToolCallBlock在消息流中的排列顺序如何？按流式接收顺序自然排列即可
