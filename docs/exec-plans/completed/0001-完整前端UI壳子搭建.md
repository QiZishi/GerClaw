# 0001-完整前端UI壳子搭建 — 执行计划

> 任务编号：0001 | 创建日期：2026-07-05 | 优先级：P0 | 阶段：一（UI 壳子，允许 mock）

---

## 1. 任务目标

构建 GerClaw 完整前端 UI 壳子，所有页面可交互、所有按钮与布局设计完成，允许使用 mock 数据调试交互效果，**不实现真实功能逻辑**，不调用任何真实模型/工具 API。本计划为两阶段 Mock 策略的第一阶段（依据 `gerclaw设计要求.md` §15.2、`ARCHITECTURE.md` ADR-006），mock 数据集中于 `src/data/mock/`，第二阶段（0002 起）将全部删除并接入真实 API。

## 2. 范围

### 2.1 做什么

#### A. 工程初始化（沿用旧骨架）
- Next.js 15 App Router + TypeScript（strict）+ Tailwind CSS 4 + shadcn/ui
- 目录结构对齐 `ARCHITECTURE.md` 与 `FRONTEND.md`（`app/(patient)/`+`app/(doctor)/` 路由组、`lib/`、`context/`、`stores/`、`services/llm/voice/search/document/`、`data/mock/`、`styles/`）
- `.env.example`（所有 NEXT_PUBLIC_ 环境变量模板）、`next.config.js`（静态导出 `output: 'export'`）、`tsconfig.json`（路径别名 `@/*`）、ESLint、Vitest 配置
- Zustand stores（appStore/chatStore/sessionStore，含 localStorage 持久化中间件）、React Context（ThemeProvider/AppProvider）

#### B. 三栏布局完整实现（对齐 §4.1 / §13 / UI布局.md）
- **左侧边栏**（展开 260-280px / 折叠 60-70px）：
  - 顶部：Logo+名称+模式标签、折叠控制按钮、新建对话按钮、历史搜索框
  - 中部：历史会话列表（mock 数据，按今天/昨天/最近7天/更早分组）、技能管理入口
  - 底部：用户头像区（访客/医生/患者）、模式切换按钮（仅访客）、老年模式开关（仅患者端，默认开启）、主题切换按钮
- **中间聊天区**（弹性宽度）：
  - 欢迎页（功能快捷入口卡片：五大处方生成、CGA评估、用药审查、查看健康画像）
  - 消息列表（mock 消息，含用户/AI 气泡、Markdown 渲染、思维链折叠块、工具调用卡片）
  - 输入框（含 📎文件上传、⚡技能、💊处方、📋评估 按钮组 + 语音按钮 + 发送按钮）
- **右侧动态面板**（默认隐藏 0px，触发时 320-400px，可拖拽 320-500px）：
  - 按触发场景切换内容：技能管理界面、处方预览/导出、CGA 评估量表、文件预览、引用详情列表
  - 顶部关闭按钮、拖拽手柄

#### C. 9 个核心页面/视图完整搭建（对齐 SYSTEM_DESIGN_SPEC.md）
**患者端**：
1. 欢迎页（功能快捷入口卡片 + 适老化大字 + 语音优先入口）
2. 对话主界面（消息流 + 输入框 + 工具卡片）
3. 五大处方生成向导（信息收集表单 → 生成中状态 → 报告预览，右侧面板）
4. CGA 评估答题界面（量表选择 → 逐题答题 → 报告预览，右侧面板）
5. 用药审查结果展示（输入 → 结果卡片）

**医生端**：
6. 患者列表（mock 患者数据，按状态筛选）
7. 对话主界面（与患者端布局一致，主色调冷蓝 #2563EB）
8. CGA 评估工作区（患者评估进度、结果汇总、历史对比）
9. 处方审核编辑界面（报告预览 + 富文本编辑 + 风险提示 + 审核按钮）

#### D. 全部功能入口与按钮就位（可点击触发 UI 状态切换，不调真实 API）
- 角色切换（患者/医生，刷新式切换，UI 差异化：患者端暖蓝 #0EA5E9 / 医生端冷蓝 #2563EB）
- 主题切换（浅/深，即时生效，localStorage 持久化）
- 老年模式开关（即时生效，大字体 18px / 大按钮 48px / 高对比度）
- 五大处方三入口：💊输入框按钮、欢迎页快捷卡片、对话触发（mock 识别"帮我开处方"）
- CGA 评估入口、用药审查入口
- 处方导出按钮组（PDF/Markdown/DOCX 图标按钮，mock 下载提示）
- 语音朗读按钮（mock 播放动画）、编辑/审核按钮（医生端）
- 文件上传（mock 上传进度 + 解析状态标签）、技能选择（mock 技能列表 + 标签）、联网搜索（mock 搜索结果卡片）

#### E. §4.2.3 七项可视化 UI 组件（mock 样例展示）
1. 思维链折叠块（ThinkingBlock）— 默认折叠"思考中..."，点击展开
2. 工具调用卡片（ToolCallBlock）— 运行中/完成/失败三态 + 耗时 + 展开详情
3. 子智能体加载（SubAgentTree）— 折叠树形结构，主智能体→子智能体调用链
4. 决策过程（DecisionTimeline）— ReAct 垂直时间线，步骤+状态+耗时
5. 联网搜索结果卡片（SearchResultCard）— 标题+favicon+摘要+链接，[1][2]角标
6. 文档解析状态标签（FileTag + ToolCard）— 上传中→解析中→完成/失败
7. 流式文本输出（StreamingText）— 打字机效果 + 闪烁光标 + 停止生成按钮

#### F. 适老化老年模式（§13.7）
- 基础字号 18px（普通 14px）、标题 20-28px
- 按钮最小 48×48px（普通 32×32px）
- 对比度 ≥7:1（AAA）
- 行高 1.8、动画速度 0.7x
- 关键操作二次确认

#### G. 响应式适配（§13.8）
- 桌面 ≥1280px：三栏完整显示
- 小屏桌面 1024-1279px：右侧面板默认收起
- 平板 768-1023px：侧边栏折叠为图标栏，右侧面板覆盖式
- 手机 <768px：侧边栏抽屉式，右侧面板全屏覆盖

#### H. Mock 数据集中放置于 `src/data/mock/`
- `mock/sessions.ts` — mock 会话列表
- `mock/messages.ts` — mock 消息流（含思维链/工具调用/搜索结果样例）
- `mock/prescription.ts` — mock 五大处方报告 JSON（药物/运动/营养/心理/康复五类完整）
- `mock/cga.ts` — mock CGA 评估结果
- `mock/patients.ts` — mock 患者列表（医生端）
- `mock/search-results.ts` — mock 联网搜索结果
- `mock/skills.ts` — mock 预置技能列表
- `mock/drug-review.ts` — mock 用药审查结果

### 2.2 不做什么

- 不调用任何真实 LLM/ASR/TTS/搜索/文档解析 API（所有交互用 mock 数据 + setTimeout 模拟延迟）
- 不实现真实业务逻辑（处方生成、CGA 计分、用药审查规则引擎、技能加载影响对话、JSON Schema 校验）
- 不做后端代码
- 不做登录认证（访客模式）
- 不做 DDI/Beers/量表真实数据导入（用 mock 占位）
- 不做真实导出（PDF/DOCX 生成，仅 mock 下载提示）
- 不做真实音频播放（预录制音频/TTS 均用 mock 播放动画）
