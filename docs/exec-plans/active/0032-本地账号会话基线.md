# 0032 — 本地账号会话基线

> 创建：2026-07-17 | 优先级：P0 | 状态：进行中 | 前置：0014 基础设施

## 目标

落实权威设计中的账号、密码、角色注册与登录。第一阶段只建立本地患者/医生
账号和可撤销会话；不把注册时选择的医生角色误作资质、审批或跨患者权限。

## 已交付

- 加密用户名、HMAC 检索指纹、版本化 scrypt 密码 verifier 和仅保存 HMAC
  指纹的 refresh-session 数据模型与 Alembic `f72c814f2030`。
- `POST /auth/register` 与 `POST /auth/login`：仅接受受限账号名、强密码和
  `patient|doctor`；重复账号与错误密码均不泄露原始凭据。
- `POST /auth/refresh` 对 refresh token 单次轮换，`POST /auth/logout` 幂等撤销
  单个 refresh session。Compose 实测轮换后的旧 token 和登出后的 token 都返回
  `401 ACCOUNT_REFRESH_INVALID`。
- `POST /auth/password` 需要已验证账号 access token 与当前密码；成功后升级
  password verifier 并撤销该账号所有 refresh session。Compose 实测改密 204，
  改密前 refresh token 随即返回 401。
- Next.js 新增同源 `/api/account/[action]` BFF：access/refresh 均仅写入
  HttpOnly Cookie，refresh、登出、改密和停用要求双提交 CSRF 值；浏览器 JavaScript
  不接触 refresh token。会话响应严格校验 backend 返回的 `token_type: bearer`，避免合法
  注册/登录响应被 BFF 误拒绝为 502；`npm run test:account` 为 `2 passed`。`npm run lint`
  与 production `npm run build` 通过。
- 侧边栏新增登录/注册入口及键盘可提交的无障碍账户对话框；注册时可选择患者或
  医生身份，已登录账户不能在客户端切换角色。`GET /auth/session` 只向 BFF 返回
  opaque actor 与服务端验证角色，用于安全恢复已登录页面状态。
- 通用 `/api/gerclaw/*` BFF 代理优先使用该账户 access Cookie；账户请求不会再被
  访客身份 cookie 重新绑定或在 401 时错误降级为访客令牌。
- access JWT 由服务端携带已持久化角色。医生账号只保有本人范围的基础 scope，
  没有跨患者、临床规则或审批 scope。
- 已支持本人密码确认的账户停用：服务端原子撤销全部 refresh session、禁用账号，并在 Redis 写入固定 15 分钟的 access-token deny key；认证缺少该校验器时对账号 token fail closed。停用不删除医疗数据，也不表示已完成数据删除或保留策略。
- 已登录账户菜单提供“停用账户”入口；对话框明确说明停用会立即退出且不删除医疗数据，要求当前密码和独立确认勾选后才启用危险操作。患者端老年模式下输入和操作按钮均保持至少 48px。真实浏览器完成注册、打开停用对话框、输入密码和勾选的回归，刷新后控制台为 0 个业务错误；测试账户和审计事件已精确删除。
- Compose 实测医生账号注册、登录、错误密码 401；数据库检查用户名、密码和
  refresh token 均未明文出现。迁移升级和 `alembic check` 通过。
- `f82c814f2031` 新增不可变 `identity_security_events`：注册、登录、刷新、
  登出、改密（包括拒绝/重放）与业务写入同事务保存，仅含 HMAC subject
  fingerprint、opaque actor、角色、动作和结果。登出也已限流；Compose 演练
  覆盖成功/拒绝路径，并确认审计列无用户名、密码或 refresh token 明文。

## 未完成与门禁

### 2026-07-18 入口与账户管理更新

- 产品入口改为强制显示登录页；用户可选择“以游客身份进入患者端”。访客只获得患者服务的匿名、短期身份，浏览器不保存其访客 ID，重新进入不会恢复本地对话；后端仍保留受控 Trace/反馈/bad-case 所需的匿名运行记录。
- 新增仅可由显式 CLI 创建的 `admin` 账户、管理员账户目录、患者/医生角色调整与账号启停；管理变更会撤销被变更账号的刷新会话并立即 deny 其 access JWT。管理员在患者/医生工作区之间切换时，JWT 保留不可伪造的 `account_role=admin`，不会取得其他账号的数据。
- 管理员进入患者端或医生端后，账户菜单会保留“管理控制台”和另一工作区的切换入口；“管理控制台”仅改变前端呈现，所有管理 BFF 调用仍由服务端签名的 `account_role=admin` 校验。2026-07-18 Compose 浏览器已实测“控制台 → 患者端 → 控制台 → 医生端 → 控制台”往返；退出后回到登录入口，临时验收账户及其安全审计记录已精确清理。
- `gerclaw-bootstrap-admin --username <name>` 是首个管理员的唯一创建入口；公共注册 API 仍只允许患者或医生角色，不能注册管理员。

- 当前 BFF 已持有 refresh token，浏览器 JavaScript 不得直接持久化它；账户停用
  不是删除医疗数据，也未提供账户恢复、数据导出或数据删除能力。
- 医生资质、患者授权资源范围/期限/撤回、访客迁移和临床权限继续由 0025 的
  合规决策约束；在其完成前一律 fail closed。
