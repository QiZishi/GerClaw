# 0030 — 安全情感陪伴后端基线

> 创建：2026-07-17 | 优先级：P1 | 状态：进行中 | 前置：0016 Agent Harness、0023 红旗短路

## 目标

将 PRD `CLN-08` 的情感陪伴先落实为真实、独立且受控的 Chat workflow。它只提供非欺骗性的支持性对话；不把陪伴关系伪装为人类关系或医疗服务。

## 阶段 A 范围

- API 明确声明 `workflow=companion`，并拒绝 Skill、上传资料和工具上下文混入该模式。
- 复用 Chat 的 tenant/actor/session/lease/Trace/SSE 原子终态与高风险症状确定性短路。
- 陪伴模式不读取、检索或写入长期健康 Memory，不调用 RAG、联网搜索或 Skill；只使用同一受保护会话的短期聊天记录。
- 专用、简短 system prompt 要求明确 AI 身份，不作人类/亲属/医生承诺，不诱导排他或依赖，不替代危机或医疗支持。

## 非目标与门禁

- 本阶段不新增诊断、心理治疗、危机人工通知或医疗建议；急性自伤/红旗仍由既有确定性安全短路处理。
- 不把“无长期 Memory”误称为已实现可配置的陪伴记忆开关；用户可控偏好、前端入口、人工升级与医生授权需后续单独设计。
- 不使用模型从陪伴文本提取或固化健康事实。

## 验收

- [x] 严格 Chat 合同只允许该模式的无工具、无附件请求。
- [x] Harness 不注册 Memory/RAG/Search/Skill，医疗文本也不绕过陪伴边界进入证据/工具链；红旗短路仍在模型前执行。
- [x] 测试覆盖合同拒绝、无工具上下文、无长期 Memory、提示词安全约束和红旗回归。
- [x] ruff、mypy、定向 Chat/Harness 测试和 Compose API 健康检查通过。

## 实施记录

- 2026-07-17：新增 `modules/companion` policy contract，并将 `workflow=companion` 接入 Chat Harness。该模式使用独立简短 system prompt，明确 AI 身份、禁止依赖诱导/人类冒充/主动联系承诺；请求合同拒绝 Skill 与上传文件。Chat Service 改用已加密的同会话历史，跳过 Memory 的短期压缩、画像读取、提取写入、向量提交和 `memory.update` Trace，Harness 也不创建任何工具或 middleware。定向 Harness/Chat 回归 `60 passed, 8 skipped`；文档门禁通过，Compose 重建后 API `/health/live` 返回 `alive`。未以真实用户文本调用模型，不能将该启动验证表述为模型行为的生产证据。
