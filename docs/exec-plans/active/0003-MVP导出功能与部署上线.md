# 0003-MVP导出功能与部署上线 — 执行计划

> 任务编号：0003 | 创建日期：2026-07-06 | 优先级：P0 | 阶段：三（MVP可交付闭环+部署）

## 1. 任务目标

在0002核心功能真实API接入的基础上，补齐MVP可交付的最后几项关键能力：Markdown格式结果导出、图片多模态理解（上传图片可被AI理解）、适老化回归验证，最终将MVP部署到IGA Pages上线，实现GerClaw从本地开发到线上可访问的完整闭环。

## 2. 范围

### 2.1 做什么

#### A. Markdown导出功能
- 创建`src/lib/export/`目录，实现统一导出工具
- 实现Markdown导出策略：拼接模板+Markdown内容→Blob下载
- 统一导出模板：标题+日期+GerClaw标识+医疗免责声明
- 五大处方报告导出（右侧面板内容）
- CGA评估报告导出（右侧面板内容）
- 单条消息导出（AI回复的复制/导出）
- 整个会话导出（Markdown格式）
- ExportButton组件接入真实导出逻辑，替换mock toast

#### B. 图片多模态理解
- 扩展LLMMessage类型支持content数组（text/image_url）
- 修改文件上传：图片文件上传后读取为base64 data URL
- 修改LLM API路由：支持转发multimodal消息到模型
- 图片附加到用户消息时，模型可以看到图片内容
- 图片类型校验（PNG/JPG/WebP）、大小限制
- 非图片文件（PDF/DOCX/MD/TXT）保留现有mock行为，标注待MinerU接入

#### C. 全量回归与适老化验证
- npm run lint + npm run build确保0错误
- 适老化回归检查：老年模式字体≥18px/按钮≥48px/高对比度/二次确认
- 验证免责声明在所有场景下可见（消息底部+输入框底部）
- 验证Stethoscope图标一致性
- 验证录音UI/加载动画/取消录音功能

#### D. 部署上线
- 使用IGA Pages部署技能部署到线上
- 配置环境变量（API Keys通过平台环境变量注入）
- 验证线上功能可访问

### 2.2 不做什么（二阶段及以后）
- PDF/DOCX导出（需要引入jspdf/html2canvas/docx等库，Markdown导出足够MVP）
- MinerU真实文档解析（PDF/DOCX解析）
- 后端架构迁移（FastAPI/AgentScope）
- Playwright E2E自动化测试
- AnySearch联网搜索接入
