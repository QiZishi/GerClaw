# 0023 — CGA 后端确定性评估闭环

> 创建：2026-07-16 | 优先级：P0 | 状态：进行中 | 前置：0022 前端安全降级

## 1. 目标

把 CGA 从 `apps/mvp` 中的本地演示状态迁移为 FastAPI 可持久化、可恢复、可审计的临床筛查 workflow。计分、分级和风险标记必须由确定性规则完成；LLM 仅可在后续阶段承担对话引导，不能改变题号、答案或分数。

## 2. 分阶段范围

### 阶段 A（本变更集）

目标是在后续变更集中交付 PHQ-9 的最小真实纵切面：版本化量表定义、严格答案校验、幂等状态机、确定性计分/报告、PHQ-9 第 9 题即时高风险提示、租户/本人隔离和加密持久化 API。本次已完成其确定性评分核心；前端在 API 契约稳定后再由 `apps/mvp/services/` 接入。

### 阶段 B（后续同一计划）

在同一引擎契约上补齐 SAS、PSQI、Mini-Cog、MMSE；接入安全的对话引导、疲劳暂停、医生授权查看、报告导出和历史对比。未实现的量表不得出现在 API 的可用列表或前端“已可评估”入口。

## 3. 设计与安全约束

- 所有量表定义由后端版本化、只读地提供；客户端不得提交题目文本、分值或总分。
- 所有读写按 JWT `tenant_id + actor_id` 绑定。评估答案、报告和风险原因使用 `EncryptedJSON`，日志/Trace 不写入题干或答案。
- API 只输出“筛查结果”，永不诊断。报告始终附带“不能替代临床诊断”的免责声明。
- PHQ-9 第 9 题非零须在保存答案的响应中立即返回“需要立即安全评估”提醒；总分 ≥20 是独立的“尽快临床随访”状态，二者不得混淆。前者明确建议立即联系家人、医生或当地紧急医疗服务。
- 从服务端当前题目提交答案；拒绝跳题、未知题目、越界分数、已完成后继续答题和陈旧版本写入。重复提交同一答案为幂等，修改答案则安全重算。
- 量表与 API 输入输出均使用 Pydantic `extra="forbid"`；数据库迁移可回滚，所有 PHI 列加密。

## 4. 验收标准

- [ ] `GET /api/v1/cga/scales` 仅返回已支持的 PHQ-9 公开定义与版本。
- [ ] 可开始、读取、依序回答、断点恢复并完成一次 PHQ-9；所有状态/得分由服务端产生。
- [ ] 计分覆盖 0/4/5/10/15/20/27 边界；第 9 题风险和总分高风险均即时且可测试。
- [ ] 答案/报告数据库原始值为加密密文；另一主体和另一租户无法读取或写入评估。
- [ ] API、模块、repository、迁移和文档都有必要测试；执行对应 pytest、ruff、mypy、Alembic upgrade/check，并由独立审阅复测。
- [ ] 前端接入前保持 0022 的明确降级，不能把阶段 A 误称为已完成全部五项 CGA。

## 5. 非目标

- 阶段 A 不生成 LLM 医学解读、不调用搜索、RAG 或 TTS，不做诊断或处方建议。
- 阶段 A 不开放医生跨患者访问；该能力依赖 0025 的真实账号/RBAC/患者授权。
- 不把本地 `localStorage` 或前端静态分数作为服务端事实来源。
