# 0023 — CGA 后端确定性评估闭环

> 创建：2026-07-16 | 优先级：P0 | 状态：进行中 | 前置：0022 前端安全降级

## 1. 目标

把 CGA 从 `apps/mvp` 中的本地演示状态迁移为 FastAPI 可持久化、可恢复、可审计的临床筛查 workflow。计分、分级和风险标记必须由确定性规则完成；LLM 仅可在后续阶段承担对话引导，不能改变题号、答案或分数。

## 2. 分阶段范围

### 阶段 A（本变更集）

PHQ-9 最小真实纵切面已交付：版本化量表定义、严格答案校验、幂等状态机、确定性计分/报告、PHQ-9 第 9 题即时高风险提示、租户/本人隔离和加密持久化 API。`apps/mvp/services/gerclaw/cga.ts` 已经通过 Zod 契约和受限 BFF 接入服务端事实源。

### 阶段 B（后续同一计划）

在同一引擎契约上已补齐 SAS、PSQI、安全的逐题引导、疲劳暂停、患者侧 Markdown/PDF/Word 导出和本人历史报告。Mini-Cog、MMSE 与医生授权查看仍未完成；未实现的量表不得出现在 API 的可用列表或前端“已可评估”入口。

### 当前已交付边界

- 患者端仅从 `GET /cga/scales` 展示当前服务端实际支持的 PHQ-9、SAS 与 PSQI；开始、答题、恢复、完成、报告和本人历史记录均经 Zod 校验后走 Next BFF/API。
- 开始、答题和完成操作与 CGA Runtime Trace 在同一数据库事务中提交；Trace 仅含量表、定义版本、操作、答题数量、结果状态和不可逆请求指纹，不含题号、分数、风险标记、补充说明、评估 ID 或原始请求体。
- 本地静态量表、前端本地评分、旧 GAD-7 定义和医生端空数据占位组件已移除，防止错误量表或模拟结果被重新接入生产界面。
- apps/mvp/public/audio/cga/ 存放由当前公共量表定义生成的版本绑定 WAV 资源。清单逐项记录量表、定义版本、题目、选项映射、文件字节数和 SHA-256；客户端只有在服务端返回相同量表/版本/题目时才会播放对应静态题目朗读。
- 认知量表的人工确认、医生跨主体查看、患者授权和完整历史比较仍为未交付能力，不得由界面暗示可用。

## 3. 设计与安全约束

- 所有量表定义由后端版本化、只读地提供；客户端不得提交题目文本、分值或总分。
- 所有读写按 JWT `tenant_id + actor_id` 绑定。评估答案、报告和风险原因使用 `EncryptedJSON`，日志/Trace 不写入题干或答案。
- API 只输出“筛查结果”，永不诊断。报告始终附带“不能替代临床诊断”的免责声明。
- PHQ-9 第 9 题非零须在保存答案的响应中立即返回“需要立即安全评估”提醒；总分 ≥20 是独立的“尽快临床随访”状态，二者不得混淆。前者明确建议立即联系家人、医生或当地紧急医疗服务。
- 从服务端当前题目提交答案；拒绝跳题、未知题目、越界分数、已完成后继续答题和陈旧版本写入。重复提交同一答案为幂等，修改答案则安全重算。
- 量表与 API 输入输出均使用 Pydantic `extra="forbid"`；数据库迁移可回滚，所有 PHI 列加密。

- [x] API、模块、repository、迁移和文档都有必要测试；执行对应 pytest、ruff、mypy、Alembic upgrade/check，并由独立审阅复测。

## 5. 非目标

- 阶段 A 不生成 LLM 医学解读、不调用搜索或 RAG，不做诊断或处方建议。题目朗读优先使用版本绑定的静态 WAV；仅当本地资源缺失或加载失败时，才使用带暂停、继续、停止和进度控制的实时 TTS 无障碍兜底。
- 阶段 A 不开放医生跨患者访问；该能力依赖 0025 的真实账号/RBAC/患者授权。
- 不把本地 `localStorage` 或前端静态分数作为服务端事实来源。
