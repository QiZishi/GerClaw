# 0031 — 慢病管理加密测量账本

> 创建：2026-07-17 | 优先级：P1 | 状态：进行中 | 前置：0014 数据基础、0023 风险告警

## 目标

将慢病管理从健康画像中独立为真实的、用户本人范围的病情与测量数据链路，同时严格区分自述记录、算术趋势与临床结论。

## 阶段 A 已交付

- `chronic_care` 模块以 `chronic-care` API 创建和列出自述病情，添加、列出测量并读取同标签同单位的最近两项算术方向。
- 标签、数值、单位与测量时间加密；所有操作按 `tenant_id + actor_id` 绑定，测量记录只追加不修改。
- `rising`、`falling`、`unchanged`、`insufficient_data` 只描述数值比较，不表示异常、风险、治疗目标或依从性。
- 创建操作与 PHI-free Trace 同一数据库事务提交；Trace 仅记录固定 feature/operation/version/count/outcome 和请求 HMAC，不记录病情、数值、单位、时间或 UUID。

## 非目标与门禁

- 自述病情不等于医生确认；无目标、阈值、预警、用药、提醒、依从性、处方、临床建议或医生队列。
- 任何异常判断、风险告警来源、个体化目标或通知必须先提供版本化、可追溯、经医学审核的规则和相应授权/HITL workflow。
- 患者/医生账号、患者授权、前端入口和跨时间可视化仍未实现。
