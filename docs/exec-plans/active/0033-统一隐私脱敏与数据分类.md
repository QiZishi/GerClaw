# 0033 — 统一隐私脱敏与数据分类

> 创建：2026-07-17 | 优先级：P0 | 状态：阶段 A 进行中 | 前置：0014 基础设施、0018 外部搜索

## 目标

把分散的脱敏逻辑逐步收敛为可版本化、可测试、按外发目的执行的隐私边界；绝不把“文本替换”误表述为完整的数据生命周期、用户透明度或受控再识别能力。

## 阶段 A：外部文本 egress

- `modules/privacy_redaction/` 提供 `1.1.0` 的严格 Pydantic 契约和
  `redact_external_search_query()` / `redact_external_tts_text()`。返回可外发
  文本与 PHI-free 分类计数，绝不返回匹配原文、位置或可逆映射。
- `ProductionSearchModule` 在所有 AnySearch/Tavily 调用前经过该策略；旧的
  `sanitize_search_query()` 仅保留兼容入口并委派给统一模块。
- FastAPI `/search/query` 的每一次 AnySearch/Tavily 尝试（含 retry/fallback）会在
  网络调用前单独持久化 `prepared` egress decision，调用后写为 `succeeded` 或
  `failed`；记录同样只含用途、逻辑处理方、策略版本和 `text` 类别计数，审计失败
  会阻断该次调用。
- FastAPI MiMo TTS 在任何 provider 请求前对待朗读正文和 style 应用
  `external_tts` 策略；其 provider-payload 回归测试断言姓名、手机号和凭证
  原文均无法跨出服务端。
- FastAPI TTS 在网络调用前同一请求事务持久化 `provider_egress_events` 的
  `prepared` PHI-free 决策；首个有效 PCM16 包后标为 `succeeded`，Provider
  故障标为 `failed`。记录仅含 purpose、逻辑处理方、策略版本和按 `text/style`
  划分的类别计数；审计存储失败会阻断外发。
- FastAPI ASR 在本地格式/大小校验后、Provider 调用前以固定
  `external_asr_audio` / `audio-egress-v1` 写入 `prepared`，成功取得受限转写
  后才写为 `succeeded`，Provider 故障写为 `failed`。它不保存音频、转写或
  文件属性，findings 始终为空；空 findings 不代表音频已去标识化或取得同意。
- Next.js MinerU BFF 在真实文件解析的 submit 请求中使用每次生成的匿名
  UUID 文件名，仅保留原始文件的受支持扩展名；原始文件名不会发送给 MinerU，
  也不会写入该 BFF 的成功/失败日志。BFF 以同一 HttpOnly principal 在调用前写入
  owner-bound `external_document_parse` / `document-egress-v1` 的 `prepared`
  decision，并在 provider 成功或失败后写入终态；该记录不含文件名、正文、大小、
  页数或 findings。文件二进制内容仍是文档解析的必要输入；此项最小化和审计不等同于
  对文档正文脱敏、用户同意或完整 egress 台账。
- 当前策略识别带标签的中英文姓名、手机号、身份证、邮箱、常见凭证赋值和控制
  字符；空或超过 4,000 字符的外发 query fail closed。
- `modules/evals` 已加入 4 条版本绑定的 `privacy-redaction-case-v1` 合成
  canary，覆盖 search/TTS、带标签中英文姓名、联系方式、证件、凭证和控制字符。
  评测不调用数据库、模型或 Provider，CLI 仅输出用例 ID、用途、策略版本与
  PHI-free 类别计数，不能回显输入或预期脱敏文本。
- 已运行 `40 passed` 的隐私、Search、Voice 定向 canary，以及 Ruff、Mypy；
  重建 Compose API 后容器内 canary 输出 `1.1.0`、去标识文本与仅类别计数。
- `FailoverChatModel` 在每次 AgentScope Provider 调用前，创建不改变本地 Agent
  状态的消息副本，并对每个非空字符串字段应用独立的
  `external_model_prompt` `1.0.0` 投影；Markdown 和工具 block 结构保留，超限值
  fail closed。该投影尚未向 `provider_egress_events` 持久化 decision，也不代表
  用户同意或完整模型 Provider 台账。

## 不在本阶段范围

- 不持久化原文或可逆映射，不新增未经授权的再识别能力。
- MinerU 文档正文、遗留 Next.js TTS BFF、模型 prompt 的 egress audit/同意、导出、Bad Case、用户数据用途页面、
  保留/删除/备份清除、网页提取、AgentScope 内部搜索与其他 Provider 的处理台账仍未接入本模块；MinerU 的文件名最小化和 outcome 审计不能声称已统一覆盖。
- 字段级分类注册表需要为每个 schema 定义存储、日志、Trace、vector、export 与
  provider 处理规则；当前 Runtime `DataClass` 不能替代这份注册表。

## 下一步

3. 在真实账号/RBAC 后实现用户可见的数据用途、导出、删除和受控再识别审计。
