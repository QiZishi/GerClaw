## 评分规则

| 分数 | 含义 |
|------|------|
| 1 | 初始：未开始或只有占位符/框架代码 |
| 2 | 基础：核心功能可用，缺乏测试/文档/错误处理 |
| 3 | 可用：有基本测试覆盖、有文档、有基本错误处理 |
| 4 | 良好：测试覆盖主路径+失败路径、文档完善、边缘情况处理、有可观测性 |
| 5 | 优秀：全面测试覆盖（>80%）、完整文档、所有边缘情况处理、完善可观测性、性能优化 |

**关键规则：没有测试/eval/截图/代码作为证据，分数不能往上提。**
