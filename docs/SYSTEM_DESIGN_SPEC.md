## 0. 设计哲学

### 0.1 产品气质

- **专业可信**：医疗AI产品，配色严谨、信息层次清晰、交互克制
- **简洁现代**：对齐 Trae Work 视觉语言——内嵌式输入、折叠式思维链/工具卡、低干扰UI
- **适老优先（患者端）**：老年模式在患者端默认开启，大字号、大点击区、高对比度、语音入口突出
- **双端差异化**：医生端冷蓝专业感（`#2563EB`），患者端暖蓝亲和感（`#0EA5E9`）

### 0.2 核心设计原则（来自 DESIGN.md）

1. **以患者为中心**：界面友好、字体大、操作简单，语音交互优先
2. **医生效率优先**：信息密度适中，关键数据突出，操作路径短
3. **医疗严谨性**：风险标识醒目、循证来源可查、免责声明必现
4. **无障碍设计**：WCAG 2.1 AA级（老年模式AAA级）、键盘导航、屏幕阅读器支持
5. **语义化HTML**：header/main/nav/aside/section/article/button 等语义标签优先使用

### 1.6 动画与过渡

| 用途 | 时长 | 缓动函数 | 说明 |
|------|------|---------|------|
| 颜色/背景变化（hover/focus） | 150ms | `ease-out` | 按钮、输入框、列表项的状态切换 |
| 按钮按下反馈 | 100ms | `ease-out` | scale(0.98) 或亮度变化 |
| 面板展开/收起（侧边栏折叠/右侧面板） | 200ms | `ease-in-out` | 宽度过渡，布局不跳动 |
| 模态框出现/消失 | 300ms | `ease-in-out` | 透明度+scale(0.95→1) |
| 消息滑入/工具卡片出现 | 200ms | `ease-out` | translateY(8px→0) + opacity(0→1) |
| 旋转加载指示器 | 1s 无限循环 | `linear` | 工具调用中状态 |
| 脉冲动画（思考中/录音中/光标闪烁） | 2s 无限循环 | `ease-in-out` | opacity(0.4→1→0.4) |

**全局规则**：
- 所有可交互元素必须有 hover/active 状态过渡
- 检测 `prefers-reduced-motion: reduce` 时，禁用所有 transition/animation，状态瞬间切换
- 老年模式下所有动画时长 ×1.4（放慢），禁用脉冲和快速闪烁

## 4. 基础组件详细规格

### 4.1 按钮（Button）

#### 主按钮（Primary）
- 背景：`var(--color-brand)` 填充；hover：`var(--color-brand-hover)`；active：`var(--color-brand-hover)` + 轻微亮度降低；disabled：`--color-brand` + opacity 0.4，`cursor: not-allowed`
- 文字：白色，`--text-body`(14px/500)；老年模式18px/600
- 圆角：`--radius-md`(8px)；右侧面板大操作按钮可用 `--radius-lg`(12px)
- 高度：36px（标准）/ 48px（老年）/ 40px（手机默认）
- 水平padding：16px 20px
- 阴影：无默认阴影；hover时可加 `--shadow-sm`
- 患者端使用 `--color-brand-patient` 作为主色

#### 次要按钮（Secondary）
- 背景：透明；1px `var(--color-border-default)`边框
- 文字：`var(--color-text-primary)`，14px/500
- 圆角：`--radius-md`(8px)
- 高度：36px；水平padding：16px 20px
- hover：`var(--color-bg-hover)` 背景
- 老年模式：48px高，18px字

#### 文字按钮（Ghost/Tertiary）
- 背景：透明；无边框
- 文字：`var(--color-text-secondary)`，14px/400
- 高度：28-32px；水平padding：8px 12px
- 圆角：`--radius-sm`(4px)
- hover：`var(--color-bg-hover)` 背景，文字变 `--color-text-primary`
- 用于"取消"、"查看更多"、"上一题"等次要操作

#### 图标按钮（Icon Button）
- 正方形：默认 32×32px（桌面折叠栏28×28px）；老年模式 44×44px；手机端语音按钮 48-56px
- 背景：透明；hover：`var(--color-bg-hover)`
- 圆角：`--radius-sm`(4px)（工具按钮）或 `--radius-full`（圆形按钮，如语音、新建）
- 图标尺寸：16-20px
- 颜色：`--color-text-secondary`（默认）；品牌色（激活/品牌场景）；白色（品牌色背景上）
- Tooltip：折叠栏中图标按钮必须显示Tooltip（延迟300ms，内容在按钮下方或右侧）

#### 危险按钮（Danger）
- 主按钮样式，背景色为 `var(--color-danger)`（红）
- 文字白色
- hover为更深的红色
- 仅用于"删除"、"清空"等破坏性操作

### 4.2 输入框（Input/Textarea）

#### 普通表单输入框（设置/新建/搜索等场景）
- 高度：36px（标准单行）/ 48px（老年）
- 背景：`var(--color-bg-input)`
- 1px `var(--color-border-default)`边框
- 圆角：`--radius-md`(8px)
- 内边距：0 12px
- 字号：`--text-body`(14px)
- 聚焦：边框变为 `var(--color-brand)`（无外发光）

#### 多行文本域（Textarea）
- 继承输入框样式
- 最小高度：80px
- padding：10px 12px
- resize: vertical（或自动增高）

### 4.3 卡片（Card）

- 背景：`var(--color-bg-elevated)`（白色/深灰）
- 1px `var(--color-border-subtle)`边框
- 圆角：`--radius-md`(8px)（普通卡片）/ `--radius-lg`(12px)（大卡片/面板容器）
- 内边距：`--space-4`(16px)
- 阴影：默认无阴影或 `--shadow-sm`
- hover态：边框变 `var(--color-border-default)`，阴影变 `--shadow-md`，`transform: translateY(-1px)`（200ms ease-out）

- 间距：前后各8-12px

## 7. 主题切换实现

### 7.1 切换行为
- 主题切换按钮位于侧边栏底部（见§3.2.6）

  - 覆盖字号变量为老年字号
  - 覆盖按钮高度/间距变量
  - localStorage 存储 `elderlyMode: boolean`

## 9. 适老化专项强条（患者端必须）

以下规则在患者端老年模式下（`html.elderly-mode`）必须严格遵守，前端实现时作为硬性检查项：

### 9.1 视觉
1. 正文≥18px，标题≥24px，按钮文字≥18px/600字重
2. 所有可点击元素≥48×48px，元素之间间距≥12px
3. 颜色对比度≥7:1（WCAG AAA 级）
4. 纯图标按钮必须配文字标签（禁止仅图标无文字），仅语音大按钮等高度视觉化按钮可例外但需有清晰视觉反馈
5. 使用患者端暖蓝色 `#0369A1`（深天蓝，保证对比度），而非冷蓝
6. 强制浅色高对比度背景（白底深字），不使用深色主题
7. 重要信息禁用灰色占位文字（灰色老人可能看不清，误以为不可点击）
8. 不使用斜体（斜体在大屏/老花镜下难辨识）
9. 重要文字加粗（600-700字重）

### 9.2 交互
1. 语音麦克风按钮是页面最醒目的元素（≥64px圆形，暖色渐变，脉冲提示）
2. AI回复完成后**自动TTS朗读**（可通过暂停/停止按钮中断）
3. CGA评估每题有🔊朗读按钮，进入新题自动朗读（可关闭）
4. 关键操作（删除对话、清除数据、提交处方、退出）必须**二次确认对话框**（按钮56px高，文字明确如"确定删除这段对话吗？"）
5. 错误提示使用大白话："出了点小问题，点这里再试一次"，不用技术术语
6. 动画速度放慢至0.7倍，禁用闪烁和快速运动（如快速滑入、剧烈脉冲突变）
7. 点击区域之间留白充足，相邻按钮间距≥12px，避免误触
8. 输入框高度≥60px，点击区域大
9. 支持大字体系统设置（响应系统字体缩放125%/150%）

### 9.3 内容
1. 医学术语配大白话解释
2. 每次只问/只展示一个主要问题，避免一屏多任务
3. 说明文字简短明确，段落不超过3行
4. 重要操作结果用Toast明确反馈（如"已发送"、"已保存"大字反馈）

## 11. 技术实现注意事项

### 11.1 技术栈（来自 gerclaw设计要求.md + harness-dev 规范）
- 框架：Next.js 15 (App Router) + React 18 + TypeScript
- 样式：Tailwind CSS 4（使用 `@theme` 定义上述所有Token，直接引用CSS变量）
- 组件基础：shadcn/ui（基于 Radix UI，按本设计稿定制圆角/颜色/阴影）
- AI对话：Vercel AI SDK（useChat hook，SSE流式）
- 状态管理：Zustand（UI状态、会话状态）+ localStorage持久化
- 图标：lucide-react
- Markdown渲染：react-markdown + remark-gfm + rehype-highlight（代码高亮）

### 11.3 状态持久化（localStorage键名）
- `gerclaw:theme`: `'light' | 'dark' | 'system'`
- `gerclaw:elderly-mode`: `boolean`（患者端）
- `gerclaw:sidebar-collapsed`: `boolean`
- `gerclaw:panel-width`: `number`（像素，320-500）
- `gerclaw:role`: `'doctor' | 'patient'`
- `gerclaw:tts-autoplay`: `boolean`
- `gerclaw:conversation-history`: 会话列表（注意加密/脱敏）

### 11.4 严格禁止项（实现时硬性检查）
1. ❌ 不得参考 `gerclaw-main-origin/` 和 `gerclaw-design-origin/` 目录任何内容
2. ❌ 不得使用纯黑色 `#000000` 作为深色模式主背景（深色背景使用 `#0F172A`/`#1E293B`）
3. ❌ 不得使用外发光 `box-shadow: 0 0 Xpx brandColor` 效果（避免AI产品常见蓝光辉），聚焦状态用边框色变化
4. ❌ 老年模式下不得使用小于16px字号或小于44px可点击元素
5. ❌ 患者端老年模式下不得启用深色主题（强制浅色高对比度）
6. ❌ 医疗输出（处方/评估/审查）不得遗漏免责声明
7. ❌ 工具调用卡片不得缺少三态之一（运行中/完成/失败）的状态标识
8. ❌ 所有可交互元素不得缺少 hover/active/focus/disabled 状态样式
9. ❌ 不得使用紫色、粉色、荧光色等非医疗品牌规定的颜色
10. ❌ 输入框不得使用粗边框+外发光样式，必须是内嵌式Trae风格（与背景融合，聚焦时边框变品牌色）
