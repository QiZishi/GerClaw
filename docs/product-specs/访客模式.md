## 1. 模块概述

零门槛访客使用模式，无需注册/登录即可直接使用GerClaw全部功能，所有会话记录、自定义技能、用户设置等数据仅存储在浏览器localStorage中，不向服务器上传任何用户身份信息；首次访问显示欢迎页和医疗免责声明确认，提供存储容量管理和数据清除功能，明确告知用户数据仅存本地的隐私策略。

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户首次访问GerClaw网站
  ↓
检测localStorage中 disclaimer_accepted 标记
  ↓
不存在 → 显示全屏欢迎页：
  ├─ GerClaw Logo + 产品名称
  ├─ 简短介绍："AI老年专科医生助手"
  ├─ 完整医疗免责声明文本框（可滚动）
  ├─ 复选框：☐ 我已阅读并同意免责声明
  └─ "开始使用"按钮（默认禁用）
  ↓
用户勾选复选框 → 按钮变为可用
  ↓
用户点击"开始使用"
  ↓
localStorage写入：disclaimer_accepted: true, first_visit_at: 时间戳
  ↓
进入主界面，显示新会话欢迎页
  ↓
用户使用过程中：
  ├─ 发送消息/创建会话/添加技能 → 自动保存到localStorage
  └─ 实时监控存储容量
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| localStorage被禁用（浏览器隐私模式） | 降级为内存存储（Map/Object in memory），所有功能正常使用，刷新页面数据丢失 | 页面顶部显示黄色固定警告条："隐私模式下数据仅临时保存，刷新后将丢失"，警告条可关闭 |
| localStorage写入失败（QuotaExceededError） | 触发自动清理：删除最早的非固定会话直到可写入；清理后仍失败则降级为内存存储 | 自动清理后显示toast："存储空间不足，已清理最早的对话记录，重要内容请及时导出"；降级为内存时显示红色警告 |
| 用户清除浏览器数据/Cookie | localStorage清空，下次访问视为首次访问 | 无感知，下次访问直接显示欢迎页（符合用户预期） |
| 存储数据版本不兼容（版本升级） | 尝试数据迁移；迁移失败则重置为空数据 | Toast提示："历史数据版本已更新，已重置为初始状态" |
| localStorage数据损坏（JSON解析失败） | 捕获异常，重置为空数据 | Toast提示："本地数据损坏，已重置，如重要内容请联系支持"（MVP阶段无后端支持，仅提示） |
| 用户点击清除所有数据但输入错误 | 不执行清除，提示重新输入 | 对话框内红色文字提示："请输入'确认清除'以继续" |

## 5. 界面/API规格

### 5.1 界面

- **页面/组件名**：欢迎页、设置面板、存储警告条
- **入口**：
  - 欢迎页：首次访问自动显示
  - 设置面板：左侧边栏底部齿轮按钮点击弹出
  - 存储警告：顶部动态显示
- **关键元素**：
  - **欢迎页**：全屏遮罩（白色/浅灰背景），居中卡片（最大宽度600px）包含：
    - 顶部：GerClaw Logo + "欢迎使用GerClaw"标题
    - 中部：产品简介段落、可滚动免责声明文本框（灰色边框，最多300px高，带滚动条）、隐私提示小字
    - 底部：复选框+同意文字、蓝色"开始使用"大按钮（≥48px高）
  - **设置面板**：侧边弹出面板或模态框，包含：
    - 标题"设置"
    - "外观"分组：深色/浅色/跟随系统 三个单选按钮
    - "数据与隐私"分组：隐私说明段落、红色"清除所有本地数据"按钮
    - "关于"分组：版本号、链接
  - **存储警告条**：页面顶部全宽条，黄色/红色背景，文字说明+关闭按钮（×）
  - **清除数据确认对话框**：模态框，警告文字说明"此操作不可恢复，将删除所有对话记录、自定义技能和设置"，输入框提示"请输入'确认清除'继续"，"取消"和"确认清除（红色）"按钮
- **状态要求**：
  - **主流程**：欢迎页→勾选→点击进入→正常使用→自动保存
  - **加载中**：无（本地操作，无网络请求）
  - **为空**：无会话时侧边栏显示"开始新对话"引导
  - **错误**：localStorage错误显示警告条，降级为内存存储不阻塞使用

### 5.2 API端点（如适用）

本模块为纯前端实现，无后端API调用。

## 6. 数据模型

```typescript
// localStorage根结构（单一key存储版本化数据）
const STORAGE_KEY = 'gerclaw_v1';

interface GerClawLocalStorage {
  version: string;                 // 数据版本号，当前"1"
  disclaimerAccepted: boolean;     // 是否已同意免责声明
  firstVisitAt: number;            // 首次访问时间戳
  settings: UserSettings;          // 用户设置
  conversations: ConversationStorage;  // 会话数据（见通用对话模块）
  customSkills: CustomSkill[];     // 自定义技能（见技能管理模块）
}

interface UserSettings {
  theme: 'light' | 'dark' | 'system';
  persona: 'patient' | 'doctor';   // 当前角色：患者/医生
  fontSize: 'normal' | 'large';    // 字号设置（老年模式）
}

interface StorageUsage {
  usedBytes: number;       // 已使用字节数
  totalEstimated: number;  // 预估总容量（通常5-10MB）
  usagePercent: number;    // 使用百分比
}

// 存储阈值常量
const STORAGE_WARN_THRESHOLD = 4 * 1024 * 1024;    // 4MB 警告阈值
const STORAGE_AUTO_CLEAN_THRESHOLD = 4.5 * 1024 * 1024;  // 4.5MB 自动清理阈值
const STORAGE_TARGET_SIZE = 4 * 1024 * 1024;      // 清理后目标大小
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | localStorage读写<50ms；防抖写入（300ms合并频繁更新）；存储大小计算<10ms；欢迎页渲染<200ms |
| 安全 | 不收集任何用户身份信息（无账号、无设备ID、无IP记录）；医疗免责声明必须用户主动勾选确认才能使用；清除数据功能不可逆，必须二次确认 |
| 可靠性 | localStorage不可用时优雅降级为内存存储，不阻塞核心功能；数据损坏时重置而非崩溃；自动清理策略防止存储溢出 |
| 隐私 | 所有数据仅存本地浏览器；隐私政策清晰可见；用户可一键完全清除数据；不向第三方（除模型API外）发送任何数据 |
| 可观测性 | 存储使用率统计；localStorage读写异常记录日志；欢迎页显示/跳过记录；数据清除事件记录 |

## 8. 不做什么（Out of Scope）

- 用户注册/登录系统（二阶段P1功能）
- 云端数据同步（二阶段账号系统后实现）
- 多设备数据同步（同上）
- 数据导出为备份文件（MVP导出功能在结果导出模块，全量备份导出二阶段考虑）
- 访客身份识别（不生成访客ID、不追踪用户）
- 数据加密存储（localStorage明文存储，二阶段考虑加密）
- 访客用量限制（不限制对话次数、不设配额）
- 密码保护本地数据（二阶段考虑可选密码锁）
- 数据导入功能（从备份文件恢复）
