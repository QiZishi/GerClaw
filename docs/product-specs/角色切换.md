# 角色切换 — 产品规格

> 模块：角色切换 | 优先级：P0 | 基于PRD.md第4.2节生成

---

## 1. 模块概述

GerClaw平台实现医生端/患者端双角色模式切换的核心模块，通过侧边栏底部切换按钮实现角色快速切换，切换后采用页面刷新方式加载对应角色的UI布局和功能模块。患者端默认联动老年模式（大字体、语音优先、适老化交互），医生端始终保持标准专业模式（高信息密度、专业功能区完整展示）。访客模式无需登录即可使用所有功能，当前角色状态通过localStorage持久化保存。医生端展示患者列表（模拟数据）、处方编辑审核、CGA工作区等专业功能；患者端展示语音优先入口、大字交互、AI回复自动TTS等适老化功能。

## 4. 交互流程

### 4.1 主流程（Happy Path）

**首次访问流程：**

```
用户访问GerClaw应用URL
  ↓
检测localStorage中无gerclaw_current_role记录
  ↓
默认角色设为'patient'，保存到localStorage
  ↓
联动适老化模块，默认开启老年模式
  ↓
页面加载患者端UI：
  ├─ 侧边栏顶部显示"患者端"标识
  ├─ 侧边栏底部显示"切换到医生端"按钮+老年模式开关
  ├─ 主区显示患者端欢迎页：大字问候+大麦克风提示
  ├─ 输入框placeholder："您可以直接说话，或输入文字描述您的健康问题..."
  └─ 隐藏患者列表、CGA工作区、处方编辑按钮等医生端特有功能
  ↓
用户可直接开始语音或文字对话（无需登录）
```

**患者端→医生端切换流程：**

```
用户在患者端点击侧边栏底部"切换到医生端"按钮
  ↓
弹出确认对话框：
  标题："切换到医生端"
  正文："切换后将重新加载页面，当前会话将保留。是否继续？"
  按钮：[取消] [确认切换]
  ↓
用户点击"取消" → 对话框关闭，继续留在患者端
用户点击"确认切换" → 继续流程
  ↓
localStorage写入gerclaw_current_role = 'doctor'
  ↓
如有正在进行的API请求/流式输出，提示用户"正在生成的内容将被中断"
  ↓
window.location.reload() 触发页面刷新
  ↓
页面重新加载，初始化检测到currentRole='doctor'
  ↓
强制关闭老年模式，应用标准模式样式（14px字号、标准按钮）
  ↓
加载医生端UI：
  ├─ 侧边栏顶部显示"医生端"标识
  ├─ 侧边栏底部显示"切换到患者端"按钮，隐藏老年模式开关
  ├─ 侧边栏显示"患者列表"入口
  ├─ 主区显示医生端欢迎页：专业问候+患者列表/CGA/处方快捷入口
  ├─ 输入框placeholder："输入患者症状、查看评估结果或编辑处方..."
  ├─ 处方预览面板显示"编辑""审核通过""驳回"按钮
  └─ 主区或侧边栏显示"CGA工作区"入口
  ↓
医生端功能可用
```

**医生端→患者端切换流程：**

```
用户在医生端点击侧边栏底部"切换到患者端"按钮
  ↓
弹出确认对话框：
  标题："切换到患者端"
  正文："切换后将重新加载页面，当前会话将保留。是否继续？"
  按钮：[取消] [确认切换]
  ↓
用户确认切换
  ↓
localStorage写入gerclaw_current_role = 'patient'
  ↓
页面刷新
  ↓
加载患者端UI，默认开启老年模式
  ↓
回到首次访问的患者端状态
```

**医生端-患者列表使用流程：**

```
医生端用户点击侧边栏"患者列表"入口
  ↓
展开患者列表面板（或跳转到患者列表视图）
  ↓
展示模拟患者列表（5-10条）：
  ├─ 姓名、年龄、性别
  ├─ 最近就诊时间
  ├─ 风险等级标签（低/中/高，颜色区分）
  └─ 主要诊断列表
  ↓
用户点击某个患者项
  ↓
加载该患者的对话上下文（或新建该患者对话）
  ↓
可针对该患者进行咨询、开处方、CGA评估等操作
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| localStorage被禁用或写入失败（隐私模式） | 使用内存状态存储当前角色；刷新页面后恢复默认患者端；角色切换功能本身仍可用 | 控制台输出warn日志记录存储失败；不显示错误提示给用户（静默降级）；如刷新后回到默认角色用户可重新切换 |
| 切换角色时存在正在进行的流式对话/API请求 | 切换前提示用户确认中断；用户确认后取消进行中的请求再刷新 | 确认对话框额外提示："当前有正在生成的内容，切换后将中断。" |
| 页面刷新被浏览器/扩展程序阻止 | 捕获reload异常；显示提示引导用户手动刷新 | Alert或大提示条："页面自动刷新被阻止，请按F5或手动刷新页面以完成角色切换。" |
| localStorage中角色值被手动篡改为非法值（如undefined/null/''） | 类型校验失败，回退到默认值'patient'；覆盖写入正确默认值 | 无用户提示，自动恢复默认患者端；控制台warn记录非法值 |
| 模拟患者数据加载失败（极端情况） | 显示空状态提示，提供重试按钮；不影响其他功能使用 | 患者列表面板显示"暂无患者数据"+"重试"按钮 |
| 用户在医生端通过URL参数或DevTools强行修改角色状态 | 初始化时做角色合法性校验，非法值回退默认；CSS通过.doctor-mode类优先级确保医生端样式不被老年模式覆盖 | 无用户提示；下次刷新恢复正确状态 |
| 切换过程中网络断开（纯前端应用） | 角色切换是纯前端操作，离线也可正常切换；静态资源已缓存的情况下刷新后仍可加载 | 如静态资源未缓存导致刷新后无法加载，显示浏览器默认离线页面；不影响下次联网后使用 |

## 5. 界面/API规格

### 5.1 界面

**组件1：角色切换按钮（RoleSwitcher）**
- 页面/组件名：RoleSwitcher
- 入口：左侧边栏底部区域，主题切换按钮上方，始终可见（访客模式）
- 关键元素：
  - 当前角色图标（医生/患者图标）
  - 文字标签：根据当前角色显示"切换到医生端"或"切换到患者端"
  - 点击热区：老年模式≥48px高度，标准模式≥32px高度
  - 按钮样式：次要按钮样式（白底边框或文字按钮）
- 状态要求：
  - 默认状态：显示当前角色对应的切换目标文字和图标
  - 悬停状态：背景色变化提示可点击
  - 点击状态：弹出确认对话框
  - 禁用状态：P1账号登录后根据账号类型固定角色时禁用（MVP不禁用）

**组件2：当前角色标识（RoleIndicator）**
- 页面/组件名：RoleIndicator
- 入口：左侧边栏顶部区域，Logo附近或侧边栏底部切换按钮上方
- 关键元素：
  - 角色图标（医生/患者）
  - 角色名称文字："患者端"或"医生端"
  - 可选辅助颜色标识：患者端温暖色调，医生端专业蓝色调
- 状态要求：
  - 患者端：显示患者图标+"患者端"
  - 医生端：显示医生图标+"医生端"
  - 始终清晰可见，让用户明确当前角色

**组件3：患者列表（PatientList，医生端特有）**
- 页面/组件名：PatientList
- 入口：医生端左侧边栏导航项，或主区欢迎页快捷入口
- 关键元素：
  - 列表标题："患者列表"
  - 患者卡片/列表项，每项包含：
    - 头像（可选，默认首字母或图标）
    - 姓名（加粗）
    - 年龄+性别（如"78岁 女"）
    - 风险等级标签：低风险（绿色）/中风险（黄色）/高风险（红色）
    - 最近就诊日期
    - 主要诊断（1-3个，标签形式）
  - 点击患者项进入该患者对话上下文
- 状态要求：
  - 加载中：骨架屏占位（MVP模拟数据可直接展示，无需加载态）
  - 为空：显示"暂无患者"提示（MVP有预置模拟数据，正常不会为空）
  - 错误：显示"加载失败"+重试按钮
  - 患者端：组件完全不渲染

**组件4：医生端处方操作区（PrescriptionActions，医生端特有）**
- 页面/组件名：PrescriptionActions
- 入口：右侧处方预览面板底部或顶部操作栏
- 关键元素：
  - "编辑"按钮：点击进入处方编辑模式，可修改各处方便是内容
  - "审核通过"按钮（主色）：标记处方为已审核状态
  - "驳回"按钮（危险色）：驳回处方并可输入驳回原因
  - "导出"按钮：保留原导出PDF/Markdown/DOCX功能
- 状态要求：

  - 审核状态标记：审核通过显示绿色"已审核"徽章，驳回显示红色"已驳回"徽章

**组件5：CGA工作区入口（CGAWorkspaceEntry，医生端特有）**

  - 有活跃会话时自动隐藏

### 5.2 API端点（MVP阶段纯前端，无后端API）

本模块MVP阶段为纯前端状态管理模块，无后端API调用。所有角色状态通过localStorage持久化，患者列表使用前端硬编码模拟数据。

（二阶段账号模式将增加角色相关API：获取当前用户角色、患者列表查询、处方审核提交等，MVP不涉及。）

## 6. 数据模型

```typescript
// 用户角色类型
type UserRole = 'patient' | 'doctor';

// 风险等级（患者列表用）
type RiskLevel = 'low' | 'medium' | 'high';

// 角色切换全局状态
interface RoleState {
  // 当前角色：患者/医生（默认'patient'）
  currentRole: UserRole;
}

// localStorage存储键
const ROLE_STORAGE_KEYS = {
  CURRENT_ROLE: 'gerclaw_current_role',
} as const;

// 模拟患者数据结构（医生端患者列表用）
interface MockPatient {
  id: string;
  name: string;
  age: number;
  gender: '男' | '女';
  lastVisit: string; // ISO日期字符串，如'2026-07-01'
  riskLevel: RiskLevel;
  conditions: string[]; // 主要诊断列表，如['高血压', '2型糖尿病']
  avatar?: string; // 可选头像URL
}

// 处方审核状态（医生端用）
type PrescriptionStatus = 'draft' | 'pending_review' | 'approved' | 'rejected';

// 处方审核操作
interface PrescriptionReview {
  status: PrescriptionStatus;
  reviewedBy?: string; // 医生ID/姓名（MVP为模拟值）
  reviewedAt?: string; // ISO时间戳
  rejectReason?: string; // 驳回原因
  editedContent?: string; // 编辑后的处方内容
}

// CGA评估状态汇总（医生端CGA工作区用）
interface CGAAssessmentSummary {
  patientId: string;
  patientName: string;
  assessmentDate: string;
  scales: {
    phq9?: { score: number; severity: string };
    sas?: { score: number; severity: string };
    psqi?: { score: number; severity: string };
    miniCog?: { score: number; severity: string };
    mmse?: { score: number; severity: string };
  };
  overallRisk: RiskLevel;
}

// 角色切换确认对话框选项
interface RoleSwitchConfirmOptions {
  targetRole: UserRole;
  hasActiveGeneration: boolean; // 是否有正在生成的内容
}
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 角色切换页面刷新后首屏加载<3秒（静态资源CDN缓存）；切换按钮点击到确认对话框弹出<100ms；localStorage读写<50ms；患者列表渲染<200ms（模拟数据） |
| 安全 | 访客模式不收集任何个人身份信息；localStorage仅存储UI偏好和会话文本，不存储敏感医疗数据；医生端模拟数据仅为演示用，不包含真实患者信息；角色切换操作不触发任何外部API请求，不泄露用户偏好 |
| 可靠性 | localStorage写入失败时静默降级到内存状态；角色值非法时自动回退默认值并修复；页面刷新被阻止时有明确提示引导用户手动操作；模拟数据加载有兜底空状态 |
| 可观测性 | 关键操作控制台日志：应用初始化角色状态、角色切换请求/确认/取消、localStorage读写状态、角色切换刷新结果；错误日志包含当前角色信息便于排查；日志格式统一加[RoleSwitch]前缀 |
| 兼容性 | Chrome/Edge/Safari最新2个版本；支持桌面端和平板端；localStorage和matchMedia API在目标浏览器100%支持；键盘导航完整（Tab可聚焦切换按钮，Enter触发，Esc关闭对话框） |
| 适老化关联 | 患者端默认老年模式，角色切换模块的按钮尺寸、文字大小、确认对话框样式自动适配适老化模块规范；医生端始终标准模式，适老化规范不应用于医生端 |

## 8. 不做什么（Out of Scope）

- 账号注册/登录系统（P1功能，MVP访客模式即可）
- 根据账号类型自动确定角色并隐藏切换按钮（P1，依赖账号系统）
- 角色切换时的无刷新动态切换（MVP采用刷新式切换，简单可靠；二阶段账号模式再实现无刷新切换）
- 患者列表真实后端数据加载（MVP使用前端硬编码模拟数据，二阶段从FastAPI后端获取）
- 处方编辑的完整富文本编辑器（MVP提供基础编辑能力，专业编辑器二阶段优化）
- CGA工作区的完整历史对比和数据分析可视化（MVP展示基础汇总，高级分析P1）
- 基于角色的权限控制（MVP无登录，任何人可切换到任意端；二阶段通过账号权限控制）
- 角色切换动画和过渡效果（刷新式切换无页面内过渡动画）
- 多医生/多患者账号体系和数据隔离（MVP所有数据存在同一localStorage，二阶段账号模式做数据隔离）
- 医生端自定义字号或临时老年模式（医生端始终强制标准模式，不提供字号调节）
- 切换角色时的会话数据迁移或不同角色独立会话池（MVP会话数据全局共享，二阶段账号模式再做隔离）
