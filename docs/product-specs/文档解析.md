# 文档解析 — 产品规格

> 模块：文档解析 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

GerClaw平台的文档上传与解析核心模块，提供拖拽/点击双模式文件上传能力，支持图片(JPG/PNG)通过多模态模型直接理解、PDF/Word/Markdown/文本文件通过MinerU云API解析为结构化Markdown，上传文件以可移除标签形式展示在输入框上方，解析内容通过隔离标记包裹注入对话上下文防prompt injection，支持上传/解析进度实时显示、解析失败降级处理、右侧面板文档预览，为老年患者上传病历/检查报告、医生上传医学文献提供便捷的文档理解能力。

> **无独立页面声明**（依据用户决策与 `gerclaw设计要求.md` §4.2.3⑥）：文档解析**不设独立页面**，全部交互（上传、解析状态、结果引用）以**工具卡片**形式嵌入消息流，对齐 §4.2.3「文档解析状态」可视化项（文件标签+工具卡片双重反馈）；解析完成的 Markdown 内容作为上下文参与对话，不在独立页面展示。

## 2. 用户故事

| 编号 | 作为... | 我想要... | 以便于... |
|------|---------|----------|----------|
| US-001 | 老年患者 | 拖拽或点击上传病历照片、检查报告图片 | 让AI直接查看我的检查结果给出健康建议，不用打字描述 |
| US-002 | 老年患者 | 上传PDF格式的体检报告或出院小结 | AI能完整读取报告内容进行综合分析 |
| US-003 | 老年患者 | 看到文件上传进度和解析状态 | 知道文件正在处理中，不会重复上传 |
| US-004 | 老年患者 | 上传错误时能移除已选文件重新上传 | 操作灵活，不会因为误传文件而无法继续 |
| US-005 | 老年科医生 | 上传Word格式的医学指南或文献PDF | 让AI基于上传的专业文档回答问题，给出有依据的建议 |

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户点击📎按钮/拖拽文件到聊天区
  ↓
前端文件选择/拖拽释放
  ↓
格式校验：扩展名+MIME类型检查
  ↓
大小/页数校验：≤10MB、PDF≤20页
  ↓
文件添加到标签区域，状态：上传中（显示进度条）
  ↓
┌─────────────────────────────────────────────────┐
│ 分支判断：文件类型                                │
├─────────────────────────────────────────────────┤
│ 图片(JPG/PNG)                                     │
│  → 读取为base64/dataURL                           │
│  → 状态切换：上传完成（显示缩略图）                │
│  → 等待发送消息时作为多模态内容                   │
└─────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────┐
│ 文档(PDF/DOCX/MD/TXT)                            │
│  → 调用MinerU API: POST /parse/file 获取task_id  │
│     和签名上传URL                                 │
│  → PUT文件到签名URL（显示上传进度）               │
│  → 状态切换：解析中（旋转动画+工具卡片）          │
│  → 轮询GET /parse/{task_id}查询状态              │
│  → 解析完成：获取markdown_url，下载Markdown内容  │
│  → 状态切换：完成（绿色✓）                        │
│  → 解析内容暂存，等待发送消息时注入上下文         │
└─────────────────────────────────────────────────┘
  ↓
用户输入文本并点击发送
  ↓
组装消息：
  - 图片：转为对应协议要求的多模态格式（image_url/base64）
  - 文档：已解析Markdown用隔离标记包裹，拼接到用户消息末尾
  ↓
调用大模型API（流式）
  ↓
用户点击文件标签 → 右侧面板预览解析内容
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 文件格式不支持 | 阻止文件添加到标签区域 | toast提示："不支持的文件格式，请上传图片(JPG/PNG)或文档(PDF/DOCX/MD/TXT)" |
| 文件超过10MB | 阻止上传，不添加标签 | toast提示："文件大小不能超过10MB" |
| PDF超过20页 | 阻止上传，不添加标签 | toast提示："PDF文件不能超过20页" |
| 上传过程网络中断 | 标签显示上传失败状态，提供重试按钮 | 标签显示红色✗，hover提示"上传失败，请检查网络"，点击重试按钮重新上传 |
| MinerU API获取签名URL失败 | 标签显示解析失败状态，提供重试按钮 | 标签红色✗，hover提示"解析服务暂时不可用"，可点击重试 |
| MinerU文件上传失败 | 同上 | 同上 |
| MinerU解析超时（>5分钟） | 停止轮询，标记为失败，提示重试 | 标签红色✗，hover提示"解析超时，请重试" |
| MinerU返回failed状态 | 标记为失败，显示具体错误信息 | 标签红色✗，hover显示err_msg（如"文件损坏"、"格式不支持"） |
| Markdown下载失败 | 标记为失败，提示重试 | 标签红色✗，hover提示"结果获取失败，请重试" |
| 解析中用户移除文件 | 取消正在进行的解析请求（轮询停止），移除标签 | 标签立即消失，无额外提示 |
| MinerU API不可用 | 文档标记为失败，但不影响图片上传和其他功能 | 文档标签显示错误，图片仍可正常上传使用；toast提示"文档解析服务暂时不可用，图片仍可正常上传" |

## 5. 界面/API规格

### 5.1 界面

- **页面/组件名**：文件上传组件（FileUpload）+ 文件标签区域（AttachmentsTags）+ 文件预览面板（FilePreviewPanel）
- **入口**：
  - 点击输入框左侧📎按钮
  - 拖拽文件到主聊天区任意位置
  - 点击已上传文件标签（预览）
- **关键元素**：
  - **拖拽高亮层**：拖拽文件进入窗口时显示的半透明遮罩，居中提示"释放文件以上传"，带边框高亮
  - **文件选择器**：隐藏的`<input type="file">`，通过📎按钮触发，`multiple`属性支持多选，`accept`限制格式
  - **文件标签（AttachmentsTags区域，输入框上方）**：
    - 标签容器：横向滚动区域，flex布局
    - 单个标签：圆角背景，包含：
      - 图片：小缩略图（32x32px，object-cover）
      - 文档：文件类型图标（PDF蓝色、Word蓝色、Markdown灰色、文本灰色）
      - 文件名：截断显示（最大120px宽度，hover显示完整名tooltip）
      - 文件大小：格式化显示（如"2.3MB"、"450KB"）
      - 状态指示器：
        - 上传中：线性进度条（标签底部）
        - 解析中：旋转加载图标 + "解析中"文字
        - 完成：绿色✓图标
        - 失败：红色✗图标 + 重试按钮
      - 移除按钮：×图标，hover显示红色，点击移除
  - **工具调用卡片**（消息区内，对齐Trae Work体验）：
    - 文档解析时显示独立工具卡片：📄图标 + "文档解析" + 状态徽章（运行中旋转→完成✓→失败✗）
    - 默认折叠，点击展开显示文件名、解析耗时、结果状态
  - **右侧预览面板**：
    - 面板顶部：文件名（左侧）+ 关闭按钮（右侧×）
    - 面板内容：使用MarkdownRenderer渲染解析后的Markdown内容，支持滚动
    - 面板宽度：默认360px，可拖拽320-500px
- **状态要求**：
  - **主流程**：拖拽/点击选择→上传中（进度条）→解析中（动画）→完成（✓）→可预览/可发送
  - **加载中**：上传中显示进度条；解析中显示旋转动画+工具卡片运行中状态
  - **为空**：标签区域为空时不占高度或显示"拖放文件到此处或点击📎上传"提示
  - **错误**：标签红色✗状态，hover显示错误tooltip，显示重试按钮；toast提示具体错误原因

### 5.2 API端点（MinerU云API）

> MVP阶段为纯前端直连MinerU云API（无需登录，IP限频），无需自有后端代理。

| 方法 | 路径 | 描述 | 认证 | 请求参数 | 响应格式 |
|------|------|------|------|---------|---------|
| POST | `https://mineru.net/api/v1/agent/parse/file` | 获取文件上传签名URL和task_id | 无需认证（IP限频） | Body(JSON): `{file_name: string, language?: string, enable_table?: bool, is_ocr?: bool, enable_formula?: bool, page_range?: string}` | `{code: 0, data: {task_id: string, file_url: string}, msg: "ok"}` |
| PUT | `{data.file_url}` | 上传文件到OSS签名URL | 无需认证（签名URL自带权限） | Body: 二进制文件内容 | 空Body，200状态码表示成功 |
| GET | `https://mineru.net/api/v1/agent/parse/{task_id}` | 轮询解析状态和结果 | 无需认证 | Path参数: task_id | `{code: 0, data: {task_id: string, state: "waiting-file"\|"uploading"\|"pending"\|"running"\|"done"\|"failed", markdown_url?: string, err_msg?: string}}` |
| GET | `{data.markdown_url}` | 下载解析完成的Markdown文件 | 无需认证（CDN链接） | - | Markdown纯文本内容 |

**MinerU API配置（环境变量预留）**：
```env
# === MinerU文档解析服务 ===
MINERU_API_BASE_URL=https://mineru.net/api/v1/agent
MINERU_API_KEY=  # 待定：当前API无需认证，如后续需要API Key在此配置
MINERU_POLL_INTERVAL=3000  # 轮询间隔（毫秒）
MINERU_POLL_TIMEOUT=300000  # 轮询超时（毫秒，5分钟）
MINERU_MAX_FILE_SIZE=10485760  # 最大文件大小（字节，10MB）
MINERU_MAX_PDF_PAGES=20  # PDF最大页数
```

## 6. 数据模型

```typescript
// 文件类型
type FileType = 'image' | 'pdf' | 'docx' | 'markdown' | 'text';

// 文件解析状态
type FileParseStatus = 
  | 'idle'        // 初始状态
  | 'uploading'   // 上传中
  | 'parsing'     // 解析中（MinerU轮询中）
  | 'completed'   // 解析完成
  | 'failed';     // 解析/上传失败

// 支持的MIME类型映射
const SUPPORTED_MIME_TYPES: Record<string, FileType> = {
  'image/jpeg': 'image',
  'image/png': 'image',
  'application/pdf': 'pdf',
  'application/vnd.openxmlformats-officedocument.wordprocessingml.document': 'docx',
  'text/markdown': 'markdown',
  'text/plain': 'text',
};

// 支持的扩展名映射
const SUPPORTED_EXTENSIONS: Record<string, FileType> = {
  '.jpg': 'image',
  '.jpeg': 'image',
  '.png': 'image',
  '.pdf': 'pdf',
  '.docx': 'docx',
  '.md': 'markdown',
  '.markdown': 'markdown',
  '.txt': 'text',
};

// 上传的文件对象
interface UploadedFile {
  id: string;                    // 文件唯一ID（uuid）
  file: File;                    // 原始File对象
  name: string;                  // 文件名
  size: number;                  // 文件大小（字节）
  type: FileType;                // 文件类型
  mimeType: string;              // MIME类型
  status: FileParseStatus;       // 解析状态
  uploadProgress: number;        // 上传进度（0-100）
  thumbnailUrl?: string;         // 图片缩略图（base64/objectURL）
  parsedContent?: string;        // MinerU解析后的Markdown内容
  taskId?: string;               // MinerU任务ID
  error?: string;                // 错误信息
  startTime: number;             // 开始上传时间戳
  endTime?: number;              // 完成/失败时间戳
  durationMs?: number;           // 解析耗时
}

// MinerU API响应类型
interface MinerUResponse<T = any> {
  code: number;
  data: T;
  msg: string;
  trace_id?: string;
}

interface MinerUCreateTaskResponse {
  task_id: string;
  file_url: string;
}

interface MinerUStatusResponse {
  task_id: string;
  state: 'waiting-file' | 'uploading' | 'pending' | 'running' | 'done' | 'failed';
  markdown_url?: string;
  err_msg?: string;
}

// 上下文注入的文档包裹格式
const DOCUMENT_WRAPPER = {
  prefix: (filename: string) => `\n\n--- BEGIN UPLOADED DOCUMENT: ${filename} ---\n`,
  suffix: (filename: string) => `\n--- END UPLOADED DOCUMENT: ${filename} ---\n\n`,
};

// 文件预览面板状态
interface FilePreviewState {
  isOpen: boolean;
  fileId?: string;
}
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 文件上传进度实时更新（每200ms刷新一次）；MinerU轮询间隔3秒；小文件（<1MB）上传+解析总耗时<10s（理想情况）；标签区域渲染流畅，10个文件同时存在无卡顿 |
| 安全 | 上传文件做类型校验（扩展名+MIME双重校验），防止恶意文件上传；解析后的Markdown内容用固定隔离标记包裹，明确标注为"用户上传内容"，防止prompt injection；文件内容不持久化到localStorage（仅当前会话内存持有，刷新页面需重新上传）；API Key如需配置通过环境变量注入，不硬编码 |
| 可靠性 | 上传失败支持手动重试；MinerU解析失败支持重试（不重新上传文件，重新轮询或重新提交）；MinerU不可用时图片多模态功能不受影响；网络中断恢复后不自动重试，需用户手动触发避免重复上传；轮询设置5分钟超时，超时后标记失败 |
| 可观测性 | 文件上传/解析关键步骤记录console日志（上传开始、上传完成、解析开始、解析完成、解析失败）；MinerU API请求/响应错误记录error级别日志（含task_id、错误信息）；解析耗时统计记录在file对象中供工具卡片展示；日志统一前缀`[GerClaw][Document]` |
| 兼容性 | Chrome/Edge/Safari最新2个版本；拖拽功能使用HTML5原生Drag and Drop API；FileReader、XMLHttpRequest（上传进度）为浏览器标准API，兼容所有现代浏览器；患者端老年模式下标签高度≥48px，移除/重试按钮点击区域≥44px |
| 适老化 | 患者端老年模式：文件标签高度≥48px，文字≥16px，×和重试按钮点击区域≥44px；状态图标清晰明显，完成/失败颜色对比度高（绿色成功、红色失败，符合WCAG AA标准）；错误提示文字简洁明确，避免技术术语（如不说"500错误"，说"解析服务暂时不可用"） |

## 8. 不做什么（Out of Scope）

- 二阶段后端FastAPI代理：MVP阶段前端直连MinerU API，二阶段迁移到后端代理隐藏API调用细节
- OCR图片文字提取：图片直接通过多模态模型理解，不单独调用MinerU OCR
- PPTx/Xlsx格式支持：设计要求提到MinerU支持PPTx/Xlsx，但MVP阶段按用户需求仅支持PDF/Word/Markdown/文本和图片
- 文件历史记录：刷新页面后已上传文件不保留（localStorage不存储文件内容，仅会话内有效）
- 文件管理（重命名、分类、搜索）：仅支持上传和移除，无复杂文件管理
- 批量导出已上传文件：MVP无此需求
- 文档内容编辑：预览面板仅展示解析内容，不支持编辑修改
- 断点续传：上传失败后重新上传整个文件，不支持断点续传
- 病毒扫描：MVP纯前端无病毒扫描能力，依赖MinerU API侧的安全防护
- 私有文档存储：文件仅在MinerU和用户浏览器之间传输，GerClaw自有服务不存储用户文档
- 图片编辑/标注：上传图片不支持裁剪、旋转、标注等编辑操作
