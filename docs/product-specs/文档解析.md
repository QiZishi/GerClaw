# 文档解析 — 产品规格

> 模块：文档解析 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

GerClaw平台的文档上传与解析核心模块，提供拖拽/点击双模式文件上传能力，支持图片以多模态内容发送、PDF 和 Word 通过 MinerU 解析为结构化 Markdown，MD/TXT 通过受限本地读取展示；上传文件以可移除标签形式展示在输入框上方，支持解析状态、失败重试和右侧面板预览，为老年患者上传病历/检查报告、医生上传医学文献提供便捷的文档理解能力。解析 Markdown 经会话级加密登记、所有权校验和 Harness 隔离后，才作为受控参考资料参与当前对话。

> **无独立页面声明**（依据用户决策与 `gerclaw设计要求.md` §4.2.3⑥）：文档解析**不设独立页面**，全部交互（上传、解析状态、结果引用）以**工具卡片**形式嵌入消息流，对齐 §4.2.3「文档解析状态」可视化项（文件标签+工具卡片双重反馈）；解析内容可预览，并在安全登记后作为当前会话的受控上下文参与对话。

> **当前实现状态（2026-07-16）**：聊天输入中的 PDF 和 DOCX 已通过 Next.js BFF 实际调用 MinerU 的签名上传、任务轮询与 Markdown 下载；MD/TXT 因 MinerU 轻量接口不支持而采用有大小上限的本地读取。解析 Markdown 会通过同源 BFF 登记到 FastAPI 的 AES-256-GCM 加密 `uploaded_documents` 表，严格绑定 `tenant + actor + chat session`；新会话在首条消息前登记，已有会话解析成功后立即登记。Chat 只接受已登记、未撤销且属于当前会话的 UUID；Harness 将它包裹为不可信参考资料，移除附件即撤销并擦除正文。当前采用有界直接上下文，不写入公共医学知识库、Qdrant 或私有向量检索；长文档/跨会话检索另行设计。

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户点击📎按钮/拖拽文件到聊天区
  ↓
前端文件选择/拖拽释放
  ↓
格式校验：扩展名+MIME类型检查
  ↓
文件大小校验：≤10MB
  ↓
文件添加到标签区域，状态：解析中（稳定文字反馈）
  ↓
┌─────────────────────────────────────────────────┐
│ 分支判断：文件类型                                │
├─────────────────────────────────────────────────┤
│ 图片(JPG/PNG)                                     │
│  → 读取为base64/dataURL                           │
│  → 状态切换：上传完成（显示缩略图）                │
│  → 等待发送消息时作为多模态内容                   │
└─────────────────────────────────────────────────┘
  ↓
┌─────────────────────────────────────────────────┐
│ PDF/DOCX                                         │
│  → POST /api/mineru/parse（Next.js BFF）          │
│  → BFF 向 MinerU 请求签名、上传、轮询并下载 Markdown│
│  → 状态切换：解析中（稳定文字状态+工具卡片）       │
│  → 解析完成：保存 Markdown 供预览，状态切换为完成  │
│ MD/TXT                                            │
│  → 在 1MB 上限内本地读取，保存 Markdown 供预览     │
└─────────────────────────────────────────────────┘
  ↓
用户输入文本并点击发送
  ↓
组装消息：
  - 图片：转为对应协议要求的多模态格式（image_url/base64）
  - 文档：浏览器先向 FastAPI 登记 Markdown，再只传递当前会话已验证的 UUID；Harness 从加密存储读取并以 JSON 数据封装隔离为不可信参考资料
  ↓
调用大模型API（流式）
  ↓
用户点击文件标签 → 右侧面板预览解析内容
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 文件格式不支持 | 阻止文件添加到标签区域 | toast提示："不支持的文件格式，请上传图片(JPG/PNG)或文档(PDF/DOCX/MD/TXT)" |
| 文件超过10MB | 阻止上传，不添加标签 | toast提示："文件大小不能超过10MB" |
| 上传/解析过程网络中断 | 标签显示解析失败状态，提供重试按钮 | 文件卡片直接显示“解析服务暂时不可用，请稍后重试”等可读错误，点击重试重新提交 |
| MinerU API获取签名URL失败 | 标签显示解析失败状态，提供重试按钮 | 文件卡片直接显示“解析服务暂时不可用，请稍后重试”，可点击重试 |
| MinerU文件上传失败 | 同上 | 同上 |
| MinerU解析超时 | BFF 在单次请求超时后标记失败，提示重试 | 文件卡片直接显示“文档解析超时，请稍后重试” |
| MinerU返回failed状态 | 标记为失败，显示安全收敛后的错误信息 | 文件卡片直接显示“文档解析服务暂时不可用，请稍后重试”；不向患者暴露 Provider 原始错误或 trace |
| Markdown下载失败 | 标记为失败，提示重试 | 文件卡片直接显示“文档解析服务暂时不可用，请稍后重试” |
| 解析中用户移除文件 | 从当前界面移除标签和预览；不承诺取消已发往 Provider 的单次请求 | 标签立即消失，无额外提示 |
| MinerU API不可用 | 文档标记为失败，但不影响图片上传和其他功能 | 文档标签显示错误，图片仍可正常上传使用；toast提示"文档解析服务暂时不可用，图片仍可正常上传" |

## 5. 界面/API规格

### 5.1 界面

- **页面/组件名**：文件上传组件（FileUpload）+ 文件标签区域（AttachmentsTags）+ 文件预览面板（FilePreviewPanel）
- **入口**：
  - 点击输入框左侧📎按钮
  - 拖拽文件到主聊天区任意位置
  - 点击已上传文件标签（预览）
- **关键元素**：
  - **拖拽高亮层**：拖拽文件进入窗口时显示的半透明遮罩，居中提示"释放文件以上传"，带边框高亮
  - **文件选择器**：隐藏的`<input type="file">`，通过📎按钮触发，`multiple`属性支持多选，`accept`限制格式
  - **文件标签（AttachmentsTags区域，输入框上方）**：
    - 标签容器：横向滚动区域，flex布局
    - 单个标签：圆角背景，包含：
      - 图片：小缩略图（32x32px，object-cover）
      - 文档：文件类型图标（PDF蓝色、Word蓝色、Markdown灰色、文本灰色）
      - 文件名：截断显示（最大120px宽度，hover显示完整名tooltip）
      - 文件大小：格式化显示（如"2.3MB"、"450KB"）
      - 状态指示器：
        - 解析中：稳定“正在解析文档”文字；不显示无真实来源的百分比或阶段性进度，也不使用持续旋转图标
        - 完成：绿色✓图标
        - 失败：红色✗图标 + 直接可读的错误原因 + 重试按钮
      - 移除按钮：×图标，hover显示红色，点击移除
  - **解析状态卡片**（输入区附件区域，对齐Trae Work的信息层级）：
    - 文档解析时显示独立工具卡片：📄图标 + “文档解析” + 稳定文字状态（运行中→完成✓→失败✗）
    - 默认折叠，点击展开显示文件名、解析耗时、结果状态
  - **右侧预览面板**：
    - 面板顶部：文件名（左侧）+ 关闭按钮（右侧×）
    - 面板内容：使用MarkdownRenderer渲染解析后的Markdown内容，支持滚动
    - 面板宽度：默认360px，可拖拽320-500px
- **状态要求**：
  - **主流程**：拖拽/点击选择→稳定文字状态“解析中”→完成（✓）→可预览；当前文档不随聊天消息发送
  - **加载中**：解析中显示稳定的文字状态与状态卡片，避免高频闪烁和伪进度
  - **为空**：标签区域为空时不占高度或显示"拖放文件到此处或点击📎上传"提示
  - **错误**：标签红色✗状态，卡片内直接显示错误原因和重试按钮；toast 提供补充提示

### 5.2 API端点（Next.js BFF 与 MinerU）

> 浏览器只调用同源 Next.js BFF `POST /api/mineru/parse`；BFF 在校验文件大小、格式与 Provider URL allowlist 后，才向 MinerU 请求签名、上传、轮询并下载 Markdown。禁止浏览器直连 MinerU 或暴露 API Key。下表外部调用由 BFF 执行，不是浏览器公开接口。

| 方法 | 路径 | 描述 | 认证 | 请求参数 | 响应格式 |
|------|------|------|------|---------|---------|
| POST | `https://mineru.net/api/v1/agent/parse/file` | 获取文件上传签名URL和task_id | 无需认证（IP限频） | Body(JSON): `{file_name: string, language?: string, enable_table?: bool, is_ocr?: bool, enable_formula?: bool, page_range?: string}` | `{code: 0, data: {task_id: string, file_url: string}, msg: "ok"}` |
| PUT | `{data.file_url}` | 上传文件到OSS签名URL | 无需认证（签名URL自带权限） | Body: 二进制文件内容 | 空Body，200状态码表示成功 |
| GET | `https://mineru.net/api/v1/agent/parse/{task_id}` | 轮询解析状态和结果 | 无需认证 | Path参数: task_id | `{code: 0, data: {task_id: string, state: "waiting-file"\|"uploading"\|"pending"\|"running"\|"done"\|"failed", markdown_url?: string, err_msg?: string}}` |
| GET | `{data.markdown_url}` | 下载解析完成的Markdown文件 | 无需认证（CDN链接） | - | Markdown纯文本内容 |

**MinerU 配置（仅服务端环境变量）**：
```env
# === MinerU文档解析服务 ===
MINERU_URL=https://mineru.net/api/v1/agent
# 或 MINERU_API_BASE_URL=https://mineru.net/api/v1/agent
MINERU_ALLOWED_HOSTS=mineru.net,oss-example.mineru.net,cdn-example.mineru.net
MINERU_API_KEY=  # 如 Provider 要求认证，仅在服务端配置
```

## 6. 数据模型

```typescript
// 文件类型
type FileType = 'image' | 'pdf' | 'docx' | 'markdown' | 'text';

// 文件解析状态
type FileParseStatus = 
  | 'idle'        // 初始状态
  | 'uploading'   // 上传中
  | 'parsing'     // 解析中（MinerU轮询中）
  | 'completed'   // 解析完成
  | 'failed';     // 解析/上传失败

// 支持的MIME类型映射
const SUPPORTED_MIME_TYPES: Record<string, FileType> = {
  'image/jpeg': 'image',
  'image/png': 'image',
  'application/pdf': 'pdf',
  'application/vnd.openxmlformats-officedocument.wordprocessingml.document': 'docx',
  'text/markdown': 'markdown',
  'text/plain': 'text',
};

// 支持的扩展名映射
const SUPPORTED_EXTENSIONS: Record<string, FileType> = {
  '.jpg': 'image',
  '.jpeg': 'image',
  '.png': 'image',
  '.pdf': 'pdf',
  '.docx': 'docx',
  '.md': 'markdown',
  '.markdown': 'markdown',
  '.txt': 'text',
};

// 上传的文件对象
interface UploadedFile {
  id: string;                    // 文件唯一ID（uuid）
  file: File;                    // 原始File对象
  name: string;                  // 文件名
  size: number;                  // 文件大小（字节）
  type: FileType;                // 文件类型
  mimeType: string;              // MIME类型
  status: FileParseStatus;       // 解析状态
  uploadProgress: number;        // 上传进度（0-100）
  thumbnailUrl?: string;         // 图片缩略图（base64/objectURL）
  parsedContent?: string;        // MinerU解析后的Markdown内容
  taskId?: string;               // MinerU任务ID
  error?: string;                // 错误信息
  startTime: number;             // 开始上传时间戳
  endTime?: number;              // 完成/失败时间戳
  durationMs?: number;           // 解析耗时
}

// MinerU API响应类型
interface MinerUResponse<T = any> {
  code: number;
  data: T;
  msg: string;
  trace_id?: string;
}

interface MinerUCreateTaskResponse {
  task_id: string;
  file_url: string;
}

interface MinerUStatusResponse {
  task_id: string;
  state: 'waiting-file' | 'uploading' | 'pending' | 'running' | 'done' | 'failed';
  markdown_url?: string;
  err_msg?: string;
}

// 目标态：后端完成文件登记、所有权校验和会话绑定后，才可使用的上下文包裹格式。
// 当前前端不会将 parsedContent 发送给聊天接口。
const DOCUMENT_WRAPPER = {
  prefix: (filename: string) => `\n\n--- BEGIN UPLOADED DOCUMENT: ${filename} ---\n`,
  suffix: (filename: string) => `\n--- END UPLOADED DOCUMENT: ${filename} ---\n\n`,
};

// 文件预览面板状态
interface FilePreviewState {
  isOpen: boolean;
  fileId?: string;
}
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 不以固定计时伪造上传/解析进度；BFF 轮询间隔 2 秒，单次解析请求设 60 秒超时；标签区域渲染流畅，10个文件同时存在无卡顿 |
| 安全 | 上传文件做类型校验（扩展名+MIME双重校验）与大小限制；浏览器经 BFF 访问 MinerU，BFF 校验 Provider URL allowlist，API Key 仅由环境变量注入且不下发浏览器；文件内容不持久化到 localStorage。Markdown 仅以加密列保存、按 tenant/actor/session 验证、以固定隔离标记进入 Harness；活跃 HTML 和典型指令性文本会被删除，不进入公共 RAG、Qdrant、日志或 Trace |
| 可靠性 | 上传/解析失败支持用户手动重新提交；MinerU 不可用时图片多模态功能不受影响；网络中断恢复后不自动重试，需用户手动触发避免重复上传；BFF 为单次解析请求设置 60 秒超时并返回可理解的失败提示 |
| 可观测性 | 文件上传/解析关键步骤记录console日志（上传开始、上传完成、解析开始、解析完成、解析失败）；MinerU API请求/响应错误记录error级别日志（含task_id、错误信息）；解析耗时统计记录在file对象中供工具卡片展示；日志统一前缀`[GerClaw][Document]` |
| 兼容性 | Chrome/Edge/Safari最新2个版本；拖拽功能使用HTML5原生Drag and Drop API；FileReader、XMLHttpRequest（上传进度）为浏览器标准API，兼容所有现代浏览器；患者端老年模式下标签高度≥48px，移除/重试按钮点击区域≥44px |
| 适老化 | 患者端老年模式：文件标签高度≥48px，文字≥16px，×和重试按钮点击区域≥44px；状态图标清晰明显，完成/失败颜色对比度高（绿色成功、红色失败，符合WCAG AA标准）；错误提示文字简洁明确，避免技术术语（如不说"500错误"，说"解析服务暂时不可用"） |

## 8. 不做什么（Out of Scope）

- 私有向量 RAG、跨会话长期文档库、医生跨患者访问、长期保留/导出与独立病毒扫描：尚未实现；当前仅支持会话级、有界直接上下文
- OCR图片文字提取：图片直接通过多模态模型理解，不单独调用MinerU OCR
- PPTx/Xlsx格式支持：设计要求提到MinerU支持PPTx/Xlsx，但MVP阶段按用户需求仅支持PDF/Word/Markdown/文本和图片
- 文件历史记录：刷新页面后已上传文件不保留（localStorage不存储文件内容，仅会话内有效）
- 文件管理（重命名、分类、搜索）：仅支持上传和移除，无复杂文件管理
- 批量导出已上传文件：MVP无此需求
- 文档内容编辑：预览面板仅展示解析内容，不支持编辑修改
- 断点续传：上传失败后重新上传整个文件，不支持断点续传
- 病毒扫描：当前不提供独立病毒扫描能力，不将 Provider 解析能力视为病毒扫描保证
- 私有文档存储：当前浏览器经 Next.js BFF 将文件一次性传给 MinerU 解析，GerClaw 尚无持久化文档库；后续存储须随所有权与保留策略一并设计
- 图片编辑/标注：上传图片不支持裁剪、旋转、标注等编辑操作
