# 通用对话 — 产品规格

> 模块：通用对话 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

GerClaw平台的基础对话核心模块，提供基于大语言模型的流式SSE对话能力，支持多模态输入（文本/语音/图片/文件）、Markdown渲染（含代码高亮、引用角标）、思维链折叠展示、工具调用可视化卡片、localStorage访客会话持久化，以及主备模型自动切换的高可用配置，为老年患者和医生提供专业、流畅、可解释的AI对话交互体验。

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户进入页面
  ↓
localStorage加载历史会话列表 → 渲染左侧边栏
  ↓
（默认选中最新会话或显示欢迎页）
  ↓
用户在输入框输入文本/上传图片/上传文件
  ↓
用户点击发送按钮（或按Enter）
  ↓
用户消息立即渲染到聊天区（右对齐气泡）
  ↓
调用Vercel AI SDK发起流式请求（主模型）
  ↓
显示AI消息占位（左对齐气泡+头像）
  ↓
流式接收chunk：
  ├─ 收到thinking delta → 渲染"思考过程"折叠块，显示"思考中..."动画
  ├─ 收到tool_call delta → 渲染工具卡片（运行中旋转状态）
  ├─ 收到tool_result delta → 更新工具卡片状态（完成✓/失败✗，显示耗时）
  └─ 收到text delta → 打字机效果追加文本，实时Markdown渲染
  ↓
流结束 → 收起思考过程区块，显示消息操作按钮
  ↓
自动保存会话到localStorage
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 主模型API超时（>60s） | 自动取消当前请求，尝试backup1模型，再失败尝试backup2 | 如所有模型均失败，显示友好错误消息："当前服务繁忙，请稍后重试" |
| 主模型返回5xx/429错误 | 自动切换到下一个备份模型重试，记录切换日志 | 切换过程无用户感知；全部失败时显示错误提示 |
| 图片上传失败（格式/大小超限） | 阻止上传，不发送消息 | 明确提示："仅支持JPG/PNG/WebP格式，单张不超过10MB" |
| 文件上传失败（MinerU解析错误） | 标记文件标签为失败状态，不将文件内容加入上下文 | 文件标签显示红色错误状态，hover提示"解析失败，请重试"，提供重试按钮 |
| localStorage存储已满（约5-10MB） | 自动清理最早的非固定会话，提示用户导出重要内容 | 提示："本地存储空间不足，已清理最早的对话记录，重要内容请及时导出" |
| 网络断开 | SSE流中断，标记AI消息为不完整状态 | 显示"网络中断，点击重试"按钮，保留已接收内容 |
| 用户发送空消息 | 阻止发送 | 无提示，发送按钮保持禁用状态 |

## 5. 界面/API规格

### 5.1 界面

- **页面/组件名**：通用对话界面（主聊天区）
- **入口**：页面默认加载即为通用对话界面，左侧边栏点击任意会话进入
- **关键元素**：
  - 左侧边栏：新建对话按钮、会话搜索框、历史会话列表（分组：今天/昨天/最近7天/更早）、会话项hover操作（重命名/删除/固定）
  - 主聊天区：欢迎页（Logo+问候语+快捷入口+示例提示词）、消息列表（用户气泡右对齐/AI气泡左对齐）、消息操作按钮组（复制/重新生成/语音朗读）、思考过程折叠块、工具调用卡片、引用角标、回到底部悬浮按钮
  - 输入区域：标签区域（已加载技能/已上传文件，可点击×移除）、输入框（左侧功能按钮：📎文件/⚡技能/💊处方/📋评估；中间多行文本输入；右侧按钮：🎤麦克风/✈️发送/⏹停止）、底部免责声明提示
- **状态要求**：
  - **主流程**：消息正常发送→流式回复→操作按钮显示
  - **加载中**：AI回复时显示占位头像，思考过程显示"思考中..."动画，工具卡片旋转状态，输入框显示停止按钮
  - **为空**：新会话显示完整欢迎页（Logo+快捷入口+示例提示词），历史会话列表为空时显示引导提示
  - **错误**：模型全部失败显示错误提示卡片，文件解析失败在标签上显示红色错误状态，网络中断在消息末尾显示重试按钮

### 5.2 API端点（如适用）

> MVP阶段为纯前端静态导出，无自有后端API，所有LLM请求由浏览器直连模型API（通过Vercel AI SDK统一封装）。

| 方法 | 路径 | 描述 | 认证 | 请求参数 | 响应格式 |
|------|------|------|------|---------|---------|
| POST | `{PRIMARY_MODEL_URL}/chat/completions` | OpenAI协议主模型流式对话 | Bearer Token（环境变量注入） | `{model, messages, stream: true, ...}` | SSE流：`data: {choices: [{delta: {content, reasoning_content, tool_calls}}]}` |
| POST | `{BACKUP1_MODEL_URL}/...` | 备份模型1对话（协议由`*_PROTOCOL`决定） | 对应协议认证 | 同对应协议要求 | 对应协议SSE流 |
| POST | MinerU云API端点（待定） | 文件解析为Markdown | API Key（后续提供） | FormData: 文件上传 | JSON: `{markdown_content, status}` |

AGENT_BACKUP1_URL=...
AGENT_BACKUP1_API_KEY=...
AGENT_BACKUP1_MODEL=...
AGENT_BACKUP1_PROTOCOL=...
AGENT_BACKUP1_PREFERENCE=backup1
AGENT_BACKUP2_URL=...
AGENT_BACKUP2_API_KEY=...

## 6. 数据模型

```typescript
// 会话元数据
interface Conversation {
  id: string;
  title: string;
  createdAt: number;
  updatedAt: number;
  isPinned: boolean;
  messageIds: string[];
}

// 消息角色
type MessageRole = 'user' | 'assistant' | 'system';

// 消息内容块类型
type ContentBlockType = 'text' | 'thinking' | 'tool_call' | 'tool_result' | 'image' | 'file' | 'citation';

// 基础内容块
interface ContentBlock {
  id: string;
  type: ContentBlockType;
}

// 文本块
interface TextBlock extends ContentBlock {
  type: 'text';
  content: string;
}

// 思维链块
interface ThinkingBlock extends ContentBlock {
  type: 'thinking';
  content: string;
  isCollapsed: boolean;
  isStreaming: boolean;
}

// 工具调用状态
type ToolCallStatus = 'running' | 'completed' | 'failed';

// 工具调用块
interface ToolCallBlock extends ContentBlock {
  type: 'tool_call';
  toolName: string;
  toolIcon?: string;
  status: ToolCallStatus;
  params: Record<string, unknown>;
  result?: unknown;
  error?: string;
  durationMs?: number;
  startTime: number;
  endTime?: number;
  isCollapsed: boolean;
}

// 图片块
interface ImageBlock extends ContentBlock {
  type: 'image';
  url: string; // base64或objectURL
  mimeType: string;
  filename?: string;
  size: number;
}

// 文件块
type FileParseStatus = 'uploading' | 'parsing' | 'completed' | 'failed';

interface FileBlock extends ContentBlock {
  type: 'file';
  filename: string;
  mimeType: string;
  size: number;
  parseStatus: FileParseStatus;
  parsedContent?: string; // MinerU解析后的Markdown
  error?: string;
}

// 引用块
interface CitationBlock extends ContentBlock {
  type: 'citation';
  index: number;
  title: string;
  source: string;
  url?: string;
  snippet?: string;
}

// 消息
interface Message {
  id: string;
  conversationId: string;
  role: MessageRole;
  blocks: ContentBlock[];
  createdAt: number;
  isStreaming?: boolean;
  error?: string;
}

// localStorage存储结构
interface LocalStorageData {
  version: string;
  conversations: Record<string, Conversation>;
  messages: Record<string, Message>;
  conversationOrder: string[]; // 会话ID按更新时间倒序
}

// 模型配置（运行时从环境变量读取）
interface ModelConfig {
  url: string;
  apiKey: string;
  modelName: string;
  protocol: 'openai' | 'dashscope' | 'anthropic';
  preference: 'primary' | 'backup1' | 'backup2';
}
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏加载<3s（静态资源CDN）；LLM首token响应<2s（流式）；SSE流解析无明显卡顿；Markdown实时渲染流畅；localStorage读写<100ms |
| 安全 | API Key通过环境变量注入，不硬编码到仓库；HTTPS-only（部署平台保证）；前端不存储敏感医疗数据（localStorage仅存会话文本）；用户输入做XSS防护（Markdown渲染时转义危险HTML）；禁止给出确定性诊断结论 |
| 可靠性 | API超时设置：LLM 60s；失败自动重试（最多2次）；主备模型自动切换（primary→backup1→backup2）；localStorage写入异常时降级为内存存储并提示用户；网络中断支持重试 |
| 可观测性 | 关键操作记录console日志（模型切换、API错误、存储异常）；模型切换时记录warn级别日志；API错误记录error级别日志并包含错误类型和模型信息 |
| 兼容性 | Chrome/Edge/Safari最新2个版本；桌面端和平板端完整支持，手机端基础可用；患者端老年模式字体≥18px、按钮≥48px、高对比度 |
| 适老化 | 患者端默认老年模式：正文≥18px，标题≥20px，按钮≥48px点击区域，高对比度配色，语音入口突出，关键操作二次确认 |

## 8. 不做什么（Out of Scope）

- 账号注册/登录系统（访客模式localStorage即可，二阶段实现）
- 后端FastAPI代理服务（MVP纯前端直连，二阶段迁移）
- RAG本地知识库检索（联网搜索替代，二阶段实现）
- 健康画像持久化管理（二阶段实现）
- AgentScope多智能体编排（单模型prompt实现工具调用，二阶段实现）
- 完整权限引擎和HITL人工审批（prompt控制+免责声明替代，二阶段实现）
- 数据脱敏/隐私合规层（二阶段实现）
- 消息搜索/筛选（MVP仅支持会话列表搜索）
- 会话导出/分享（MVP导出功能在五大处方/CGA模块，通用对话导出二阶段实现）
- 语音交互（ASR/TTS为独立模块，本模块仅预留语音朗读按钮接口）
- 子智能体加载树形可视化（MVP仅支持单轮工具调用卡片，多智能体二阶段实现）
- 上下文窗口自动压缩/摘要（MVP依赖模型自身上下文窗口，超长时提示用户新建对话）
