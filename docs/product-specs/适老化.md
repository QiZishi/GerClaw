# 适老化与角色切换 — 产品规格

> 模块：适老化与角色切换 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

GerClaw平台的核心体验差异化模块，实现患者端/医生端双角色模式切换和患者端老年适老化UI适配。患者端默认开启老年模式（大字体≥18px/正文、≥20px/标题、≥48px/按钮、WCAG AAA高对比度≥7:1、语音优先入口、操作步骤文字提示、关键操作二次确认、减少动画），医生端始终保持标准专业模式（14px标准字号、高信息密度、专业术语完整展示），通过页面顶部/侧边栏切换按钮实现角色快速切换，切换后UI布局、功能区和交互模式同步变化，访客模式无需登录直接使用。

## 4. 交互流程

### 4.1 主流程（Happy Path）

**首次访问流程：**

```
用户访问GerClaw应用URL
  ↓
检测localStorage中无角色偏好 → 默认进入患者端+老年模式
  ↓
显示患者端欢迎页：大GerClaw Logo + "您好，我是您的健康助手"大字问候
  ↓
欢迎页展示：超大麦克风按钮提示"点击麦克风直接说话咨询"
  ↓
输入框placeholder："您可以直接说话，或输入文字描述您的健康问题..."
  ↓
侧边栏底部可见：老年模式开关（默认开）、切换到医生端按钮
  ↓
用户可直接开始使用（无需登录）
```

**患者端→医生端切换流程：**

```
用户在患者端点击侧边栏底部"切换到医生端"按钮
  ↓
弹出确认对话框："切换到医生端将重新加载页面，当前会话将保留。是否继续？"
  对话框按钮：[取消] [确认切换]（按钮≥48px，文字明确）
  ↓
用户点击"确认切换"
  ↓
localStorage保存currentRole = 'doctor'
  ↓
页面刷新
  ↓
加载医生端标准模式：
  - 基准字号14px
  - 侧边栏显示患者列表入口
  - 显示CGA工作区、处方审核入口
  - 欢迎页显示专业问候和工具入口
  - 老年模式开关隐藏（医生端无此选项）
  - 输入框placeholder："输入患者症状、查看评估结果或编辑处方..."
```

**老年模式开关切换流程：**

```
患者在患者端点击侧边栏"老年模式"开关
  ↓
即时切换UI（不刷新页面）：
  - 关闭老年模式：字号缩小到14px，按钮缩小到≥32px，行高1.6，对比度降到AA级(4.5:1)，允许深色主题，AI回复不自动朗读
  - 开启老年模式：字号放大到≥18px，按钮≥48px，行高1.8，强制浅色高对比度，AI回复自动朗读
  ↓
开关状态保存到localStorage
```

**关键操作二次确认流程（以删除对话为例）：**

```
用户（老年模式下）点击会话项的删除按钮
  ↓
弹出大尺寸确认对话框（遮罩层+居中模态框）
  ↓
对话框内容：
  标题（大字20px+）："确认删除对话"
  正文（18px+）："删除后这个对话的所有内容将无法恢复，您确定要删除吗？"
  按钮：
    - [取消]（次要按钮，左侧，≥48px）
    - [确认删除]（主色危险按钮，右侧，≥48px）
  ↓
用户点击"取消" → 对话框关闭，不执行删除
用户点击"确认删除" → 执行删除，对话框关闭，显示操作反馈
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| localStorage被禁用或写入失败 | 使用内存状态存储，刷新后恢复默认（患者端+老年模式） | 控制台输出warn日志，不影响用户使用；不显示错误提示给用户（静默降级） |
| 浏览器不支持prefers-reduced-motion | 使用老年模式默认的慢速动画设置，仍避免快速闪烁动画 | 无用户提示，自动使用兼容的慢动画方案 |
| 切换角色时网络断开（本地环境无网络也可用） | 纯前端应用，离线也可切换角色；如正在进行API调用则切换时取消进行中的请求 | 如有正在生成的内容提示："正在生成的内容将被中断，切换后需重新开始对话" |
| 高对比度模式下某些自定义颜色不达标 | 使用CSS变量统一管控配色，所有颜色组合预先生成并通过对比度检测；兜底使用纯黑(#000000)纯白(#FFFFFF)组合 | 无用户提示，自动使用合规配色 |
| 用户在医生端手动修改localStorage强行开启老年模式 | 医生端忽略老年模式标志，始终强制标准模式；下次刷新仍为医生端标准模式 | 无用户提示，医生端UI不受localStorage中elderMode标志影响 |
| TTS自动朗读时TTS服务不可用 | 自动朗读失败静默处理，不弹出错误打断流程；用户仍可手动点击播放按钮重试 | 控制台日志记录TTS失败原因；不显示错误toast以免打扰老年用户 |

## 5. 界面/API规格

### 5.1 界面

**组件1：角色切换按钮（侧边栏底部）**
- 页面/组件名：RoleSwitcher
- 入口：左侧边栏底部区域，主题切换按钮上方
- 关键元素：
  - 图标+文字按钮：当前角色图标（医生/患者图标）+"切换到医生端"/"切换到患者端"
  - 按钮尺寸：老年模式≥48px高度；标准模式≥32px
  - 点击后弹出确认对话框
- 状态要求：
  - 主流程：点击→弹出确认框→确认→刷新切换
  - 加载中：切换过程中显示简单加载状态
  - 账号模式（P1）：账号登录后切换按钮隐藏，根据账号类型自动确定角色
  - 错误：localStorage写入失败时静默降级，仍可切换但刷新后恢复默认

**组件2：老年模式开关（患者端侧边栏底部）**
- 页面/组件名：ElderModeToggle
- 入口：左侧边栏底部，仅患者端显示，医生端隐藏
- 关键元素：
  - 开关控件 + 文字标签"老年模式"（≥18px）
  - 辅助说明文字（可选）："大字体、大按钮、语音优先"
  - 默认状态：开启（on）
- 状态要求：
  - 主流程：点击开关→即时切换UI样式→状态保存localStorage
  - 开启状态：开关高亮主色，所有老年模式样式生效
  - 关闭状态：开关灰色，切换为标准样式（14px字号，32px按钮）
  - 医生端：组件完全不渲染

**组件3：确认对话框（关键操作二次确认）**
- 页面/组件名：ConfirmDialog（老年模式专用样式）
- 入口：删除对话、清空历史、导出报告、提交评估、切换角色等关键操作触发
- 关键元素：
  - 遮罩层：半透明黑色背景
  - 模态框：圆角大卡片，最小宽度320px
  - 标题：≥20px粗体
  - 正文：≥18px常规字重，行高1.8
  - 按钮区：两个按钮横向排列，间距≥16px，每个≥48px高度
    - 取消按钮：次要样式（白底边框）
    - 确认按钮：主色/危险色样式
  - 可通过Esc键或点击遮罩关闭（等同于取消）
- 状态要求：
  - 主流程：触发→显示对话框→取消/确认→执行对应动作
  - 老年模式：所有尺寸放大，文字更大，按钮更大
  - 标准模式：使用ShadCN Dialog标准尺寸

**组件4：患者端超大麦克风按钮**
- 页面/组件名：VoiceMicButton（老年模式变体）
- 入口：主聊天区输入框最右侧
- 关键元素：
  - 老年模式：圆形按钮≥64×64px，主色背景，白色麦克风图标
  - 标准模式：按钮≥40×40px，与其他功能按钮统一尺寸
  - 录音中状态：红色脉冲动画（慢速0.7x）+ 录音时长大字显示
- 状态要求：
  - 老年模式默认显示麦克风（即使输入框有文字也优先显示麦克风？不——有文字时显示发送按钮，但麦克风仍保持大尺寸）
  - 录音状态反馈清晰可见

**组件5：患者列表（医生端特有）**
- 页面/组件名：PatientList
- 入口：医生端左侧边栏或主区快捷入口
- 关键元素：
  - 患者列表项：姓名、年龄、性别、最近就诊时间、风险等级标签
  - 点击患者项进入该患者对话上下文
  - 使用模拟数据（MVP阶段）：预置5-10个示例患者数据
- 状态要求：
  - 加载中：骨架屏
  - 为空：显示"暂无患者"提示
  - 错误：显示"加载失败"+重试按钮（MVP模拟数据不涉及网络错误）

**医生端特有功能区标识：**
- CGA工作区入口：医生端侧边栏/主区可见，患者端隐藏
- 处方编辑/审核按钮：处方预览面板显示"编辑"、"审核通过"、"驳回"按钮（仅医生端）

### 5.2 API端点（无后端API，纯前端状态管理）

本模块为纯前端UI/状态管理模块，MVP阶段无后端API。所有状态通过localStorage持久化。

（二阶段账号模式后将增加角色相关API，但MVP不涉及）

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 角色切换（页面刷新）首屏加载<3秒；老年模式开关切换样式生效<100ms（无刷新）；确认对话框弹出/关闭动画流畅≥30fps；CSS变量切换无样式闪烁（FOUC） |
| 安全 | 访客模式下不收集任何个人身份信息；localStorage仅存储UI偏好和会话文本；不存储语音原始数据；老年模式高对比度配色确保视力障碍用户可读；自动TTS播放不突然发出大音量（渐入或默认适中音量） |
| 可靠性 | localStorage写入失败时静默降级到内存状态；CSS变量有兜底值（浏览器不支持CSS变量时仍可读）；即使JavaScript部分加载失败，基础大字高对比度样式通过内联CSS保证可用 |
| 可观测性 | 关键操作控制台日志：角色切换（from→to）、老年模式开关切换（on/off）、确认对话框展示/确认/取消、localStorage读写状态；错误日志包含角色和模式信息便于排查 |
| 兼容性 | Chrome/Edge/Safari最新2个版本；支持CSS变量的浏览器（所有现代浏览器）；移动端平板端老年模式按钮可点击区域充足；键盘Tab导航在老年模式下焦点环清晰可见 |
| 无障碍 | WCAG AAA级对比度（≥7:1）；所有交互元素有ARIA标签；支持屏幕阅读器（语义化HTML，role属性正确）；键盘导航完整；尊重prefers-reduced-motion系统设置；老年模式下不使用颜色作为唯一信息传达方式（图标+文字双重视觉） |
| 适老化 | 严格遵循PRD铁律：禁止<16px字体、禁止<44px按钮（老年模式≥48px）、禁止低对比度配色、禁止纯图标无文字标签按钮、禁止自动播放声音除用户主动触发的TTS（老年模式默认开启的自动TTS视为用户主动设置） |

## 8. 不做什么（Out of Scope）

- 账号登录/注册系统（P1功能，MVP访客模式即可）
- 根据账号自动识别角色（P1，账号登录后隐藏切换按钮）
- 自定义字号调节（仅提供标准/老年两档，不提供字号滑块无级调节）
- 色盲模式/色弱模式专门配色（P1+，高对比度模式已覆盖部分需求）
- 屏幕阅读器专项优化 beyond 基础ARIA标签（P1持续优化）
- 医生端自定义布局/字号调节（医生端始终标准模式，不提供主题自定义）
- 角色切换时的会话数据迁移/隔离（MVP刷新页面但localStorage会话保留，二阶段账号模式再做数据隔离）
- 患者列表真实数据加载（MVP使用前端硬编码模拟数据，二阶段从后端API获取）
- 权限控制（MVP无登录，任何人可切换到任意端；二阶段通过账号权限控制）
- 老年模式下的语音引导/操作提示语音播报（除AI回复自动TTS外，不做全局操作语音导航）
