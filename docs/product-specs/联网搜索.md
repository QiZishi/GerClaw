# 联网搜索 — 产品规格

> 模块：联网搜索 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

GerClaw平台的医学信息实时检索模块，集成AnySearch（主）和Tavily（兜底）双搜索API，由LLM自动判断是否触发搜索（最新医学信息、药物查询、疾病指南、循证证据等场景），搜索结果以独立卡片式展示在对话流中，AI回复正文通过[1][2]上标角标引用来源，点击角标在右侧动态面板展开引用详情预览，搜索内容添加隔离标记防止prompt injection，确保医疗建议的循证可溯源性，MVP阶段纯前端实现。

> **无独立页面声明**（依据用户决策与 `gerclaw设计要求.md` §4.2.3⑤）：联网搜索**不设独立页面**，搜索结果以**独立卡片**形式嵌入消息流，对齐 §4.2.3「联网搜索结果」可视化项（每条结果：标题+来源 favicon+摘要+链接）；AI 正文用 [1][2] 角标引用，点击链接可在右侧面板预览，全程在对话流内完成，不跳转独立搜索页。

> **二阶段实现说明（0018）**：生产事实源已迁入 FastAPI `SearchModule`。AnySearch `/mcp` JSON-RPC 永远优先，Tavily 仅作降级；AgentScope `web_search` 的 SSE `tool_result.results` 直接驱动既有卡片。Provider key 不再进入浏览器，CGA workflow 从 Toolkit 层移除搜索工具。下文标注为“MVP 纯前端”的历史实现仅说明原型，不再是生产架构。

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户发送医学时效性问题（如"2024年老年糖尿病诊疗指南有什么更新？"）
  ↓
LLM接收消息 → 系统prompt引导判断需要联网搜索
  ↓
LLM返回tool_call → 调用web_search工具，传入query参数
  ↓
前端展示工具调用卡片：🔍 联网搜索 "2024年老年糖尿病诊疗指南有什么更新？" → 运行中旋转动画
  ↓
前端调用SearchModule.search(query)
  ↓
优先调用AnySearch API → 设置10秒超时
  ↓
AnySearch返回结果 → 解析为统一SearchResult[]格式
  ↓
（若AnySearch失败）自动重试1次 → 仍失败则调用Tavily API兜底
  ↓
搜索结果返回 → 工具卡片状态变为完成✓，显示耗时
  ↓
前端在对话流中渲染搜索结果卡片组（序号[1][2]...+标题+摘要+来源）
  ↓
搜索结果传入LLM（带隔离标记）→ LLM继续生成回复
  ↓
AI流式输出 → 引用搜索结果处自动插入[1][2]可点击角标
  ↓
用户点击[1]角标 → 右侧面板自动展开，显示第1条引用详情
  ↓
用户可在面板中切换查看其他引用，或点击"在新标签页打开"查看原网页
  ↓
用户点击面板关闭按钮 → 右侧面板收起
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| AnySearch API超时（10秒无响应） | 自动重试1次AnySearch，仍超时则降级调用Tavily | 工具卡片显示状态变化：AnySearch超时→重试→切换Tavily→完成；用户看到搜索正常完成，无需额外提示（控制台记录日志） |
| AnySearch返回错误（4xx/5xx/Key无效） | 不重试，直接降级调用Tavily | 工具卡片显示切换状态，搜索结果正常返回；控制台warn级别日志记录AnySearch错误 |
| AnySearch和Tavily都失败 | 终止搜索流程，返回空结果给LLM，LLM基于已有知识回答并说明无法获取最新信息 | 工具卡片显示失败✗状态；AI回复末尾提示"抱歉，当前无法连接到搜索服务，以下回答基于已有知识，可能不包含最新信息，建议咨询医生" |
| Tavily API Key未配置 | AnySearch失败后不再尝试Tavily，直接返回搜索失败 | 工具卡片失败；AI提示搜索服务不可用；控制台warn日志记录Tavily Key缺失 |
| 两个API Key都未配置 | 禁用搜索工具，LLM不触发tool_call | 搜索工具不出现；AI对时效性问题明确告知"无法联网获取最新信息，建议咨询专业医生" |
| 搜索返回0条结果 | 返回空结果给LLM，工具卡片显示"未找到相关结果" | 工具卡片显示完成但提示"未找到相关结果"；AI告知未搜索到相关信息，建议换关键词或咨询医生 |
| 搜索结果包含恶意内容/prompt injection | 隔离标记包裹搜索内容，系统prompt明确LLM区分搜索结果和指令，LLM应忽略搜索结果中的指令类内容 | 用户无感知；安全层过滤明显有害内容；控制台记录可疑内容 |
| CGA评估进行中触发搜索意图 | 系统上下文标记CGA模式，LLM不调用搜索工具 | 用户无感知；CGA评估按量表流程进行，不受外部搜索干扰 |

## 5. 界面/API规格

### 5.1 界面

**组件1：联网搜索工具调用卡片**
- 位置：对话流中，AI消息气泡之前
- 关键元素：
  - 🔍 搜索图标 + "联网搜索"文字
  - 搜索关键词展示（引号包裹）
  - 状态徽章：运行中（旋转动画）→ 完成✓（绿色）→ 失败✗（红色）
  - 耗时显示（完成后显示，如"耗时1.2s"）
  - 展开/折叠按钮：默认折叠，点击展开显示API调用参数和原始返回
  - 失败状态显示"重试"按钮
- 状态要求：
  - 主流程：运行中旋转→完成✓+耗时→展开显示详情
  - 加载中：搜索关键词后显示旋转加载动画
  - 为空：0条结果时显示"未找到相关结果"
  - 错误：红色✗+错误信息+重试按钮

**组件2：搜索结果卡片组**
- 位置：工具调用卡片之后，AI消息气泡之前
- 关键元素：
  - 标题栏："🔍 搜索结果 (n条)"
  - 每条结果卡片：
    - 左上角序号蓝色圆角badge [1] [2]...
    - 来源favicon（16×16px，自动获取域名favicon，失败则显示默认地球图标）
    - 标题（蓝色可点击链接，hover下划线）
    - 来源名称 + 发布时间（灰色小字，如"国家卫健委 · 2024-03"）
    - 摘要文本（2-3行，超出省略）
  - 底部："以上结果来自联网搜索，AI将基于这些信息生成回答"提示
- 状态要求：
  - 主流程：搜索完成后依次渲染结果卡片
  - 加载中：显示骨架屏占位
  - 为空：显示"未找到相关结果"空状态
  - 错误：不渲染结果卡片，仅显示工具失败状态

**组件3：正文引用角标**
- 位置：AI回复正文中，引用搜索结果的文字右上角
- 关键元素：
  - 蓝色上标文字 [1] [2]...
  - 可点击，鼠标hover显示pointer手势和tooltip（引用标题预览）
  - 点击后高亮对应角标，同时右侧面板展开
- 状态要求：
  - 主流程：AI流式输出时随文本渲染角标
  - 激活状态：点击后角标变为深蓝色加粗，右侧面板显示对应引用
  - 未找到引用：角标正常显示但点击提示"引用详情不可用"

**组件4：右侧面板-引用详情**
- 入口：点击正文角标自动展开
- 宽度：320-400px（默认360px，可拖拽320-500px）
- 关键元素：
  - 顶部标题栏："📚 引用来源" + 关闭按钮×
  - 引用列表（左侧或顶部）：垂直列表，每项显示[序号] + 标题截断，当前选中项高亮
  - 详情区域：
    - 大标题（可点击跳转原网页）
    - 来源标签（来源域名favicon + 名称 + 发布时间，S/A/B/C级来源有对应颜色标记）
    - 来源等级badge（S级金色/A级绿色/B级蓝色/C级灰色，MVP可简化为只显示来源名称）
    - 完整摘要文本（可滚动查看）
    - 原文链接按钮："🔗 在新标签页打开"
    - 内容提取按钮（可选MVP不做）："📄 提取全文内容"
- 状态要求：
  - 主流程：点击角标→面板展开→显示对应引用详情→可切换其他引用
  - 加载中：切换引用时详情区域显示骨架屏
  - 为空：无引用时面板不展开
  - 错误：提取全文失败时显示错误提示

## 6. 数据模型

```typescript
// 搜索来源（统一结果结构）
interface SearchResult {
  id: string;           // 唯一ID（如"search-0"、"search-1"）
  index: number;        // 序号（1-based，对应[1][2]角标）
  title: string;        // 结果标题
  snippet: string;      // 摘要文本
  url: string;          // 原文链接
  source: string;       // 来源名称（如"国家卫健委"、"丁香园"）
  favicon?: string;     // 来源favicon URL（可选，前端可自动从域名获取）
  publishedDate?: string; // 发布日期（ISO格式或显示文本，如"2024-03-15"）
  sourceLevel?: 'S' | 'A' | 'B' | 'C' | 'D'; // 来源权威等级（MVP可选）
  searchEngine: 'anysearch' | 'tavily'; // 实际使用的搜索引擎
  rawResult?: any;      // 原始API返回（调试用，不展示给用户）
}

// 搜索状态
type SearchState = 'idle' | 'searching' | 'retrying' | 'switching' | 'completed' | 'failed';

// 搜索引擎配置（从环境变量读取）
interface SearchConfig {
  anysearchApiKey?: string;  // process.env.ANYSEARCH_API_KEY（可选，匿名可用）
  tavilyApiKey?: string;     // process.env.TAVILY_API_KEY
  anysearchEndpoint: string; // 默认 "https://api.anysearch.com/mcp"
  tavilyEndpoint: string;    // 默认 "https://api.tavily.com/search"
  timeoutMs: number;         // 超时时间，默认10000ms（10秒）
  maxRetries: number;        // 重试次数，默认1次
  defaultMaxResults: number; // 默认返回结果数，默认5
}

// 搜索工具调用状态（用于工具卡片可视化）
interface SearchToolCall {
  id: string;
  toolName: 'web_search';
  query: string;
  state: SearchState;
  startTime: number;
  endTime?: number;
  durationMs?: number;
  engine?: 'anysearch' | 'tavily';
  error?: string;
  results?: SearchResult[];
}

// 引用在右侧面板的展示状态
interface CitationPanelState {
  isOpen: boolean;
  messageId: string | null;  // 所属AI消息ID
  activeCitationIndex: number | null; // 当前选中的引用序号
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 搜索API总响应时间（含降级重试）≤15秒；AnySearch单调用超时10秒，Tavily单调用超时10秒；搜索结果卡片渲染无卡顿；角标点击到右侧面板展开延迟≤300ms |
| 安全 | API Key通过部署平台环境变量注入，不硬编码到git仓库；搜索结果内容用隔离标记包裹防止prompt injection；系统prompt明确防注入规则；不发送患者PHI（姓名/身份证号/手机号）到搜索API；HTTPS-only（部署平台保证） |
| 可靠性 | 双搜索引擎自动降级（AnySearch→Tavily）；单引擎超时重试1次；两个引擎都失败时优雅降级（AI基于已有知识回答并提示）；CGA评估模式下强制禁用搜索；网络中断时友好提示不崩溃 |
| 可观测性 | 控制台日志：搜索开始（query/engine）、搜索成功（engine/结果数/耗时）、搜索失败（engine/错误信息/重试/降级）、降级切换触发、角标点击、面板展开/关闭；错误日志包含query、engine、error message；统计搜索成功率、降级率 |
| 兼容性 | Chrome/Edge/Safari最新2个版本；桌面端和平板端完整支持；fetch API和AbortController（超时控制）支持；CORS跨域请求正常（搜索API需支持CORS或通过部署平台代理） |
| 适老化 | 角标点击区域≥24×24px（比普通文字大，方便老人点击）；引用详情面板正文字体≥16px（老年模式≥18px）；按钮尺寸≥44px（老年模式≥48px）；来源名称和时间高对比度显示；避免仅用颜色区分来源等级，加文字/图标辅助 |
| 医疗合规 | 所有搜索引用可追溯（标题+URL+来源）；AI回答末尾必须有免责声明；不得将搜索到的广告/推广内容作为医学建议（D级来源过滤）；紧急症状（胸痛/呼吸困难/卒中）搜索后仍需提示立即就医 |

## 8. 不做什么（Out of Scope）

- 本地医学知识库RAG检索（P1功能，Qdrant+BGE-M3向量检索，MVP用联网搜索替代）
- 搜索结果权威等级自动分级标注（S/A/B/C级，MVP简化为只显示来源名称，不做等级颜色标记，二阶段完善）
- 网页全文内容提取并让LLM基于长文回答（MVP只用snippet摘要，extract接口预留但不接入主流程）
- 搜索历史记录本地存储和复用（MVP每次查询实时搜索，不做缓存，二阶段加Redis缓存）
- 多语言搜索（MVP默认中文搜索，不做翻译后搜索）
- 搜索结果的可信度AI二次审核（二阶段加医疗安全审核中间件）
- 图片搜索/学术搜索（PubMed等专业数据库，MVP通用health领域搜索即可）
- 二阶段FastAPI后端代理搜索请求（MVP纯前端直连API，二阶段迁移Python后端代理隐藏Key）
- 搜索结果排序/重排（BGE-Reranker，二阶段实现，MVP直接用API返回顺序）
- 用户手动触发搜索按钮（MVP完全由LLM自动判断触发，不提供手动搜索入口）
