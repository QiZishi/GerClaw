# 工具调用可视化 — 产品规格

> 模块：工具调用可视化 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

AI工具调用过程的可视化展示模块，将模型调用外部工具（联网搜索、文件解析、用药审查、处方生成、CGA评估等）的过程展示为独立卡片，包含工具图标、名称、状态徽章（运行中/完成/失败三态），运行中显示旋转动画和实时耗时计时，完成显示绿色对勾和总耗时，失败显示红色叉号和错误信息；卡片可展开查看调用参数、返回结果（JSON格式化高亮）和耗时详情，交互体验对齐Trae Work，嵌入消息流中在AI回复之间/之前显示，让用户清晰了解AI调用了哪些工具、结果如何。

> **§4.2.3 七项执行过程可视化统筹**：依据 `gerclaw设计要求.md` §4.2.3，前端需可视化展示七项执行过程，本模块为统筹模块，七项分布如下：
> ① **思维链**（Chain of Thought）— 详见 [思维链展示.md](./思维链展示.md)，可折叠思考过程区块
> ② **工具调用** — 本文件核心，独立卡片三态展示
> ③ **子智能体加载** — 折叠树形结构，展示主智能体→子智能体调用链（见下方 F-011）
> ④ **决策过程** — ReAct 循环的 Thought/Action/Observation 垂直时间线（见下方 F-012）
> ⑤ **联网搜索结果** — 详见 [联网搜索.md](./联网搜索.md)，独立卡片式展示
> ⑥ **文档解析状态** — 详见 [文档解析.md](./文档解析.md)，文件标签+工具卡片双重反馈
> ⑦ **流式文本输出** — 打字机效果（见下方 F-013）

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户发送消息，AI开始流式回复
  ↓
收到第一个tool_call delta → 插入ToolCallBlock卡片
  ├─ 显示工具图标+工具名称+"运行中"旋转动画
  ├─ 开始计时，实时更新耗时（每100ms）
  └─ 卡片默认折叠
  ↓
（可能同时有多个工具并行/串行调用，每个工具独立卡片）
  ↓
收到该工具的tool_result delta
  ├─ 成功结果：状态→完成，绿色✓，停止计时，显示总耗时
  └─ 错误结果：状态→失败，红色✗，停止计时，显示错误信息，显示重试按钮
  ↓
用户点击卡片任意位置（除重试按钮外）→ 展开详情
  ├─ 显示"调用参数"区域：格式化JSON参数
  ├─ 显示"返回结果"区域：格式化JSON/Markdown结果，复制按钮
  └─ 显示"耗时"：开始时间→结束时间，Xms
  ↓
用户再次点击卡片 → 折叠详情
  ↓
流结束 → 所有工具卡片状态固定
  ↓
（失败工具）用户点击重试按钮
  ↓
卡片重置为运行中状态，重新调用工具，重新计时
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 工具调用超时（超过30s无结果） | 标记为失败状态，显示超时错误 | 卡片红色✗，错误信息："工具调用超时，请重试"，显示重试按钮 |
| 工具返回错误结果（API错误、参数错误） | 标记为失败，显示错误消息 | 卡片红色✗，显示具体错误信息（如"搜索API返回429限频"），显示重试按钮 |
| 工具结果JSON格式异常无法解析 | 标记为完成，但结果区域显示原始文本 | 卡片绿色✓（调用成功），结果区域显示"返回格式异常，原始内容："+原始文本 |
| 用户在工具运行中切换会话/刷新 | 工具状态标记为中断，卡片保留 | 显示为失败状态，错误："已中断"，提供重试按钮 |
| 重试再次失败 | 保持失败状态，允许继续重试 | 错误信息更新，重试按钮仍可用 |
| 并行多个工具调用 | 每个工具独立卡片，独立计时，独立状态 | 多个卡片按调用顺序排列，各自独立运行互不影响 |
| 老年模式适配 | 卡片点击区域≥48px，状态图标更大更明显 | 无额外提示，自动适配大字号高对比度 |

## 5. 界面/API规格

### 5.1 界面

- **页面/组件名**：ToolCallBlock（工具调用卡片）
- **入口**：AI消息气泡内的消息流中，当有tool_call/tool_result时自动渲染，位于ThinkingBlock之后、TextBlock之前
- **关键元素**：
  - **容器**：圆角卡片，浅灰色/白色背景，边框分隔，整体宽度同消息气泡宽度
  - **折叠状态（运行中）**：
    - 左侧：工具类型图标（如🔍）
    - 中间：工具名称（如"联网搜索"）+ 灰色小字"运行中..." + 实时计时（蓝色，如"1.5s"）
    - 右侧：旋转spinner图标
    - 整体高度约40-44px，hover背景变灰提示可点击
  - **折叠状态（完成）**：
    - 左侧：工具图标
    - 中间：工具名称 + 灰色小字总耗时（如"2.3s"）
    - 右侧：绿色✓对勾图标 + 展开箭头↓
  - **折叠状态（失败）**：
    - 左侧：工具图标
    - 中间：工具名称 + 红色错误文字摘要
    - 右侧：红色✗叉号图标 + "重试"文字按钮 + 展开箭头↓
  - **展开状态**：
    - 头部同折叠状态，箭头变为↑
    - 详情区域（白底/稍深背景）：
      - "调用参数"小节：```json 代码块样式，格式化高亮参数，右上角复制按钮
      - "返回结果"小节：格式化JSON高亮（如为Markdown可选择渲染/原始切换），右上角复制按钮
      - "耗时信息"小节：灰色小字，"开始时间：HH:mm:ss · 结束时间：HH:mm:ss · 耗时：Xms"
  - **动画**：状态切换淡入淡出，展开/折叠平滑高度动画（200ms），运行中spinner持续旋转，计时数字平滑更新
- **工具图标映射**：
  | 工具类型 | 图标 | 中文名称 |
  |---------|------|---------|
  | web_search / anysearch / tavily | 🔍 Search | 联网搜索 |
  | file_parse / mineru | 📄 File | 文件解析 |
  | asr / stt / voice | 🎤 Mic | 语音识别 |
  | tts | 🔊 Speaker | 语音合成 |
  | prescription_generate | 💊 Pill | 处方生成 |
  | cga_assessment | 📋 Clipboard | CGA评估 |
  | drug_review / ddi / beers | 💊 Pill | 用药审查 |
  | 其他/unknown | 🔧 Wrench | 工具调用 |
- **状态要求**：
  - **主流程**：运行中（旋转+计时）→完成（绿勾+耗时）→可展开查看详情
  - **加载中**：spinner旋转，实时计时更新
  - **为空**：无工具调用时不渲染任何卡片
  - **错误**：红叉+错误信息+重试按钮，可展开查看错误详情

### 5.2 API端点（如适用）

本模块为纯前端UI组件，通过Vercel AI SDK接收tool_call和tool_result流式delta。

**Vercel AI SDK对接**：
- 工具调用开始：SSE `tool_call` delta 包含 `toolCallId`、`toolName`、`args`（参数JSON）
- 工具调用结果：SSE `tool_result` delta 包含 `toolCallId`、`result`（返回内容）
- 支持多工具并行/串行：通过`toolCallId`关联调用和结果，每个工具独立卡片

## 6. 数据模型

```typescript
// 工具调用状态
type ToolCallStatus = 'running' | 'completed' | 'failed';

// 工具类型（用于图标映射）
type ToolType =
  | 'web_search'
  | 'file_parse'
  | 'asr'
  | 'tts'
  | 'prescription_generate'
  | 'cga_assessment'
  | 'drug_review'
  | 'unknown';

// 工具调用内容块（继承自通用对话ContentBlock）
interface ToolCallBlock extends ContentBlock {
  type: 'tool_call';
  toolCallId: string;       // 工具调用唯一ID（SDK提供）
  toolName: string;         // 工具原始名称（如"anysearch"）
  toolType: ToolType;       // 归一化工具类型（用于图标）
  displayName: string;      // 显示中文名（如"联网搜索"）
  toolIcon?: string;        // 图标名称/组件
  status: ToolCallStatus;
  params: Record<string, unknown>;  // 调用参数
  result?: unknown;         // 返回结果（成功时）
  error?: string;           // 错误信息（失败时）
  startTime: number;        // 开始时间戳（ms）
  endTime?: number;         // 结束时间戳（ms）
  durationMs?: number;      // 总耗时（ms）
  isCollapsed: boolean;     // 是否折叠详情
  isStreaming: boolean;     // 是否仍在执行中
}

// 工具图标和名称映射配置
const TOOL_DISPLAY_MAP: Record<string, { icon: string; name: string; type: ToolType }> = {
  anysearch: { icon: 'search', name: '联网搜索', type: 'web_search' },
  tavily_search: { icon: 'search', name: '联网搜索', type: 'web_search' },
  mineru_parse: { icon: 'file-text', name: '文件解析', type: 'file_parse' },
  asr_transcribe: { icon: 'mic', name: '语音识别', type: 'asr' },
  generate_prescription: { icon: 'pill', name: '处方生成', type: 'prescription_generate' },
  cga_assess: { icon: 'clipboard', name: 'CGA评估', type: 'cga_assessment' },
  ddi_check: { icon: 'pill', name: '用药审查', type: 'drug_review' },
  beers_check: { icon: 'pill', name: 'Beers审查', type: 'drug_review' },
};
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 计时更新流畅（每100ms更新不卡顿）；展开/折叠动画60fps；JSON高亮渲染大JSON（<100KB）时无明显延迟 |
| 视觉 | 三态颜色明确区分（蓝/运行中、绿/完成、红/失败）；对齐Trae Work设计风格；卡片样式简洁不突兀 |
| 可访问性 | 支持键盘Tab聚焦、Enter展开折叠；ARIA标签标注状态（"工具联网搜索运行中，已耗时1.5秒"）；状态图标不仅靠颜色区分（同时有✓/✗/spinner形状差异） |
| 可靠性 | toolCallId正确关联调用和结果，不会张冠李戴；并行工具状态独立互不影响；重试功能正确重新触发调用 |
| 适老化 | 卡片高度≥44px，状态图标≥18px，颜色对比度满足WCAG AA；老年模式下字号放大、图标更醒目 |
| 可观测性 | 工具执行成功/失败/耗时时记录console日志；失败时记录错误详情便于排查 |

## 8. 不做什么（Out of Scope）

- 工具调用编辑/修改参数后重新运行（MVP仅支持相同参数重试）
- 工具调用历史统计/成功率展示
- 工具调用链可视化（多工具依赖关系树，二阶段AgentScope多智能体后考虑）
- 工具调用结果直接渲染为UI卡片（搜索结果卡片是联网搜索模块独立负责，本组件只负责通用工具调用过程展示）
- 工具调用取消功能（MVP无法中止已发送的工具调用请求）
- 工具收藏/常用工具快捷入口
- 工具调用详情中的Markdown富渲染（仅JSON格式化高亮，结果内容由上层模块决定是否额外渲染）
- 工具调用耗时的性能分析/瀑布图
- 导出工具调用记录日志
