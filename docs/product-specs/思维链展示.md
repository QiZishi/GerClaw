# 思维链展示 — 产品规格

> 模块：思维链展示 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

模型思考过程的可视化展示模块，将大语言模型的reasoning/thinking内容默认折叠为低对比度区块，流式输出时显示"思考中..."动画，思考完成后可点击展开/折叠查看完整推理过程；折叠状态采用灰色小字号样式并截断前2行，展开后完整展示思维链文本；思维链内容不导出、不复制，仅作为可解释性辅助展示，不干扰正文阅读体验。

## 2. 用户故事

| 编号 | 作为... | 我想要... | 以便于... |
|------|---------|----------|----------|
| US-001 | 老年科医生 | 看到AI的思考过程（默认折叠） | 了解AI是如何一步步分析病情的，增加对AI建议的信任感，必要时验证推理逻辑 |
| US-002 | 老年患者 | 思考过程默认收起 | 不被复杂的推理文字干扰，直接看最终建议就好 |
| US-003 | 所有用户 | AI思考时看到"思考中..."提示 | 知道AI还在工作，没有卡住，耐心等待回复 |
| US-004 | 医生用户 | 点击可以展开查看完整思考过程 | 深入了解AI的诊断推理链条，用于教学或验证AI循证逻辑 |
| US-005 | 老年患者 | 导出对话/处方时不包含思考过程 | 打印给医生看的报告干净整洁，只有最终结论 |

| F-002 | 默认折叠状态 | 思考完成后区块默认折叠，显示前2行文本+省略号 | P0 | 折叠时：浅灰色背景（#f5f5f5或opacity-60）、小字号（比正文小2px）、显示前2行文字末尾加"..."省略号、右侧显示"展开"按钮和向下箭头 |

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户发送消息，AI开始流式回复
  ↓
收到第一个thinking delta → 渲染ThinkingBlock
  ├─ 显示旋转spinner + "思考中..."
  ├─ 流式追加thinking内容到隐藏区域
  └─ 折叠状态仅显示前2行（实时更新）
  ↓
收到text delta或tool_call delta（思考结束）
  ↓
spinner消失，"思考中..."替换为"思考过程"
  ↓
（默认折叠）用户可点击区块展开
  ├─ 点击 → 平滑展开动画 → 显示完整思考内容 + "收起"按钮
  └─ 再次点击 → 平滑折叠回2行截断状态
  ↓
对话流结束 → ThinkingBlock状态固定为isStreaming: false
  ↓
用户后续可随时点击展开/折叠查看
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 模型未返回thinking内容（无reasoning字段） | 不渲染ThinkingBlock，直接显示正文 | 无感知，界面干净 |
| thinking内容为空字符串 | 不渲染区块 | 无感知 |
| 流式过程中thinking中断（网络错误） | 保留已接收的思考内容，标记为不完整 | 无特殊提示，与正文错误处理一致（正文显示重试按钮） |
| thinking内容超长（>10000字） | 正常渲染，展开后内部滚动或全显示 | 无提示，可正常滚动查看（实际模型thinking不会这么长） |
| 老年模式下样式适配 | 字号相应放大，按钮点击区域≥48px | 无额外提示，自动适配老年模式大字体 |
| 用户复制消息时 | 复制内容过滤掉thinking文本 | 剪贴板中只有正文内容 |

## 5. 界面/API规格

### 5.1 界面

- **页面/组件名**：ThinkingBlock（思维链折叠块）
- **入口**：AI消息气泡内部，位于正文之前，当有thinking内容时自动渲染
- **关键元素**：
  - **容器**：圆角矩形块，浅灰色背景，左侧有3px灰色竖线标识
  - **折叠状态**：
    - 顶部：💭"思考过程" 文字（小字号灰色）+ 右侧"展开"文字+↓箭头
    - 中部：前2行文本截断，末尾加省略号，小字号灰色
    - 整体高度约48-60px，点击热区覆盖整个区块
  - **思考中状态**：
    - 顶部：旋转spinner图标 + "思考中..." 文字（灰色）
    - 不显示截断文本（或显示已接收内容的前2行半透明）
  - **展开状态**：
    - 顶部：💭"思考过程" + 右侧"收起"文字+↑箭头
    - 中部：完整思维链文本，带padding-left缩进（12-16px），等宽或常规字体，行高1.6，小字号但可读
    - 底部：无额外内容，整体边框包裹
  - **展开/折叠动画**：CSS transition，高度平滑过渡（max-height动画），时长200-300ms
- **状态要求**：
  - **主流程**：思考中→折叠→点击展开→点击收起
  - **加载中**："思考中..."spinner动画持续旋转
  - **为空**：无thinking内容时不渲染任何元素
  - **错误**：不单独显示错误，与消息整体错误状态一致

### 5.2 API端点（如适用）

本模块为纯前端展示组件，无自有API，通过Vercel AI SDK接收流式delta。

**Vercel AI SDK对接**：
- 从SSE流中解析thinking/reasoning增量
- OpenAI协议兼容：从`choices[0].delta.reasoning_content`字段获取（Qwen等模型支持）
- Anthropic协议：从`thinking`类型的content block获取
- 兜底：解析正文内容中`<think>...</think>`标签包裹的部分（本地模型或prompt要求模型输出标签时）

## 6. 数据模型

```typescript
// 思维链内容块（继承自通用对话ContentBlock）
interface ThinkingBlock extends ContentBlock {
  type: 'thinking';
  content: string;           // 完整思维链文本
  isCollapsed: boolean;      // 当前是否折叠（默认true）
  isStreaming: boolean;      // 是否仍在流式接收中
  startedAt: number;         // 思考开始时间戳（用于统计耗时，可选）
  endedAt?: number;          // 思考结束时间戳
}

// 思考过程展示常量
const THINKING_BLOCK_CONFIG = {
  collapsedMaxLines: 2,           // 折叠时显示行数
  collapsedOpacity: 0.6,          // 折叠时文字透明度
  fontSize: 'text-sm',            // 小字号（14px）
  bgColor: 'bg-muted/50',         // 浅灰背景
  borderColor: 'border-l-2 border-l-muted-foreground/30',  // 左侧竖线
  animationDurationMs: 250,       // 展开/折叠动画时长
};
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 展开/折叠动画流畅（60fps），不卡顿；流式追加thinking内容时不导致整个消息列表重渲染卡顿 |
| 视觉 | 低对比度不抢正文注意力，同时可点击区域清晰明确；对齐Trae Work的思维链展示风格 |
| 可访问性 | 折叠区块有ARIA属性（aria-expanded、aria-controls），支持键盘Tab聚焦、Enter/Space切换展开折叠 |
| 可靠性 | thinking内容解析容错：三种解析方式（字段/block/标签）任一识别成功即可显示；解析失败不影响正文显示 |
| 适老化 | 老年模式下字号同步放大，点击区域≥48px高，文字对比度满足WCAG AA标准（即使是低对比度也要可读） |
| 隐私安全 | 思维链内容不进入导出文件、不进入复制剪贴板、不保存到其他持久化（仅存消息内存/localStorage作为消息一部分，但导出时过滤） |

## 8. 不做什么（Out of Scope）

- 思维链编辑/修改（用户不能编辑AI的思考过程）
- 思维链单独导出/复制（只能随整体消息，但整体导出时排除）
- 思维链搜索/高亮关键词
- 多段思维链展示（MVP假设每条AI消息最多一个thinking块）
- 思维链反馈（点赞/点踩思考过程，二阶段考虑）
- 思考过程中显示token计数/耗时统计（可在二阶段作为调试功能添加）
- 关闭思维链功能的用户设置（MVP默认显示折叠，不提供关闭开关，因为折叠状态干扰很小）
- 思维链内容的Markdown渲染（纯文本展示即可，不需要Markdown格式，因为是内部推理）
