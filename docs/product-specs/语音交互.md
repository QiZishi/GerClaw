# 语音交互 — 产品规格

> 模块：语音交互 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

GerClaw平台面向老年用户的语音优先交互核心模块，集成Mimo ASR（语音识别）和TTS（语音合成）能力，提供点击式录音、实时波形可视化、转写预览的语音输入，以及AI回复语音朗读、播放进度控制的语音输出，降低老年患者数字鸿沟，实现"说得出、听得见"的适老化医疗交互体验，MVP阶段纯前端实现（MediaRecorder录音+Web Audio API播放PCM16流）。

## 4. 交互流程

### 4.1 主流程（Happy Path）

**语音输入流程：**

```
用户点击输入框右侧麦克风按钮
  ↓
检查麦克风权限 → 未授权则弹出权限请求
  ↓
权限允许 → 按钮变为"停止录音"状态（红色高亮）
  ↓
开始MediaRecorder录音 → 显示实时波形动画 + 录音时长计时
  ↓
（可选）实时流式ASR转写预览，边说边显示文字
  ↓
用户再次点击按钮 → 停止录音
  ↓
音频编码为WAV base64 → 调用mimo-v2.5-asr接口
  ↓
SSE流式接收识别结果 → 显示最终转写文本到输入框
  ↓
自动发送文本消息到对话
```

**语音朗读流程：**

```
用户悬停AI消息 → 显示播放按钮（喇叭图标）
  ↓
用户点击播放按钮
  ↓
调用mimo-v2.5-tts接口（audio.format=pcm16, voice=冰糖, stream=true）
  ↓
SSE逐chunk接收delta.audio.data → base64解码为PCM16
  ↓
Web Audio API AudioBufferSourceNode流式播放
  ↓
按钮变为暂停图标 → 显示播放进度
  ↓
播放完成 → 按钮恢复播放状态
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 麦克风权限被拒绝 | 停止录音流程，不发起ASR请求 | 显示提示："无法访问麦克风，请在浏览器设置中允许麦克风权限后重试"，提供"了解如何开启"帮助 |
| 录音时长过短（<1秒） | 不发起ASR请求，提示用户重新录音 | 显示："录音时间太短，请重新说话" |
| ASR API调用失败/超时（30s） | 停止转写loading，显示文本输入框可手动输入 | 显示："语音识别失败，请重试或改用文字输入" |
| ASR返回空结果 | 不发送空消息，提示用户重新录音或手动输入 | 显示："未能识别到语音内容，请靠近麦克风重新说话" |
| TTS API调用失败/超时（30s） | 停止播放流程，恢复播放按钮 | 显示："语音合成失败，请稍后重试" |
| TTS播放中断（网络错误） | 停止播放，保留已播放进度，可点击重试 | 按钮恢复播放状态，允许用户重新点击播放 |
| MIMO_API_KEY未配置 | 禁用语音相关按钮，控制台输出错误日志 | 语音按钮置灰不可点击，hover提示"语音服务未配置" |
| 浏览器不支持MediaRecorder/Web Audio API | 禁用语音按钮 | 语音按钮置灰，提示"当前浏览器不支持语音功能，请使用最新版Chrome/Edge/Safari" |

## 5. 界面/API规格

### 5.1 界面

**组件1：麦克风按钮（输入框右侧）**
- 入口：主聊天区输入框右侧
- 关键元素：
  - 默认状态：🎤 麦克风图标（语音输入模式）
  - 录音中状态：🔴 停止图标（红色高亮）+ 录音时长（如"0:05"）
  - 识别中状态：加载旋转图标 + "识别中..."
  - 波形可视化组件（录音时显示在输入框上方或按钮周围）
  - 实时转写预览区域（录音时临时显示在输入框内或上方浮层）
- 状态要求：
  - 主流程：点击→录音中→点击停止→识别中→文本填入输入框→自动发送
  - 加载中：识别过程中按钮禁用，显示加载动画
  - 为空：无录音时显示默认麦克风图标
  - 错误：识别失败时toast提示，按钮恢复默认状态

**组件2：消息语音播放按钮（AI消息右上角）**
- 入口：每条AI消息悬停时显示操作按钮组中的播放按钮
- 关键元素：
  - 未播放：🔊 喇叭图标
  - 播放中：⏸️ 暂停图标 + 简单进度条/进度指示
  - 老年模式：按钮更大（≥48px），默认自动播放
- 状态要求：
  - 主流程：点击→流式合成→播放中→播放完成
  - 加载中：合成中显示加载旋转
  - 暂停：点击暂停后停在当前位置，再次点击继续
  - 错误：合成失败时toast提示，按钮恢复播放状态

**ASR语音识别接口：**

| POST | https://api.xiaomimimo.com/v1/chat/completions | mimo-v2.5-asr语音转文字 | Header `api-key: $MIMO_API_KEY` | model: "mimo-v2.5-asr"<br>stream: true<br>messages[0].role: "user"<br>messages[0].content[0].type: "input_audio"<br>messages[0].content[0].input_audio.data: "data:audio/wav;base64,{base64音频}"<br>asr_options.language: "auto" | SSE流式，OpenAI chat completion格式<br>每chunk choices[0].delta.content为增量识别文本，最终chunk为完整文本 |

| 方法 | 路径 | 描述 | 认证 | 请求参数 | 响应格式 |

## 6. 数据模型

```typescript
// 录音状态
type RecordingState = 'idle' | 'requesting_permission' | 'recording' | 'recognizing' | 'error';

// TTS播放状态
type TTSPlaybackState = 'idle' | 'synthesizing' | 'playing' | 'paused' | 'error';

// 语音配置（从环境变量读取）
interface VoiceConfig {
  mimoApiKey: string;
  asrModel: string;      // 默认 "mimo-v2.5-asr"
  ttsModel: string;      // 默认 "mimo-v2.5-tts"
  ttsVoice: string;      // 默认 "冰糖"
  asrLanguage: string;   // 默认 "auto"
  ttsStylePrompt: string; // 默认 "用温柔体贴的语调，语速适中，像在关心一位老人的健康状况"
}

// 录音会话数据
interface RecordingSession {
  id: string;
  startTime: number;
  mediaRecorder: MediaRecorder | null;
  audioChunks: Blob[];
  duration: number;       // 秒
  waveformData: number[]; // 实时音量数据用于可视化
  interimTranscript: string; // 实时转写预览
  finalTranscript: string;   // 最终转写结果
}

// TTS播放会话数据
interface TTSPlaybackSession {
  messageId: string;
  audioContext: AudioContext | null;
  pcmChunks: Float32Array[]; // 已接收的PCM数据块
  currentSource: AudioBufferSourceNode | null;
  startTime: number;
  pausedAt: number;      // 暂停位置（秒）
  totalDuration: number; // 总时长（估算）
  isStreaming: boolean;  // 是否还在接收TTS流
}
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | ASR识别延迟<3秒（录音停止到出最终结果）；TTS首包延迟<1秒（点击播放到听到声音）；流式播放无卡顿（chunk缓冲≥2个后开始播放）；录音波形动画帧率≥30fps |
| 安全 | API Key通过部署平台环境变量注入，不硬编码到源码；前端不存储用户语音原始数据（仅内存中处理，播放/识别完成后释放）；HTTPS-only环境（浏览器API要求） |
| 可靠性 | ASR/TTS超时设置30秒，失败自动重试最多2次；网络中断时停止录音/播放并给出友好提示；浏览器不支持时优雅降级（禁用语音功能，保留文本输入） |
| 可观测性 | 关键操作控制台日志：录音开始/停止、ASR开始/完成/失败、TTS开始/首包/完成/失败、音频chunk接收统计；错误日志包含错误类型和message |
| 兼容性 | Chrome/Edge/Safari最新2个版本；桌面端和平板端完整支持；手机端基础可用；支持MediaRecorder API和Web Audio API的浏览器 |
| 适老化 | 录音按钮≥48px点击区域；波形动画颜色高对比度；转写预览字体≥16px；老年模式自动播放开关可配置；录音状态有视觉+（可选）听觉反馈 |

## 8. 不做什么（Out of Scope）

- 语音唤醒（关键词唤醒，P1功能，本地SenseVoice实现）
- VAD语音活动检测（自动检测说话结束停止录音，MVP采用点击式手动停止）
- 多轮实时全双工语音对话（边说边播，MVP为半双工：录音→转写→AI回复→TTS播放）
- 音色切换（MVP固定音色"冰糖"）
- 语音消息本地存储保存（MVP不保存录音原始音频，仅保存转写文本）
- 语速/音量/音调调节（MVP固定风格参数）
- 方言优化（MVP使用language=auto自动检测，不做特定方言定制）
- 二阶段后端代理（MVP纯前端直连Mimo API，二阶段迁移FastAPI后端代理）
