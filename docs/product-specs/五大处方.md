# 五大处方 — 产品规格

> 模块：五大处方 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

五大处方模块基于患者健康信息（病史、用药、检查检验、评估结果等），通过AI智能体生成结构化的五类个体化处方：药物处方、运动处方、营养处方、心理处方、康复处方。每类处方包含目标、具体建议、注意事项、循证来源四个核心部分。处方生成后在右侧动态面板预览，支持医生端审核编辑，支持导出为PDF/Markdown/DOCX格式。系统通过Function Calling/JSON Mode强制结构化输出，确保处方格式规范、内容可溯源。

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
患者端流程：
1. 用户点击输入框💊处方按钮 / 欢迎页"五大处方生成"卡片 / 对话说"帮我开处方"
2. 右侧面板自动展开，显示处方生成向导和信息收集表单
3. 系统在聊天区友好问候，询问基本信息或引导上传病历
   - 老年模式：语音自动问候，提示可以说话或打字
4. 用户通过对话描述病情 / 上传病历文件 / 确认已有信息
5. 右侧面板实时更新：
   - 已收集字段填入表单
   - 缺失字段黄色高亮
   - 顶部进度条更新（如"已收集 8/15 项"）
6. 系统逐一询问缺失的关键字段（每次只问一个问题）
7. 用户回答（语音/文本），系统确认并更新进度
8. 信息收集完整后，系统提示"信息已完整，正在为您生成处方..."
9. 主聊天区展示执行过程可视化：
   - 思考过程（折叠）
   - 联网检索循证依据（工具调用卡片）
   - 结构化生成（步骤时间线）
   - 四重校验（校验通过/失败状态）
10. 生成完成：
    - 右侧面板展示完整处方报告（带目录导航）
    - 主聊天区给出关键结论摘要
    - "查看完整报告"按钮高亮
    - 老年模式：自动语音播报关键结论
11. 用户可：
    - 滚动/点击目录浏览处方
    - 点击🔊语音朗读
    - 点击导出按钮选择PDF/Markdown/DOCX导出
    - 对内容有疑问可继续对话询问
```

```
医生端流程：
1. 医生选择患者（或当前对话上下文已有患者）
2. 点击处方按钮启动生成
3. 信息收集阶段医生可直接在右侧面板表单手动输入/修改患者信息
4. 信息确认后点击"生成处方"按钮
5. 系统生成处方初稿，右侧面板展示报告（带"待审核"标签）
6. 医生可直接在右侧面板编辑各处方内容（富文本编辑）
7. 药物处方自动显示用药风险提示（DDI/Beers/剂量）
8. 医生核对循证来源标注
9. 修改完成后点击"审核通过并应用"
10. 处方状态更新为"已审核"，可导出或发送给患者
11. 可在患者档案中查看历史处方
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 患者信息严重缺失（关键字段缺失>50%） | 提示信息不足，继续通过对话收集 | "为了给您更准确的建议，我还需要了解一些信息：您目前的血压是多少？" |
| 信息收集超过10轮仍不完整 | 基于已有信息生成处方，标注信息不完整提示 | "根据您目前提供的信息，我先为您生成初步建议，部分内容可能不够准确，建议补充更多信息后重新生成。" |
| LLM JSON输出格式错误 | 自动重试（最多2次），仍失败则降级为非结构化输出+提示 | "处方生成遇到一点小问题，正在重试..." / 重试失败："生成遇到问题，请点击重新生成" |
| 四重校验发现内容问题 | 高亮问题项，给出修正建议，支持重新生成 | "处方中发现以下需要注意的问题：[问题列表]，建议修改后使用" |
| 循证来源检索失败 | 处方正常生成，相关建议标注"临床常规建议，建议医生确认" | "部分建议未能检索到最新循证依据，请医生审核确认" |
| 药物相互作用高风险 | 醒目标识风险，强制提示医生审核 | "⚠️ 检测到高风险药物相互作用：[详细说明]，请医生务必核对确认" |
| 导出失败 | 提示导出失败，支持重试 | "导出失败，请检查网络后重试" |
| 患者在生成过程中中断 | 保存当前进度，支持从断点继续 | "已为您保存当前进度，下次可以继续生成处方" |
| 检测到急重症症状（胸痛/呼吸困难/意识障碍等） | 立即停止处方生成，高风险提示立即就医 | "⚠️ 您描述的症状可能比较严重，请立即停止自我药疗，马上到医院急诊就诊或拨打120！" |

## 5. 界面/API规格

### 5.1 界面

**5.1.1 信息收集阶段（右侧面板）**

- 面板宽度：380-420px（可拖拽320-500px）
- 顶部：标题"五大处方生成" + 进度条 + 关闭按钮
- 表单区域（滚动）：
  - 基本信息组：姓名、性别、年龄、身高、体重、BMI
  - 疾病史组：已确诊疾病列表
  - 用药组：当前用药列表
  - 检查组：关键检验检查结果（血压、血糖、血脂、肝肾功能等）
  - 生活方式组：吸烟、饮酒、运动习惯、饮食习惯
  - 心理评估组：睡眠、情绪状态（可关联CGA结果自动填充）
- 字段状态：已填正常，待填灰色，缺失高亮黄色，有问题红色
- 底部："信息完整，生成处方"按钮（信息收集完整前置灰）

**5.1.2 报告预览阶段（右侧面板）**

- 顶部：
  - 报告标题"五大处方报告"
  - 患者姓名+生成时间
  - 审核状态标签（AI生成待审核/已审核）
  - 导出按钮组（PDF/Markdown/DOCX图标）
  - 医生端额外显示：编辑按钮、审核通过按钮
  - 关闭按钮
- 目录导航（可折叠）：
  - 患者信息摘要
  - 健康诊断
  - 1. 药物处方
  - 2. 运动处方
  - 3. 营养处方
  - 4. 心理处方
  - 5. 康复处方
  - 引用来源
  - 免责声明
- 报告内容区（滚动）：Markdown渲染，支持代码块、表格、列表、引用角标
- 底部：免责声明固定区域

**5.1.3 医生端编辑态**

- 每个处方章节右上角显示"编辑"按钮
- 点击后进入编辑模式：富文本编辑器（支持加粗、列表、缩进）
- 编辑状态有"保存"和"取消"按钮
- 药物处方编辑时额外显示用药风险检查面板
- 整体保存后退出编辑态，更新预览

### 5.2 API端点（二阶段，MVP为前端直连LLM）

| 方法 | 路径 | 描述 | 认证 | 请求参数 | 响应格式 |
|------|------|------|------|---------|---------|
| POST | /api/prescription/generate | 生成五大处方 | 是（医生/患者） | patient_info: object（患者完整信息） | SSE流式：progress事件 + prescription_ready事件（含结构化JSON） |
| POST | /api/prescription/validate | 校验处方报告 | 是（医生） | report: PrescriptionReport | { valid: boolean; errors: ValidationError[]; warnings: ValidationWarning[] } |
| POST | /api/prescription/export | 导出处方 | 是 | report_id: string; format: 'pdf' \| 'markdown' \| 'docx' | 文件流（Content-Type对应文件MIME类型） |
| GET | /api/prescription/:id | 获取处方详情 | 是 | id: string（处方ID） | PrescriptionReport（完整结构） |
| PUT | /api/prescription/:id | 更新处方（医生编辑） | 是（医生） | id: string; updates: Partial<PrescriptionReport> | PrescriptionReport（更新后完整结构） |
| POST | /api/prescription/:id/approve | 医生审核通过 | 是（医生） | id: string | { success: boolean; approved_at: string; approved_by: string } |
| GET | /api/patients/:id/prescriptions | 获取患者历史处方列表 | 是（医生） | patient_id: string | { list: PrescriptionSummary[]; total: number } |

## 6. 数据模型

```typescript
// 循证来源
interface EvidenceSource {
  id: string;          // 引用ID，如 "1", "2"
  title: string;       // 来源标题
  source: string;      // 来源（如"中国老年高血压管理指南2023"、"柳叶刀2024"）
  summary?: string;    // 摘要
  url?: string;        // 原文链接（如有）
}

// 用药注意事项
interface MedicationCaveat {
  drug_name: string;   // 关联药品名称（可空表示通用）
  content: string;     // 注意事项内容
  monitoring?: string; // 需要监测的指标
  urgency: 'info' | 'warning' | 'danger'; // 提示级别
}

// 单个药品
interface MedicationItem {
  brand_name?: string;      // 商品名（如"施慧达"、"立普妥"）
  generic_name: string;     // 通用名（如"苯磺酸左氨氯地平片"）
  specification: string;    // 规格（如"2.5mg"、"20mg"、"75mg*7片"）
  dosage: string;           // 用法用量（如"每天一次，每次1片"、"每晚睡前1片"）
  notes?: string;           // 用药备注
  evidence_ids: string[];   // 循证来源ID列表
}

// 运动阶段
interface ExercisePhase {
  name: string;         // 阶段名称（热身/有氧训练/拉伸）
  duration: string;     // 时长（如"5分钟"、"20分钟"、"5-10分钟"）
  intensity: string;    // 强度描述（如"慢走3-4km/h"、"心率72-75次/分"）
  instructions: string; // 具体做法
  progression?: string; // 进阶方案
}

// 营养评估
interface NutritionAssessment {
  height_cm: number;        // 身高cm
  weight_kg: number;        // 体重kg
  bmi: number;              // BMI
  bee_kcal?: number;        // 基础能量消耗kcal
  smm_kg?: number;          // 骨骼肌量kg
  bf_percent?: number;      // 体脂百分比
  smi?: number;             // 骨骼肌质量指数
  mna_sf_score?: number;    // MNA-SF评分
  nutrition_status: string; // 营养状况结论
  dietary_problems?: string; // 饮食问题描述
}

// 食物推荐量
interface FoodIntake {
  food_group: string;   // 食物类别（主食/蔬菜/牛奶/水果/蛋类/瘦肉鱼禽豆制品/食用油/食盐等）
  daily_amount: string; // 每日推荐量
  notes?: string;       // 备注（如"1/3为粗杂粮"、"以深色蔬菜为主"）
}

// 营养补充建议
interface NutritionSupplement {
  name: string;         // 补充剂名称（如"乳清蛋白粉"、"益生菌"、"可溶性膳食纤维粉"）
  dosage: string;       // 用量
  condition?: string;   // 适用条件（如"膳食蛋白质无法达标时"）
}

// 心理评估
interface PsychologicalAssessment {
  sleep_status?: string;      // 睡眠情况
  phq9_score?: number;        // PHQ-9抑郁评分
  anxiety_score?: number;     // 焦虑评分
  diagnosis: string;          // 心理状态结论（如"焦虑抑郁状态"、"睡眠障碍"）
  current_meds?: string[];    // 当前心理相关用药
}

// 处方基类
interface BasePrescription {
  type: 'medication' | 'exercise' | 'nutrition' | 'psychological' | 'rehabilitation';
  title: string;        // 处方标题（如"康复处方"）
  goal: string;         // 目标
  recommendations: string | string[]; // 具体建议（文本或条目化）
  precautions: string[]; // 注意事项列表
  evidence_ids: string[]; // 循证来源ID列表（该章节涉及的主要来源）

// 康复处方
interface RehabilitationPrescription extends BasePrescription {
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 处方生成首token响应<3s；完整处方生成（含检索）<30s；PDF/Markdown导出<2s；DOCX导出<5s |
| 可靠性 | LLM JSON格式错误自动重试2次；循证检索失败降级为"待医生确认"标注；导出失败支持重试 |
| 安全性 | 所有处方底部强制显示免责声明；药物高风险交互醒目提示；禁止给出确定性诊断结论；急重症症状优先提示就医；医生审核前标注"AI生成待审核" |
| 可观测性 | 记录处方生成耗时、校验通过率、导出成功率、编辑修改率；医生编辑的字段记录原始值和修改后的值用于改进prompt |
| 适老化 | 患者端正文≥18px，标题≥20px，按钮≥48px点击区域；老年模式支持全语音交互；关键信息语音播报；医学术语配通俗解释 |
| 合规性 | 循证来源可追溯；不编造医学文献；药物剂量建议符合老年人用药指南；二阶段需符合医疗数据隐私法规 |

## 8. 不做什么（Out of Scope）

- 不直接对接医院HIS/EMR系统（二阶段预留接口）
- 不实现电子处方流转/药房对接（仅作建议，不具处方权）
- 不做处方执行追踪和随访提醒（P1阶段）
- 不做基于可穿戴设备数据的实时处方调整
- 不实现患者用药依从性追踪（P1）
- 不做多语言（仅中文）
- 不实现处方打印模板自定义（固定专业模板）
