# 技能管理 — 产品规格

> 模块：技能管理 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

GerClaw平台的技能扩展模块，提供预置医疗技能（用药提醒、随访问卷生成、风险评估、健康宣教）和用户自定义技能的完整管理能力。通过将技能指令注入系统prompt改变AI对话行为，支持医生/患者加载特定技能完成专项任务（如用药提醒设置、随访问卷生成）。MVP阶段纯前端实现，自定义技能存储于localStorage，支持 skill.md 文件上传（含 .zip 压缩包自动解压，§3.2.1）、自然语言描述生成（LLM 辅助，§3.2.1）、在线文本编辑、CRUD管理、会话级加载/卸载，以及基础安全校验防止技能篡改系统角色设定。

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户进入聊天界面
  ↓
点击左侧边栏"技能管理"入口
  ↓
右侧动态面板展开，展示技能列表
  ├─ 预置技能区：用药提醒、随访问卷生成、风险评估、健康宣教
  └─ 自定义技能区：用户创建的技能列表（初始为空）
  ↓
用户点击某个技能的"加载"按钮
  ↓
技能加入当前会话，输入框上方显示技能标签
  ↓
用户发送消息
  ↓
系统组装prompt：系统角色设定 + 已加载技能内容 + 对话历史 + 用户消息
  ↓
AI按照技能指令约束回复
  ↓
（用户可随时点击其他技能加载，或点击标签×移除技能）
  ↓
用户点击"新建技能"按钮
  ↓
选择"文本编辑"或"上传skill.md"
  ├─ 文本编辑：填写名称、描述，编辑器输入Markdown内容→预览→安全检查→保存
  └─ 上传文件：选择.md文件→解析内容→确认/编辑信息→安全检查→保存
  ↓
新技能出现在自定义技能列表，持久化到localStorage
```

### 4.2 技能加载与对话流程

```
用户加载技能A和技能B
  ↓
输入框上方显示 [技能A ×] [技能B ×] 标签
  ↓
用户发送消息
  ↓
系统prompt构建顺序：
  1. 基础系统角色设定（医生/患者，医疗安全红线）
  2. 免责声明
  3. ===== 已加载技能 =====
     【技能A名称】：技能A描述和指令
     【技能B名称】：技能B描述和指令
  4. 对话历史
  5. 用户当前消息
  ↓
调用LLM获取流式回复
  ↓
AI回复受技能指令约束
```

### 4.3 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| skill.md文件解析失败（格式错误/内容为空） | 阻止保存，停留在编辑界面 | 提示："文件解析失败，请检查文件格式是否为有效的Markdown" |
| 技能名称为空或重复 | 阻止保存，高亮名称输入框 | 提示："请输入技能名称" / "技能名称已存在，请使用其他名称" |
| 技能内容超过长度限制（>10000字） | 阻止保存 | 提示："技能内容过长，请精简至10000字以内" |
| 安全检查检测到危险关键词 | 弹出警告对话框，列出检测到的风险内容 | 警告："检测到技能内容中包含可能修改系统角色的指令，这可能导致医疗安全风险。是否仍要保存？"，提供"强制保存"和"返回编辑"选项 |
| localStorage存储已满 | 阻止创建新技能 | 提示："本地存储空间不足，请删除不需要的技能后再试" |
| 上传非.md/.skill文件 | 阻止上传 | 提示："仅支持上传.md或.skill格式的Markdown文件" |
| 技能加载数量过多（>10个） | 给出警告但允许加载 | 提示："已加载较多技能，可能会影响AI回复质量，建议只加载当前需要的技能" |
| 删除正在使用的技能（当前会话已加载） | 二次确认，确认后删除并自动从会话中移除 | 确认对话框："该技能已在当前会话加载，删除后将自动从会话中移除，是否继续？" |
| 预置技能尝试编辑/删除 | 操作按钮隐藏或禁用 | 预置技能卡片不显示编辑/删除按钮，hover提示"预置技能不可修改" |

## 5. 界面/API规格

### 5.1 界面

#### 5.1.1 技能管理面板（右侧动态面板）

- **页面/组件名**：SkillManagerPanel
- **入口**：左侧边栏底部"技能管理"按钮，或输入框左侧⚡技能按钮
- **关键元素**：
  - **面板顶部**：标题"技能管理"、关闭按钮、"新建技能"主按钮
  - **分类标签页**：全部 / 预置技能 / 我的技能
  - **搜索框**：按名称/描述搜索技能
  - **技能卡片列表**（网格或列表布局）：
    - 技能图标（预置技能用固定图标，自定义技能用默认图标）
    - 技能名称
    - 技能描述（截断显示，hover显示完整）
    - 类型标签：`预置`（蓝色）/ `自定义`（绿色）
    - 状态标签：`已启用`/`已禁用`
    - 操作按钮组：
      - 预置技能：[加载] / [已加载 ✓] / [查看详情]
      - 自定义技能：[加载] / [已加载 ✓] / [编辑] / [删除] / 启用开关
  - **空状态**（自定义技能为空时）："还没有自定义技能，点击上方按钮创建你的第一个技能" + 引导图示

- **状态要求**：
  - **主流程**：技能列表正常加载→点击加载→标签显示→对话生效
  - **加载中**：技能保存时按钮显示loading状态，上传文件时显示进度条
  - **为空**：自定义技能区显示空状态引导
  - **错误**：保存失败显示错误提示条，文件解析失败在上传区显示错误信息

#### 5.1.2 技能编辑器模态框

- **页面/组件名**：SkillEditorModal
- **入口**：点击"新建技能"或自定义技能的"编辑"按钮
- **关键元素**：
  - 标题："新建技能" / "编辑技能：xxx"
  - 技能名称输入框（必填）
  - 技能描述输入框（必填，简短描述，用于列表展示）
  - Tab切换：「编辑」/「预览」
  - Markdown文本编辑器（支持基础语法高亮）
  - 底部按钮：[取消] [保存]
- **安全检查弹窗**：检测到风险时弹出，显示风险详情，提供[返回编辑] [强制保存]

#### 5.1.3 输入框标签区域

- **组件名**：LoadedSkillTags
- **位置**：输入框正上方
- **关键元素**：
  - 已加载技能标签（胶囊样式）
  - 预置技能标签：蓝色边框+蓝色图标
  - 自定义技能标签：绿色边框+绿色图标
  - 每个标签右侧有×关闭按钮
  - 标签过多时支持横向滚动

#### 5.1.4 技能快速选择（输入框⚡按钮）

- **组件名**：SkillQuickPicker
- **入口**：输入框左侧⚡技能按钮
- **交互**：点击后弹出下拉菜单，展示常用/已启用技能列表，点击直接加载，支持搜索
- **菜单项**：技能名称、描述（简短）、加载状态（已加载的显示✓）

### 5.2 API端点（如适用）

> MVP阶段为纯前端静态导出，技能管理所有逻辑在浏览器端实现，无后端API。技能数据通过localStorage持久化。LLM调用由前端直连模型API，技能内容通过prompt注入传递。

| 方法 | 路径 | 描述 | 认证 | 请求参数 | 响应格式 |
|------|------|------|------|---------|---------|
| - | localStorage | 自定义技能CRUD持久化 | 无 | key: `gerclaw_skills_v1` | JSON: `{skills: CustomSkill[], updatedAt: number}` |
| - | localStorage | 会话已加载技能存储 | 无 | key: `gerclaw_session_skills_{conversationId}` | JSON: `{loadedSkillIds: string[]}` |
| POST | `{MODEL_URL}/chat/completions` | 带技能的对话（技能注入system prompt） | Bearer Token | messages中system角色包含技能指令 | SSE流（同通用对话模块） |

## 6. 数据模型

```typescript
// 技能类型
type SkillType = 'builtin' | 'custom';

// 技能基础接口
interface BaseSkill {
  id: string;
  name: string;
  description: string;
  content: string; // Markdown格式的技能指令内容
  type: SkillType;
  enabled: boolean;
  createdAt: number;
  updatedAt: number;
}

// 预置技能（代码内置，不可修改）
interface BuiltinSkill extends BaseSkill {
  type: 'builtin';
  icon: string; // 预置图标名
  category: 'medication' | 'followup' | 'assessment' | 'education';
}

// 自定义技能（用户创建，localStorage存储）
interface CustomSkill extends BaseSkill {
  type: 'custom';
  source: 'text' | 'file'; // 创建来源：文本编辑 / 文件上传
  originalFilename?: string; // 上传文件的原始文件名
}

// 预置技能ID枚举
type BuiltinSkillId = 
  | 'medication-reminder'
  | 'followup-questionnaire'
  | 'risk-assessment'
  | 'health-education';

// 会话级已加载技能状态（每个会话独立）
interface SessionSkillState {
  conversationId: string;
  loadedSkillIds: string[]; // 已加载技能ID列表（有序）
  updatedAt: number;
}

// localStorage存储结构
interface SkillStorage {
  version: string; // 数据版本号，用于迁移
  customSkills: Record<string, CustomSkill>;
  customSkillOrder: string[]; // 自定义技能ID按创建时间排序
}

// 技能安全检查结果
interface SkillSafetyCheckResult {
  isSafe: boolean;
  warnings: SkillSafetyWarning[];
}

interface SkillSafetyWarning {
  type: 'role_override' | 'ignore_instructions' | 'prompt_injection';
  keyword: string;
  position: number; // 在内容中的位置
  message: string;
}

// 技能在system prompt中的注入格式
const SKILL_PROMPT_TEMPLATE = `
===== 已加载技能（你必须严格遵循以下技能的指令执行任务）=====

{% for skill in loadedSkills %}
【{{skill.name}}】
{{skill.content}}

{% endfor %}
===== 技能指令结束 =====
`;
```

### 6.1 预置技能内容定义

```typescript
// 预置技能初始数据
const BUILTIN_SKILLS: BuiltinSkill[] = [
  {
    id: 'medication-reminder',
    name: '用药提醒',
    description: '帮助老年患者设置用药提醒，包括用药时间、剂量、注意事项，生成易懂的用药指导',
    content: `# 用药提醒技能
## 任务目标
帮助老年患者设置清晰的用药提醒，确保按时按量正确服药。

## 执行步骤
1. 询问患者需要提醒的药物名称、每次剂量、服用频率（如每日几次、饭前/饭后）
2. 询问是否有特殊注意事项（如不能与某些食物同服、服用后需观察反应）
3. 生成结构化的用药时间表，用大字和简单语言表述
4. 额外提醒漏服处理方法：
   - 想起时立即补服（若已接近下次服药时间则跳过，不可加倍服用）
   - 记录漏服情况，复诊时告知医生
5. 附上重要安全提醒：
   - 不可自行增减剂量或停药
   - 出现不适及时就医
   - 药物需放在儿童接触不到的地方

## 输出格式
使用清晰的分点列表，关键信息（服药时间、剂量）加粗显示。语言要通俗易懂，避免专业术语。`,
    type: 'builtin',
    enabled: true,
    icon: 'pill',
    category: 'medication',
    createdAt: 0,
    updatedAt: 0,
  },
  {
    id: 'followup-questionnaire',
    name: '随访问卷生成',
    description: '根据患者病情和治疗方案，生成规范化的随访问卷，用于出院后随访或慢病管理',
    content: `# 随访问卷生成技能
## 任务目标
为医生患者生成标准化的随访问卷，用于跟踪治疗效果、病情变化、不良反应。

## 执行步骤
1. 了解患者基本情况：疾病诊断、目前治疗方案、出院/就诊时间
2. 确定随访类型：术后随访/慢病随访/用药后随访
3. 生成随访问卷，包含以下维度：
   - 一般情况：饮食、睡眠、精神状态、体重变化
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 技能列表加载<100ms（localStorage读取）；技能保存<50ms；技能切换（加载/卸载）即时生效，标签更新无延迟；安全检查<10ms（正则匹配）；Markdown预览实时渲染无卡顿 |
| 安全 | 技能内容注入system prompt前必须经过安全检查；安全检查正则覆盖常见prompt injection模式（"忽略之前的指令"、"你现在是"、"忘记你是"、"扮演"等）；预置技能内容硬编码在代码中不可被用户修改；localStorage数据加简单校验防止篡改；技能内容不得覆盖系统角色设定的医疗安全红线；禁止技能内容包含确定性诊断指令 |
| 可靠性 | localStorage操作异常时（如隐私模式禁用）降级到内存存储并提示用户；技能数据加载失败时使用默认预置技能列表，不影响基础功能；单个技能解析失败不影响其他技能加载 |
| 可观测性 | 关键操作记录console日志：技能创建/编辑/删除/加载/卸载、安全检查警告、存储异常；安全检查触发时记录warn级别日志，包含触发关键词和技能ID；存储异常记录error级别日志 |
| 兼容性 | Chrome/Edge/Safari最新2个版本；localStorage作为持久化方案，降级方案为内存存储；Markdown编辑器基础功能在所有支持浏览器正常工作 |
| 适老化 | 技能管理面板在患者端老年模式下：技能卡片更大（便于点击），按钮≥48px，字体≥18px，操作流程简化，关键操作有明确提示，危险操作二次确认弹窗文字大而清晰 |
| 可扩展性 | 数据结构预留version字段支持后续数据迁移；技能分类可扩展；安全检查规则可配置；为二阶段后端存储预留接口抽象层 |

## 8. 不做什么（Out of Scope）

- 自然语言生成技能（用自然语言描述，AI自动生成skill.md）—— P1功能，二阶段实现
- 技能市场/技能分享/技能导入导出（MVP仅支持本地创建，二阶段实现云端技能库）
- 技能版本管理和回滚（MVP保存即覆盖，无历史版本）
- 技能执行审批/HITL（高风险技能医生审批）—— P1功能，二阶段权限引擎实现
- 技能使用统计和审计日志（MVP仅console日志，无持久化审计）
- 技能分组/标签分类（MVP仅分预置/自定义两类，自定义标签二阶段实现）
- 技能热更新（预置技能代码内置，版本更新随前端发布）
- 技能评分/评价系统（二阶段实现）
- 后端API同步技能（MVP纯前端localStorage，二阶段迁移PostgreSQL+后端API）
- 技能参数化配置（MVP技能指令是静态Markdown，不支持动态参数填入）
- 压缩包形式上传技能（仅支持单个.md文件上传，包含资源文件的压缩包二阶段实现）
- 技能执行沙箱/隔离（技能仅影响prompt，无独立代码执行能力，AgentScope工具调用二阶段实现）
