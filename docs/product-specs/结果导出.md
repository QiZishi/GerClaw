# 结果导出 — 产品规格

> 模块：结果导出 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

纯前端多格式导出模块，支持将对话记录、五大处方、CGA评估报告导出为PDF、Markdown、DOCX三种格式，导出内容包含标题、生成日期、医疗免责声明和完整正文，文件名自动按"GerClaw_类型_日期"格式生成，所有导出逻辑在浏览器端完成，无需后端服务。

## 4. 交互流程

### 4.1 主流程（Happy Path）

```
用户在右侧面板查看处方/报告/对话
  ↓
点击面板顶部"导出"按钮
  ↓
弹出格式选择下拉菜单：PDF / Markdown / DOCX
  ↓
用户点击选择一种格式
  ↓
导出按钮显示加载状态（旋转图标+"导出中..."）
  ↓
前端执行导出逻辑：
  ├─ Markdown：直接拼接文本模板 → Blob
  ├─ DOCX：使用docx.js构建Document → Packer.toBlob
  └─ PDF：html2canvas渲染DOM → jsPDF添加图片 → 分页处理
  ↓
生成Blob URL → 创建<a>标签触发download
  ↓
浏览器弹出文件保存对话框（或自动下载到下载目录）
  ↓
导出按钮恢复正常状态，显示"导出成功"toast提示
```

### 4.2 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 内容为空（无处方/无报告/空对话） | 禁用导出按钮，点击无反应 | 按钮灰显，hover提示"暂无可导出内容" |
| PDF导出时html2canvas渲染失败 | 降级为纯文本PDF（无样式）或提示失败 | 显示"PDF导出失败，建议使用Markdown格式" |
| 导出内容过大（>10MB） | 提示内容较大，可能需要等待 | Toast提示"内容较大，导出中请稍候..." |
| 浏览器Blob/下载API不支持 | 提示用户使用现代浏览器 | 显示"您的浏览器不支持文件导出，请升级到最新版Chrome/Edge/Safari" |
| DOCX生成时中文字体缺失 | 使用系统默认字体，确保内容完整 | 无提示，文档可正常打开（字体可能随系统变化） |
| 用户在导出过程中切换面板 | 继续在后台完成导出，不中断 | 导出完成后仍显示成功提示，文件正常下载 |

## 5. 界面/API规格

### 5.1 界面

- **页面/组件名**：导出按钮与格式选择菜单
- **入口**：右侧动态面板（五大处方预览面板、CGA报告面板、对话详情面板）顶部操作栏
- **关键元素**：
  - 导出按钮：📥导出图标+文字"导出"，位于面板标题右侧，与其他操作按钮（刷新/关闭）并排
  - 格式选择菜单：点击后弹出下拉菜单，三个选项各带图标：
    - PDF：📄 PDF文档（适合打印分享）
    - Markdown：📝 Markdown（适合编辑存档）
    - DOCX：📘 Word文档（适合Word编辑）
  - 加载状态：按钮内显示旋转spinner，文字变为"导出中..."
  - 成功提示：右下角toast显示"导出成功，文件已开始下载"
- **状态要求**：
  - **主流程**：按钮可点击→弹出菜单→选择格式→加载→下载成功
  - **加载中**：按钮禁用，显示旋转动画，菜单关闭
  - **为空**：面板无内容时按钮灰显禁用
  - **错误**：导出失败显示红色toast错误提示，按钮恢复可点击

### 5.2 API端点（如适用）

本模块为纯前端实现，无后端API调用。所有导出逻辑使用浏览器API和前端库完成。

## 6. 数据模型

```typescript
// 导出内容类型
type ExportContentType = 'conversation' | 'prescription' | 'cga_report';

// 导出格式
type ExportFormat = 'pdf' | 'markdown' | 'docx';

// 导出配置
interface ExportConfig {
  contentType: ExportContentType;
  title: string;           // 导出标题，如"五大处方"、"CGA评估报告"
  date: string;            // 生成日期，格式YYYY-MM-DD
  content: string | HTMLElement;  // 导出内容：Markdown直接传字符串，PDF传DOM元素
  disclaimer?: string;     // 自定义免责声明，默认使用标准医疗免责声明
}

// 导出文件名生成规则
// GerClaw_{title}_{date}.{ext}
// 示例：
// - GerClaw_五大处方_2026-07-04.pdf
// - GerClaw_CGA评估报告_2026-07-04.docx
// - GerClaw_对话记录_2026-07-04.md
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | Markdown导出<100ms；DOCX导出<500ms（10页内容）；PDF导出<3s（10页内容）；所有导出操作不阻塞主线程超过100ms |
| 安全 | 纯前端操作，数据不上传服务器；导出文件不包含思维链、工具调用等内部信息；免责声明必须包含在内不可移除 |
| 可靠性 | 三种导出格式独立实现，一种失败不影响其他两种；导出失败时给出明确提示，不静默失败 |
| 兼容性 | Chrome/Edge/Safari最新2个版本；PDF导出确保中文字符正确显示（使用html2canvas截图方案避免字体问题）；DOCX兼容Word 2016+和WPS |
| 可观测性 | 导出成功/失败记录console日志；导出耗时统计（info级别）；导出失败记录错误详情（error级别） |

## 8. 不做什么（Out of Scope）

- 云存储/云端导出（不保存导出文件到服务器，纯本地下载）
- 批量导出多个会话/报告（MVP仅支持单个内容导出）
- 自定义导出模板（使用统一固定模板，不支持用户自定义页眉页脚）
- 导出为图片（PNG/JPG）格式
- 打印预览功能（浏览器自带打印功能可替代，用户可PDF后打印）
- 导出文件加密/密码保护
- 分享导出文件链接（无后端无法生成可分享链接）
- 导出历史记录管理（不记录用户导出过哪些文件）
