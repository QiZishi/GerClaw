# 用药审查 — 产品规格

> 模块：用药审查 | 优先级：P0 | 基于PRD.md第4节生成

---

## 1. 模块概述

用药审查模块是GerClaw平台的医疗安全核心组件，为老年患者用药安全提供三重保障：DDI药物-药物相互作用检查、Beers Criteria老年人潜在不适当用药(PIM)筛查、剂量合理性与重复用药检查。MVP阶段采用纯前端规则引擎实现，所有审查数据（DDI规则库、Beers标准、老年剂量参考表）以JSON/TS数据文件内置在前端，无需依赖后端服务。模块支持两种使用场景：(1) 与五大处方模块联动，药物处方生成时自动触发审查；(2) 医生端独立入口，手动输入用药列表进行专项审查。审查结果按四级风险分级输出（禁忌/严重/中等/轻微），对应红/橙/黄/蓝四色标识，并给出明确的临床处理建议和老年患者特殊提示。所有审查结论仅供医生参考，不替代专业药师/医师判断，必须附带医疗免责声明。

## 2. 用户故事

| 编号 | 作为... | 我想要... | 以便于... |
|------|---------|----------|----------|

## 4. 交互流程

### 4.1 主流程（Happy Path）——独立审查

```
医生端独立审查流程：
1. 医生点击欢迎页"用药审查"卡片 / 输入框⚡菜单 / 医生端工具面板入口
2. 右侧面板自动展开用药审查界面
3. 界面展示：
   - 患者年龄输入框（默认65岁）
   - 药物输入区：逐行添加药物（输入框+添加按钮，支持回车添加）
   - 已添加药物列表（可删除）
   - "开始审查"主按钮（≥2种药物时激活，1种也可检查Beers/剂量）
4. 医生输入患者年龄，逐行添加待审查药物（支持通用名/商品名）
5. 药物添加过程中实时匹配归一化：
   - 识别到商品名时显示 "(通用名: xxx)" 提示
   - 未识别药物显示灰色"未收录"提示（仍可审查）
6. 医生点击"开始审查"按钮
7. 主聊天区展示执行过程可视化（工具调用卡片）：
   - 📋 加载DDI规则库... ✓
   - 🔍 检查药物相互作用(DDI)... (找到X组交互)
   - 📖 筛查Beers老年人不适当用药... (发现X项PIM)
   - ⚖️ 检查剂量合理性... (检查完成)
   - 🔄 检查重复用药... (完成)
   - 📊 汇总风险分级...
8. 右侧面板显示审查结果（按风险从高到低排序）：
   - 顶部：审查概要（共检查X种药物，发现Y项风险：禁忌a/严重b/中等c/轻微d）
   - 🔴 禁忌级（如有）：红色卡片置顶，每条包含药物对、机制、后果、强烈建议
   - 🟠 严重级：橙色卡片
   - 🟡 中等级：黄色卡片
   - 🔵 轻微级：蓝色卡片
   - 多重用药预警（如≥5种）
   - 底部：免责声明 + 导出/复制按钮
9. 医生可：
   - 点击每条风险展开/折叠详情
   - 修改药物列表后重新审查
   - 点击导出Markdown / 复制结果
   - 对结果有疑问可在聊天区继续询问
```

### 4.2 主流程（Happy Path）——处方联动审查

```
药物处方联动流程：
1. 医生/患者触发五大处方生成
2. 信息收集完成后系统生成处方
3. 药物处方生成完毕，自动在后台调用用药审查引擎（无需用户点击）
4. 主聊天区工具调用卡片显示"用药审查"步骤
5. 审查完成后：
   - 无风险：药物处方底部显示"✅ 用药审查通过，未发现明显风险"
   - 有风险：药物处方中对应药物高亮标记
     * 禁忌/严重级：药物名标红/橙，右侧显示警告图标，点击展开风险详情
     * 中等/轻微级：药物名标黄/蓝，显示信息图标
   - 处方预览底部嵌入完整审查报告摘要
   - "查看详细审查报告"按钮可展开完整审查结果
6. 医生编辑药物处方时，修改后自动重新触发审查
7. 医生审核处方时需确认已知晓用药风险
```

### 4.3 异常流程

| 异常场景 | 系统行为 | 用户提示 |
|---------|---------|---------|
| 药物列表为空或只有1种（无法做DDI） | DDI检查跳过，仍执行Beers/剂量/重复检查 | "仅1种药物，跳过药物相互作用检查，继续执行Beers和剂量筛查" |
| 药物名称未在内置库中识别 | 仍进行已识别药物的检查，未识别药物标注提示 | "'XXX'未在内置药物库中收录，无法进行完整交互检查，结果仅供参考" |
| 所有检查均未发现问题 | 显示"通过"状态和安全提示 | "✅ 基于内置规则库审查，未发现已知的明显用药风险。仍需结合患者具体情况临床判断。" |
| 检测到禁忌级药物组合 | 红色醒目置顶显示，强烈警示 | "🔴【禁忌】检测到绝对禁忌的药物组合！禁止联用！请立即调整用药方案！" |
| ≥75岁患者中度DDI升级 | 标注风险升级原因 | "⚠️ 患者≥75岁，该药物相互作用风险从中度升级为严重，需特别注意" |
| 重复用药检测到 | 明确指出重复的药物/类别 | "🔄 检测到重复用药：[药A]和[药B]通用名相同/属于同一类别，建议调整" |
| 审查规则库版本较旧 | 提示规则库版本和更新建议 | "当前规则库版本：2023版Beers，建议定期更新医学知识" |
| 导出失败 | 提示重试，同时支持复制文本 | "导出失败，请重试。您也可以使用'复制全部'功能复制结果。" |

## 5. 界面/API规格

### 5.1 界面

**5.1.1 独立审查面板（右侧动态面板）**

- 面板宽度：380-450px（可拖拽320-500px）
- 顶部：
  - 标题"💊 用药审查"
  - 关闭按钮
  - 审查中显示进度状态
- 配置区（始终显示）：
  - 患者年龄：数字输入框（默认65，范围18-120），标签"患者年龄"
  - 多重用药计数：实时显示"当前X种药物"
- 药物输入区：
  - 输入框：placeholder"输入药物通用名或商品名，回车添加"
  - 支持回车快速添加，或点击"+ 添加"按钮
  - 输入时下拉显示匹配建议（内置药物库模糊匹配）
- 已添加药物列表：
  - 每个药物显示为标签：药物名 + ×删除按钮
  - 商品名显示为：商品名 (通用名: xxx)
  - 未识别药物显示为灰色标签
- 操作按钮区：
  - "清空列表"次级按钮
  - "示例药物（老年高危组合）"快捷填充按钮（用于演示）
  - "开始审查"主按钮（药物≥1时激活）
- 审查结果区（审查完成后显示，可滚动）：
  - 审查概要卡片：
    * 检查药物数量
    * 风险统计：🔴禁忌 X / 🟠严重 X / 🟡中等 X / 🔵轻微 X
    * 多重用药预警（如适用）
  - 风险列表（按等级从高到低）：
    * 每条风险为可折叠卡片
    * 卡片头：颜色条 + 等级图标 + 药物/问题摘要
    * 展开内容：作用机制、可能后果、临床建议、老年提示（👴标识）
  - 操作按钮：复制全部、导出Markdown
  - 免责声明区（灰色背景，固定底部）

## 6. 数据模型

```typescript
// 风险等级
type RiskLevel = 'contraindicated' | 'major' | 'moderate' | 'minor';

// 风险等级显示配置
interface RiskLevelConfig {
  level: RiskLevel;
  label: string;           // 中文标签：禁忌/严重/中等/轻微
  color: string;           // 主色
  bgColor: string;         // 背景色
  borderColor: string;     // 边框色
  icon: string;            // 图标
  order: number;           // 排序优先级（0最高）
}

// 单个药物输入
interface DrugInput {
  name: string;            // 输入的原始名称
  normalized_name?: string; // 归一化通用名（如识别到）
  brand_name?: string;     // 商品名（如识别到）
  category?: string;       // 药物类别
  recognized: boolean;     // 是否在内置库中识别
}

// DDI交互规则
interface DDIRule {
  drug_a: string;          // 药物A（通用名或类别）
  drug_b: string;          // 药物B（通用名或类别）
  severity: RiskLevel;     // 严重级别
  mechanism: string;       // 作用机制
  consequence: string;     // 可能后果
  recommendation: string;  // 临床处理建议
  elderly_note?: string;   // 老年人特殊提示
  is_category_match?: boolean; // 是否为类别匹配
  source?: string;         // 来源依据
}

// Beers PIM条目
interface BeersCriterion {
  drug_or_class: string;   // 药物或类别
  rationale: string;       // 风险理由
  recommendation: string;  // 建议
  severity: RiskLevel;     // 风险等级（Beers标准中多为strong/weak）
  age_group?: string;      // 适用年龄组（如≥65岁、≥75岁）
  avoid_in_condition?: string[]; // 特定疾病/情况避免使用
  alternative?: string;    // 替代方案建议
}

// 剂量参考规则
interface DoseRule {
  drug_name: string;       // 药物名
  standard_dose: string;   // 常规成人剂量
  elderly_dose: string;    // 老年人推荐剂量（通常减半）
  max_dose?: string;       // 最大剂量
  adjustment_note?: string; // 剂量调整说明（如肾功调整）
}

// 重复用药检测结果
interface DuplicateFinding {
  type: 'same_generic' | 'same_class'; // 重复类型
  drugs: string[];         // 重复的药物列表
  category?: string;       // 药物类别（同类重复时）
  severity: RiskLevel;
  recommendation: string;
}

// 单条风险项
interface RiskItem {
  id: string;
  type: 'ddi' | 'beers' | 'dose' | 'duplicate' | 'polypharmacy';
  severity: RiskLevel;
  title: string;           // 风险标题（一句话摘要）
  drugs_involved: string[]; // 涉及药物
```

## 7. 非功能要求

| 维度 | 要求 |
|------|------|
| 性能 | 单患者用药审查（≤10种药物）前端计算响应时间<200ms；规则库初始化加载<500ms；无需网络请求即可完成核心审查 |
| 可靠性 | 内置规则库前端静态导入，无网络依赖；未识别药物不阻断审查流程；异常输入（空值、特殊字符）优雅降级处理 |
| 安全性 | 所有审查报告强制显示免责声明；禁止给出"绝对安全"的确定性结论；禁忌/严重级风险必须醒目提示；审查数据不持久化到localStorage（仅会话内有效，保护隐私） |
| 可维护性 | DDI规则、Beers标准、剂量表均为独立JSON/TS文件，便于后续更新；规则数据与审查逻辑分离；规则版本号显示在结果中 |
| 可观测性 | 控制台记录审查执行日志（药物列表、风险数量、耗时）；高风险发现可埋点统计；规则命中率统计（用于后续扩充规则库） |
| 适老化 | 风险等级文字≥18px（患者端≥20px）；颜色编码同时搭配文字和图标，不单纯依赖颜色区分；支持TTS语音朗读完整审查报告；医学术语尽量配通俗解释 |
| 准确性 | DDI规则参考DrugBank/Micromedex老年高危组合；Beers标准使用2023版核心条目；剂量参考老年人用药指南；规则库覆盖≥100种老年常用药物和≥50组高危DDI |

## 8. 不做什么（Out of Scope）

- 不对接外部商业药物数据库（如DrugBank、Lexicomp、药智网等），MVP使用内置精选规则库
- 不实现肝肾剂量调整（需eGFR/Child-Pugh等完整检验数据，P1阶段）
- 不实现药物-疾病禁忌、药物-食物交互、药物-过敏检查（P1）
- 不实现药物-基因交互(PGx)、QT延长叠加计算、抗胆碱能负荷评分等高级检查（P1）
- 不实现STOPP/START标准（仅Beers，P1补充）
- 不实现中成药/保健品交互规则（先覆盖西药核心规则）
- 不实现审查结果的医生电子签名/双签确认（P1 HITL流程）
- 不实现患者用药审查历史记录持久化（P1健康档案模块）
- 不做全量药物库（仅覆盖老年科常用的100-200种药物，可扩充）
- 不实现RAG检索最新FDA警告/文献补充（二阶段）
- 不自动给出替代药物处方建议（仅提示风险，具体替代方案由医生决定）
