## 1. 产品核心理念

GerClaw要解决的核心问题是：**老年人想用AI获取医疗健康服务，但现有产品交互复杂、字小难操作、打字困难、医疗输出不可信**。我们通过语音优先的适老化交互+专业的老年医学评估体系+医疗安全底线，让老年患者能"说句话就获得专业健康指导"，让老年科医生能"AI辅助下3分钟完成CGA评估"。

我们做这件事是因为：中国2.8亿老年人面临数字鸿沟，老年综合评估(CGA)是老年医学核心但传统方式耗时30-60分钟，AI可以显著降低门槛、提升效率，但必须守住医疗安全底线。

## 2. 好的行为 = 应该做的

- **语音一句话就能用**：患者端打开页面，最大的按钮是麦克风，点一下就能说话，不需要学任何操作
- **大字体大按钮看得清点得准**：老年模式下正文≥18px，按钮≥48px，高对比度，任何操作都有明确文字提示
- **医疗建议带来源和免责声明**：每条医学建议都标明依据来源，末尾必有"仅供参考，不构成诊断，请咨询专业医生"
- **流式输出打字机效果**：AI回复逐字流式输出，老人可以边看边读，不是等半天出一大段
- **语音朗读自然流畅**：TTS用冰糖音色，语气温柔体贴，语速适中，像家里晚辈关心长辈一样说话
- **工具调用过程透明**：搜索了什么、调用了什么检查，用户能看到，不做黑盒决策
- **危险症状立即报警**：识别到胸痛、呼吸困难、意识障碍等急重症信号，立即红色警告"请立即拨打120或前往急诊"
- **处方和评估结构化输出**：五大处方和CGA报告不是大段文字，是分栏分区的清晰卡片，重点数据高亮

## 3. 坏的行为 = 禁止/避免的

- **禁止无证据的确定性结论**：没有本轮可追溯 citation 时不能说“您患有高血压/糖尿病/冠心病”；有本地、联网或用户资料证据并由 Runtime 标记时可给出临床结论。患者端仅在整段末尾提示一次勿自行改药、请医生复核的风险，医生端直接呈现结论、证据和下一步。
- **禁止编造医学文献**：不能编造"《中国老年高血压指南2024》指出..."这种假引用，有依据才标，没有就不标
- **禁止使用专业术语不给解释**：患者端不能说"您的BMI为22.5"就完了，必须说"您的体重指数是22.5，在正常范围内，很好！"
- **禁止小字体低对比度**：患者端老年模式下不允许出现浅灰文字(对比度<4.5:1)、小于16px的文字、纯图标按钮无文字
- **禁止自动播放声音**：除用户主动点击的TTS播放外，任何自动播放声音都是干扰
- **禁止长表单一次性展示**：CGA量表不是一上来显示50道题，而是一道一道对话引导，像聊天一样
- **禁止API Key硬编码在代码中**：所有密钥必须通过环境变量注入，不得提交到仓库
- **禁止错误暴露技术堆栈**：API失败时显示"网络连接有点问题，请稍后重试"，不是"fetch failed with CORS error"
- **禁止读取origin目录**：严禁从gerclaw-main-origin或gerclaw-design-origin读取任何文件作为参考

## 4. 成功的定性标准

我们判断产品做对了，当：

- 一位65-75岁的老年人，不需要任何人指导，能在30秒内发现麦克风按钮并开始语音对话
- 语音转文字准确率足够高，老人说带口音的普通话也能正确识别（"我头有点晕"不会识别成"我头有点云"）
- 五大处方报告打印出来给医生看，医生认为"结构和内容基本合理，参考价值高"
- CGA对话评估结束后，老人不觉得自己在"做题"，觉得"就是聊了会儿天"
- 所有医疗输出都带免责声明，没有一个地方出现确定性诊断结论
- Demo在IGA Pages上流畅运行，首屏3秒内可见，对话2秒内开始流式回复
- 开发者不需要修改代码，只改环境变量就能切换GPT-4o/qwen-plus/claude

## 5. 失败的信号

我们需要警惕和立即修复的情况：

- 老人用户在语音按钮上反复点击超过3次（说明按钮不够明显或录音反馈不清）
- TTS播放的声音机械感重、语速过快或过慢，老人听不清
- 处方/评估报告导出PDF后格式错乱、内容截断
- 网络断开时页面白屏或无提示，用户不知道发生了什么
- 主模型API调用失败但没有自动切换到备用模型
- 老年模式下仍出现14px以下的文字或纯图标按钮
- 搜索功能返回结果但AI回复中没有标注引用角标
- localStorage被写满导致新会话无法保存

## 6. 决策原则

当面临产品决策时，优先级排序：

1. **医疗安全 > 一切**：有安全隐患的功能宁可不上也不能带病上线
2. **老年人能用 > 功能丰富**：少即是多，核心功能（对话、语音、处方、评估）做好比做100个没人会用的功能强
3. **语音优先 > 文字输入**：患者端默认交互是语音，打字是备选
4. **明确告知 > 假装聪明**：不确定就说"这个问题我不太确定，建议咨询医生"，比胡说八道强一万倍
5. **结构化清晰 > 大段文字**：医疗输出用卡片、分点、表格，不用大段散文
6. **浅色高对比 > 深色酷炫**：医疗产品以可读性为先，默认浅色主题，深色主题是可选
