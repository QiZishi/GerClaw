### 问题描述
- 医生端和患者端无法切换
- 医生端在当前版本无法打开

### 修复要求

- 医生端所有页面可正常加载，无白屏/报错
- 切换后状态正确重置，无数据残留

### 验收标准
- [ ] 点击切换按钮可正常切换到医生端
- [ ] 医生端主页可正常显示
- [ ] 从医生端可正常切换回患者端

## P0-2. 聊天功能完全不可用

### 问题描述
1. 模型始终显示"思考中"状态，不输出思维链内容
2. 点击思考圆圈会中断模型聊天
3. 中断后，后续所有对话都只显示"模型对话被中断"
4. 聊天功能完全瘫痪，无法正常使用

### 修复要求
1. **流式输出问题**：排查 SSE/WebSocket 连接，确保模型输出正常返回并渲染
2. **思考状态处理**：
   - 思考中状态应可折叠/展开思维链内容
   - 点击思考圆圈不应中断对话，仅控制思维链显示/隐藏
3. **中断恢复机制**：
   - 单次中断不影响后续对话
   - 中断后可正常发起新对话
   - 移除"模型对话被中断"的持久错误状态

### 验收标准
- [ ] 发送消息后，模型正常输出回复（含思维链）
- [ ] 思考中状态点击不中断对话
- [ ] 手动中断后可正常发起新对话
- [ ] 连续多次对话无异常

## P0-3. 老年综合评估语音播放逻辑错误

### 问题描述
1. 用户按下停止语音播放按钮后，切换题目时仍自动播放
2. 仍然使用实时 TTS 模型调用生成语音，而非播放预录音频文件
3. **严重浪费 token 资源！**

### 修复要求

#### 播放状态控制（必须修复）
```
用户状态机：
  ┌─────────────────────────────────────────────────────┐
  │  进入评估 → 默认自动播放                              │
  │      ↓                                               │
  │  用户点击停止 → 状态切换为"手动模式"                   │
  │      ↓                                               │
  │  后续所有题目 → 停止自动播放                           │
  │      ↓                                               │
  │  用户点击播放 → 状态切换为"自动模式"                   │
  │      ↓                                               │
  │  后续所有题目 → 恢复自动播放                           │
  └─────────────────────────────────────────────────────┘
  
  关键：状态在整个评估会话内保持，切换题目不重置！
```

#### 预录音频调用（必须修复）
- **禁止**在评估过程中调用 TTS 模型
- **必须**使用 `/public/audio/scales/` 目录下的预录音频文件
- 播放逻辑：`new Audio(\`/audio/scales/${scaleId}_${questionId}.mp3\`)`

### 验收标准
- [ ] 点击停止按钮后，切换题目不再自动播放
- [ ] 点击播放按钮后，后续题目恢复自动播放
- [ ] 播放的音频为预录音频文件（检查网络请求，无 TTS API 调用）
- [ ] 播放/停止状态在整个评估会话内保持

# 功能优化需求

## 1. 消息操作按钮功能完善

### 1.1 按钮列表
每条 AI 回复底部需显示以下操作按钮：
- 复制按钮
- 语音播放按钮
- 导出按钮（图标为分享样式）
- 点赞按钮
- 点踩按钮
- 重新生成按钮
- 三点菜单按钮（包含：转为文档编辑、删除）

### 1.2 各按钮功能规格

| 按钮 | 功能描述 | 交互细节 |
|------|----------|----------|
| **复制** | 将 AI 最终回复内容复制到剪贴板 | 点击后显示"已复制"提示，2秒后消失 |
| **语音播放** | 将 AI 最终回复内容转语音播放 | 默认不播放；点击开始播放，按钮切换为"停止"图标；再次点击停止播放 |
| **导出** | 导出对话内容 | 弹出导出面板，默认勾选当前回复及对应提问，支持多选；导出格式：PNG图片、PDF、DOCX、Markdown |
| **点赞** | 对 AI 回复进行正面评价 | 点击后弹出评价输入框，支持文字反馈 |
| **点踩** | 对 AI 回复进行负面评价 | 点击后弹出评价输入框，支持文字反馈 |
| **重新生成** | 重新生成 AI 回复 | 调用模型重新生成，保留原回复直到新回复完成 |
| **转为文档编辑** | 将回复内容转入右侧编辑栏 | 内容渲染到右侧栏，支持编辑；完成后可复制或导出（PDF/DOCX/Markdown） |
| **删除** | 删除对话 | 弹出确认框，默认勾选当前对话，支持多选；提供"确认删除"和"取消"按钮 |

### 1.3 三点菜单
- 位置：消息操作按钮最右侧
- 点击后展开下拉菜单，包含：转为文档编辑、删除
- 点击菜单外区域关闭菜单

## 5. 老年综合评估语音优化

### 5.1 预录音频
- **范围**：所有量表题目和选项的语音
- **存储位置**：`/public/audio/scales/` 目录
- **文件格式**：MP3
- **命名规则**：`{scale_id}_{question_id}.mp3`

### 5.2 播放控制
- 进入题目页面自动播放该题语音
- 播放按钮状态：播放中显示"暂停"图标，暂停/停止显示"播放"图标
- 用户操作：
  - 点击暂停：停止播放，后续题目不再自动播放
  - 点击播放：开始播放，后续题目自动播放
  - 切换题目：根据当前播放状态决定是否自动播放新题目
  - 返回量表选择页：停止播放

### 5.3 语音答题识别
- 支持识别内容：
  - 选项文本（如"完全同意"）
  - 选项序号（如"选项1"、"第1题"）
  - 选项字母（如"A"、"a"）
- 识别成功后：高亮显示对应选项，**不自动跳转下一题**
- 用户需手动点击"下一题"按钮确认

### 6.1 完成后界面

```
作答完毕
[量表名称1]、[量表名称2] 已作答完成

[重新评估] [继续作答其他量表] [查看评估报告]
```

### 6.2 按钮功能

|------|------|

| 查看评估报告 | 汇总所有已作答量表，生成综合评估报告，显示在右侧栏 |

### 6.3 评估报告
- 生成方式：调用模型整合所有量表结果
- 显示位置：右侧栏
- 支持操作：编辑、复制、导出（PDF/DOCX/Markdown）

## 9. 作答状态持久化

### 9.1 状态保持
- 用户在评估期间，所有作答记录保留在内存中
- 每次生成报告时，传入全部历史作答数据

### 9.2 退出处理
| 场景 | 退出提示 | 保存策略 |

| 已有评估报告 | "您已完成量表评估，是否要退出？" | 保存最近一次评估报告 |

### 9.3 历史记录

  - 显示上一次评估报告
  - 提供选项：继续作答 / 重新作答 / 查阅报告
